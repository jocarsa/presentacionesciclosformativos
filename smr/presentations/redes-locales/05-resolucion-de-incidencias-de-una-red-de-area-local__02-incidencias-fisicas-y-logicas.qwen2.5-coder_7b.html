Portada

Resolución de incidencias de una red de área local

Objetivo: Comprender y gestionar las incidencias físicas y lógicas en redes locales.

Incidencias físicas vs. lógicas

  • Incidencias físicas: Problemas relacionados con el hardware de la red.
  • Incidencias lógicas: Problemas relacionados con el software y la configuración de la red.

Ejemplo de incidencia física

Un cable Ethernet roto impide la comunicación entre dos computadoras.

Ejemplo de incidencia lógica

Un error en el archivo de configuración del router causa una desconexión de todos los dispositivos.

Caso práctico: Problema de conectividad

  • Un usuario no puede acceder a Internet desde su computadora.
  • Puede ser una incidencia física (able roto) o lógica (configuración incorrecta).

Proceso de diagnóstico

  1. Identificar el problema.
  2. Determinar si es física o lógica.
  3. Aplicar soluciones específicas.

Buenas prácticas en resolución de incidencias

  • Mantener un registro detallado de todas las incidencias.
  • Realizar pruebas de red regularmente.
  • Tener herramientas de diagnóstico disponibles.

Errores comunes en resolución de incidencias

  • No identificar correctamente el tipo de incidencia.
  • No realizar pruebas adecuadas.
  • No documentar los pasos realizados y soluciones aplicadas.

Comparativa: Incidencias físicas vs. lógicas

Incidencia Física Incidencia Lógica
Problemas con el hardware. Problemas con el software o configuración.
Soluciones físicas (reemplazar componentes). Soluciones lógicas (corregir configuraciones, reinstalar software).

Mini-quiz: Identificar el tipo de incidencia

¿Qué tipo de incidencia es?

  • Física
  • Lógica

Checklist: Resolución de incidencias físicas

  1. Verificar la conectividad física.
  2. Reemplazar cables dañados.
  3. Comprobar los puertos del switch o router.

Checklist: Resolución de incidencias lógicas

  1. Revisar la configuración del router.
  2. Verificar los archivos de configuración del switch.
  3. Realizar pruebas de conectividad.

Normativa: Seguridad en la red

  • Seguir políticas de seguridad corporativas.
  • Realizar auditorías periódicas de la red.
  • Documentar todos los cambios y soluciones aplicadas.

  2. ¿Qué es una incidencia lógica? Un problema con el software o configuración de la red.
