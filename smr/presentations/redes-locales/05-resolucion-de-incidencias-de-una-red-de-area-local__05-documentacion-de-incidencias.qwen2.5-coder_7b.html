Documentación de incidencias en una red local

Objetivo: Aprender los pasos y buenas prácticas para documentar adecuadamente las incidencias en una red local.

Definición de Documentación de Incidencias

La documentación de incidencias es el proceso de registrar, clasificar y almacenar información sobre problemas o fallas que ocurren en una red local.

Importancia de la Documentación

  • Mantiene un historial de problemas resueltos.
  • Ayuda a identificar patrones y causas recurrentes.
  • Sirve como base para el mantenimiento preventivo.
  • Facilita la comunicación entre equipos y departamentos.

Estructura Básica de una Entrada de Incidencia

  • Fecha y hora del incidente.
  • Descripción detallada del problema.
  • Pasos para reproducir el error (si aplicable).
  • Impacto en la red o los usuarios.
  • Estado actual de la incidencia (abierto, en proceso, resuelto).

Formatos de Documentación

  • Sistema de gestión de incidentes (SIEM).
  • Hoja de incidencia personalizada.
  • Software de seguimiento de problemas.
  • Email o sistema de tickets.

Buenas Prácticas en la Documentación

  • Mantener la información actualizada y precisa.
  • Usar un lenguaje claro y conciso.
  • Incluir evidencias (capturas de pantalla, logs, etc.).
  • Asignar responsabilidades claras.
  • Realizar seguimientos periódicos.

Ejemplo de Entrada de Incidencia


Fecha: 15/04/2023, 14:30
Descripción: El servidor web no responde a las solicitudes.
Impacto: No se pueden acceder a los recursos del sitio web.
Estado: Abierto

Errores Comunes en la Documentación

  • No incluir suficiente información.
  • Usar lenguaje técnico innecesariamente complicado.
  • No asignar responsabilidades.
  • No realizar seguimientos.
  • No mantener la documentación actualizada.

Checklist de Documentación de Incidencias

  • Fecha y hora del incidente.
  • Descripción detallada del problema.
  • Pasos para reproducir el error (si aplicable).
  • Impacto en la red o los usuarios.
  • Evidencias (capturas de pantalla, logs, etc.).
  • Estado actual de la incidencia.
  • Responsables asignados.

Caso Práctico: Documentación de una Incidencia

Un equipo de TI reporta que el correo electrónico no funciona. Documentar este incidente correctamente es crucial para su resolución.

Proceso de Documentación

  1. Recepción del informe inicial.
  2. Registro en el sistema de gestión de incidentes.
  3. Captura de evidencias (logs, capturas de pantalla).
  4. Asignación de responsabilidades.
  5. Seguimiento periódico del estado de la incidencia.
  6. Resolución y documentación final.

Normativa Relacionada con la Documentación de Incidencias

  • Norma ISO/IE 27035:2016 (Seguridad en redes locales).
  • Reglamento General de Protección de Datos (RGPD) - Artículos 28 y 34.
  • Política Interna de Seguridad de la Información.
