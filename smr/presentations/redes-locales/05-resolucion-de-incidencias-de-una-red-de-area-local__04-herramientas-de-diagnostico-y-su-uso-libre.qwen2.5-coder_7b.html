Portada

Resolución de incidencias de una red de área local

Objetivo: Aprender a utilizar herramientas de diagnóstico para resolver problemas en redes locales.

Herramientas de diagnóstico básicas

  • Ping
  • Traceroute
  • Netstat
  • Dig
  • Telnet
  • Nslookup
  • Mtr
  • Wireshark

Ping: Comprobación de conectividad

ping www.example.com

Verifica si un host está respondiendo a los paquetes ICMP.

Traceroute: Ruta del paquete

traceroute www.example.com

Muestra la ruta que el paquete toma para llegar a un destino.

Netstat: Información de red activa

netstat -an

Muestra las conexiones, puertos y protocolos en uso.

Dig: Consultas DNS

dig example.com

Obtiene información detallada sobre un dominio DNS.

Telnet: Prueba de conexión a servicios

telnet www.example.com 80

Verifica la conectividad a un puerto específico en un servidor.

Nslookup: Consultas DNS

nslookup example.com

Muestra información sobre el dominio y su resolución DNS.

Mtr: Combina ping y traceroute

mtr www.example.com

Combina las funciones de ping y traceroute en un solo comando.

Wireshark: Captura de tráfico

Captura y analiza el tráfico de red para diagnóstico avanzado.

Buenas prácticas en diagnóstico

  • Reproduce los pasos del problema
  • Documenta todo el proceso
  • Utiliza herramientas adecuadas para cada caso
  • Comunica resultados claramente

Errores comunes en diagnóstico

  • No reproduce el problema
  • No documenta suficiente información
  • Usa herramientas incorrectas para el caso
  • No comunica resultados claramente

Ejemplo de diagnóstico

Síntoma: No puedo acceder a www.example.com.

Pasos:

  1. Ping www.example.com
  2. Traceroute www.example.com
  3. Dig example.com
  4. Telnet www.example.com 80

Caso práctico: Problema de conectividad

Síntoma: Computadora no puede acceder a Internet.

Pasos:

  1. Comprueba la conexión física
  2. Reinicia el router y el computador
  3. Ping 8.8.8.8 (Google DNS)
  4. Traceroute www.google.com

Mini-quiz: Herramientas de diagnóstico

Checklist de diagnóstico

  • Comprueba la conexión física
  • Reinicia el router y el computador
  • Ping a puertos conocidos (80, 443)
  • Traceroute al destino
  • Dig para verificar DNS

Actividad: Diagnóstico de problemas de red

Resuelve el siguiente problema:

Síntoma: No puedo acceder a correo electrónico.

Pasos:

  1. Comprueba la conexión física
  2. Reinicia el router y el computador
  3. Ping al servidor de correo (MX)
  4. Traceroute al servidor de correo
  5. Dig para verificar DNS del correo

Resumen
