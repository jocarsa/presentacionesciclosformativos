Direcciones IP: IPv4 y IPv6

Las direcciones IP son identificadores únicos que asignan a cada dispositivo en una red.

IPv4 (Internet Protocol Version 4)

  • Dirección de 32 bits (4 octetos)
  • Formato: AAA.BBB.CCC.DDD (donde A, B,, D son números entre 0 y 255)
  • Ejemplo: 192.168.1.1
  • Limitaciones: Direcciones limitadas y escasez de IPs

IPv6 (Internet Protocol Version 6)

  • Dirección de 128 bits (8 grupos de 4 hexadecimales)
  • Formato: AAAA.BBBB.CCCC.DDDD.EEEE.FFFF.GGGG.HHHH
  • Ejemplo: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Avantage: Direcciones infinitas y extensión de la red

Ejemplo de Asignación IPv4

Asignación estática:

IP: 192.168.1.100
Máscara de subred: 255.255.255.0
Puerta de enlace: 192.168.1.1

Ejemplo de Asignación IPv6

Asignación automática (DHCPv6):

IP: 2001:db8::1
Máscara de subred: /64
Puerta de enlace: fe80::1

Caso Práctico: Configuración IPv4

Configurar una máquina virtual con IP estática:

  1. Ir a la configuración de red
  2. Elegir "IPv4"
  3. Seleccionar "Usar el siguiente dirección IP"
  4. Ingresar la dirección, máscara y puerta de enlace

Caso Práctico: Configuración IPv6

Configurar una máquina virtual con IP automática (DHCPv6):

  1. Ir a la configuración de red
  2. Elegir "IPv6"
  3. Seleccionar "Obtener automáticamente el dirección IPv6"

Buenas Prácticas: IPv4

  • Usar DHCP para asignación dinámica de IPs
  • Configurar puertas de enlace y máscaras de subred correctamente
  • Mantener registros de IP asignadas

Buenas Prácticas: IPv6

  • Configurar puertas de enlace y prefijos de subred correctamente
  • Mantener registros de IPs asignadas
  • Usar herramientas de diagnóstico para verificar conectividad

Errores Comunes: IPv4

  • Máscara de subred incorrecta
  • Puerta de enlace no configurada correctamente
  • Dirección IP fuera del rango de la subred

Errores Comunes: IPv6

  • Prefijo de subred incorrecto
  • No tener configurada una puerta de enlace
  • Dirección IP fuera del rango de la red

Comparativa IPv4 vs IPv6

IPv4 IPv6
Dirección bits 32 128
Ejemplo dirección 192.168.1.1 2001:db8::1
Direcciones disponibles 4,3 billones Más de 3.4 x 10^38

Checklist Configuración IPv4

  • Verificar dirección IP asignada
  • Comprobar máscara de subred
  • Asegurar puerta de enlace configurada
  • Validar conectividad con ping a una máquina conocida

Checklist Configuración IPv6

  • Verificar dirección IP asignada
  • Comprobar prefijo de subred
  • Asegurar puerta de enlace configurada
  • Validar conectividad con ping a una máquina conocida

Actividad: Configuración de Direcciones IP

Configurar la dirección IP y máscara de subred en un dispositivo virtual.

  1. Ingresar a la configuración de red
  2. Elegir IPv4 o IPv6 según corresponda
  3. Asignar una dirección IP estática o solicitar una automática (DHCPv6)
  4. Verificar la configuración con ping y tracert/traceroute

Resumen: Direcciones IP IPv4 y IPv6

  • Direcciones IP identifican dispositivos en una red
  • IPv4 usa 32 bits, IPv6 usa 128 bits
  • IPv6 ofrece más direcciones y extiende la capacidad de la red

Normativa: Direcciones IP

La normativa ISO/IE 25947 define los estándares para IPv4 y IPv6.
