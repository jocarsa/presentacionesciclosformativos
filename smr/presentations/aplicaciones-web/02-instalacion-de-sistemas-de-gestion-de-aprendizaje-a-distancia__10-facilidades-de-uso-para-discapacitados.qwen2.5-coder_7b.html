Portada

Facilidades de uso para discapacitados en sistemas de gestión de aprendizaje a distancia

Objetivo: Comprender las herramientas y configuraciones que facilitan el acceso y participación de personas con discapacidades.

Definición de accesibilidad web

Accesibilidad web significa diseñar sitios web que sean utilizables por todas las personas, independientemente de sus capacidades o necesidades específicas.

Tipos principales de discapacidades relevantes para la accesibilidad web

  • Motricidad fina y motricidad gruesa (DAFM)
  • Visiones reduidas oeguera total (DVRCT)
  • Discapacidades auditivas (DAU)
  • Trastornos del espectro autista (TSA)

Herramientas de accesibilidad web

  • Navegadores accesibles
  • Leer texto aloud (lectores de pantalla)
  • Contraste alto y modos de visualización alternativos
  • Teclas de acceso rápido

Configuración del navegador para personas con discapacidades visuales

Navegadores accesibles: Google Chrome, Firefox, Safari (con VoiceOver)

Contraste alto: Ajustar el contraste en las preferencias del navegador.

Uso de lectores de pantalla

Lectores de pantalla como JAWS, NVDA y VoiceOver para personas con discapacidades visuales.

Configuración del navegador para personas con discapacidades auditivas

Subtítulos y leyendas de audio: Asegurar que todos los videos y audios tengan subtítulos o descripciones.

Uso de teclas de acceso rápido

Configuración de teclas de acceso rápido en navegadores para personas con discapacidades motrices finas y gruesa.

Ejemplo: Diseño accesible de un formulario web

  • Etiquetas claras y descriptivas
  • Uso de placeholders
  • Validación en tiempo real
  • Accesibilidad por teclado

Caso: Sistema de gestión de aprendizaje accesible para personas con discapacidades visuales

Implementación de contraste alto, leyendas de audio y lectores de pantalla en un sistema LMS.

Buenas prácticas en la creación de contenido accesible

  • Uso de semántico
  • Etiquetas ARIA (Accessible Rich Internet Applications)
  • Pruebas con personas con discapacidades reales

Errores comunes en la accesibilidad web

  • No usar colores únicos para diferenciar elementos
  • No proporcionar subtítulos a videos sin sonido
  • No permitir el acceso por teclado solo

Normativa y regulaciones en accesibilidad web

WCAG (Web Content Accessibility Guidelines): Estándar global para la accesibilidad de contenido web.

Chequeo de accesibilidad web

  • Usar herramientas como Wave, Axe o Lighthouse
  • Realizar pruebas con personas con discapacidades reales
