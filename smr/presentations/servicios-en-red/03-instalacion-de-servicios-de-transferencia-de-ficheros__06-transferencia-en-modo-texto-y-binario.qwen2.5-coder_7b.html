Objetivo: Comprender los conceptos básicos y procedimientos para la instalación de servicios de transferencia de ficheros en modo texto y binario.

Transferencia en modo texto

El modo texto se utiliza para transferir archivos que contienen solo caracteres imprimibles. Ejemplos: documentos, hojas deálculo, scripts de programación.

Transferencia en modo binario

El modo binario se utiliza para transferir archivos que contienen datos no imprimibles o complejos. Ejemplos: imágenes, videos, ejecutables.

Ejemplo de transferencia en modo texto

  • SCP (Secure Copy Protocol)
  • SFTP (SSH File Transfer Protocol)
  • FTPS (FTP over SSL/TLS)

Ejemplo de transferencia en modo binario

  • SCP
  • SFTP
  • FTPS
  • FTP (File Transfer Protocol)

Caso de estudio: Transferencia de imágenes

Se requiere transferir una imagen JPEG de 5MB desde un servidor local a otro en la nube.

Caso de estudio: Transferencia de scripts

Se necesita transferir un script de 10KB desde un equipo local a un servidor remoto.

Proceso de instalación de SCP

  1. Instalar OpenSSH Server en el sistema local.
  2. Instalar OpenSSH Client en el sistema remoto.
  3. Configurar las claves SSH para autenticación sin contraseña.
  4. Ejecutar el comando SCP desde la terminal.

Buenas prácticas de transferencia en modo texto

  • Usar protocolos seguros (SCP, SFTP, FTPS).
  • Autenticarse con claves SSH para evitar contraseñas.
  • Comprimir archivos grandes antes de la transferencia.

Buenas prácticas de transferencia en modo binario

  • Usar protocolos seguros (SCP, SFTP, FTPS).
  • Autenticarse con claves SSH para evitar contraseñas.
  • Comprimir archivos grandes antes de la transferencia.

Errores comunes en transferencias de texto

  • Perdida de caracteres especiales.
  • Corrupción de archivos debido a codificación incorrecta.
  • Problemas con la autenticación si se usa contraseñas.

Errores comunes en transferencias binarias

  • Corrupción de archivos debido a problemas de codificación.
  • Problemas con la autenticación si se usa contraseñas.
  • Perdida de datos importantes durante la transferencia.

Comparativa: SCP vs FTPS

SCP FTPS
Seguridad Alto (SSH) Bajo (SSL/TLS)
Velocidad Alta Moderada
Simplicidad Baja (requiere SSH) Media (requiere FTP + SSL/TLS)
