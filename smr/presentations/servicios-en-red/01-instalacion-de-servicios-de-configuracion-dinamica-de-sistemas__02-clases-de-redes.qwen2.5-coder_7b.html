Portada

Instalación de servicios de configuración dinámica de sistemas

Objetivo: Comprender las clases de redes y su importancia en la configuración dinámica de sistemas.

Clases de redes

  • Red local (LAN)
  • Red metropolitana (MAN)
  • Red área amplia (WAN)
  • Red personal (PAN)
  • Red de acceso punto a punto (P2P)

Características de las LANs

  • Cobertura limitada (generalmente dentro de un edificio o campus)
  • Velocidad alta (10 Mbps a 1 Gbps)
  • Trafico local, baja congestión
  • Seguridad interna del negocio

Ejemplo de LAN

Red corporativa: Una LAN que conecta los equipos de oficina dentro de una empresa.

Características de las MANs

  • Cobertura urbana o metropolitana
  • Velocidad moderada (10 Mbps a 100 Mbps)
  • Trafico mixto, alta congestión
  • Seguridad y cobertura más amplia que las LANs

Ejemplo de MAN

Red metropolitana: Una MAN que conecta los negocios en una ciudad.

Características de las WANs

  • Cobertura extensa (across cities, countries)
  • Velocidad variable (1 Mbps a 10 Gbps)
  • Trafico global, alta congestión
  • Seguridad y redundancia avanzada

Ejemplo de WAN

Red internacional: Una WAN que conecta empresas en diferentes países.

Características de las PANs

  • Cobertura personal (en casa o oficina)
  • Velocidad variable (1 Mbps a 10 Gbps)
  • Trafico local, baja congestión
  • Conveniencia y flexibilidad

Ejemplo de PAN

Red inalámbrica personal: Una PAN que conecta dispositivos en una casa o oficina.

Características de las P2P

  • Cobertura limitada (entre dos puntos)
  • Velocidad variable (1 Mbps a 1 Gbps)
  • Trafico local, baja congestión
  • No requiere infraestructura centralizada

Ejemplo de P2P

Red inalámbrica inmediata: Una conexión directa entre dos dispositivos.

Buenas prácticas en el uso de clases de redes

  • Asignar direcciones IP adecuadas según la clase de red
  • Utilizar subredes para segmentar la red
  • Configurar puertas de enlace y rutas correctamente
  • Mantener actualizadas las configuraciones de seguridad

Errores comunes al asignar clases de redes

  • Asignar la misma clase a diferentes subredes
  • No utilizar suficientes direcciones IP para los dispositivos
  • No configurar correctamente las puertas de enlace y rutas
  • No mantener actualizadas las configuraciones de seguridad

Checklist: Configuración de una LAN

  • Asignar direcciones IP a los dispositivos
  • Configurar la puerta de enlace predeterminada
  • Establecer las rutas necesarias
  • Verificar la conectividad con otros dispositivos
  • Aplicar configuraciones de seguridad básicas

Resumen

  • Las clases de redes (LAN, MAN, WAN, PAN, P2P) tienen diferentes características y usos.
  • Buenas prácticas incluyen asignar direcciones IP adecuadas, segmentar la red, configurar puertas de enlace y rutas, y mantener actualizadas las configuraciones de seguridad.
  • Errores comunes pueden causar problemas de conectividad y seguridad.

Normativa general

Reglamento de Seguridad en Redes (RSR): Establece las normas para la protección de los sistemas informáticos y redes.
