Objetivo: Comprender los tipos de registros utilizados en el servicio de resolución de nombres.

Tipos de registros DNS

  • A (Address): Asigna una dirección IP a un nombre de dominio.
  • CNAME (Canonical Name):rea un alias para otro nombre de dominio.
  • MX (Mail Exchange): Especifica los servidores de correo electrónico para un dominio.
  • NS (Name Server): Define los servidores de nombres autoritativos para un dominio.
  • SOA (Start of Authority): Contiene información general sobre el dominio y su configuración.
  • SRV (Service Record): Especifica la ubicación de servicios específicos en una red.
  • TXT (Text): Almacena texto arbitrario asociado con un nombre de dominio.

Ejemplo de registro A

www.example.com. IN A 93.184.216.34

Significado: El nombre www.example.com se resuelve a la dirección IP 93.184.216.34.

Ejemplo de registroNAME

blog.example.com. IN NAME www.example.com.

Significado: El nombre blog.example.com es un alias para www.example.com.

Ejemplo de registro MX

example.com. IN MX 10 mail.example.com.

Significado: El correo electrónico para example.com se envía a mail.example.com con una prioridad de 10.

Ejemplo de registro NS

example.com. IN NS ns1.example.net.

Significado: El servidor de nombres autoritativo para example.com es ns1.example.net.

Ejemplo de registro SOA

example.com. IN SOA ns1.example.net. admin.example.net. (2023040101 7200 1800 604800 3600)

Significado: Contiene información sobre el dominio example.com.

Ejemplo de registro SRV

_sip._tcp.example.com. IN SRV 5060 sipserver.example.net.

Significado: El servicio SIP se encuentra en el puerto 5060 en sipserver.example.net.

Ejemplo de registro TXT

example.com. IN TXT "v=spf1 include:_spf.google.com ~all"

Significado: Contiene información SPF para el dominio example.com.

Buenas prácticas en registros DNS

  • Usar nombres de dominios descriptivos y fáciles de recordar.
  • Asegurar la seguridad de los registros, especialmente los MX y TXT.
  • Mantener actualizados los registros para reflejar cambios en la infraestructura.

Errores comunes en registros DNS

  • Error de ortografía en los nombres de dominio o servidores.
  • Asignación incorrecta de direcciones IP.
  • Omisión de registros necesarios para la funcionalidad del dominio.

Comparativa entre tipos de registros

Checklist: Configuración de registros DNS

  • Verificar la ortografía en los nombres de dominio y servidores.
  • Asegurar que las direcciones IP sean correctas.
  • Comprobar que todos los registros necesarios estén presentes.
