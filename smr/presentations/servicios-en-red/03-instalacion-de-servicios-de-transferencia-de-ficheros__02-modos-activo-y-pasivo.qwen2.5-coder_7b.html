Objetivo: Comprender los modos activo y pasivo en la instalación de servicios de transferencia de ficheros.

Modos activo y pasivo en transferencia de ficheros

  • Activo: El cliente inicia la conexión y el servidor responde.
  • Pasivo: El servidor inicia la conexión y el cliente responde.

Ejemplo de modo activo

El cliente establece una conexión TCP al puerto 21 del servidor FTP. El servidor acepta la conexión y comienza a transferir los datos.

Ejemplo de modo pasivo

El cliente envía un comando PASV al servidor, que responde con una dirección IP y puerto. El cliente establece una nueva conexión TCP a esa dirección para transferir los datos.

Caso práctico: Transferencia de archivos via FTP activo

  1. El cliente se conecta al servidor FTP.
  2. El cliente envía un comando LIST para obtener la lista de archivos.
  3. El servidor responde con los datos del archivo.

Caso práctico: Transferencia de archivos via FTP pasivo

  1. El cliente envía un comando PASV al servidor.
  2. El servidor responde con una dirección IP y puerto.
  3. El cliente establece una nueva conexión TCP a esa dirección para transferir los datos.

Proceso de instalación FTP activo

  1. Instalar el servidor FTP (por ejemplo, vsftpd).
  2. Configurar el archivo de configuración.
  3. Iniciar el servicio del servidor FTP.

Proceso de instalación FTP pasivo

  1. Instalar el servidor FTP (por ejemplo, vsftpd).
  2. Configurar el archivo de configuración para modo pasivo.
  3. Iniciar el servicio del servidor FTP.

Buenas prácticas en modo activo

  • Usar firewalls adecuados para permitir la conexión TCP al puerto 21.
  • Configurar permisos de usuario y grupo correctamente.

Buenas prácticas en modo pasivo

  • Usar firewalls adecuados para permitir la conexión TCP al puerto especificado por el servidor.
  • Configurar permisos de usuario y grupo correctamente.

Errores comunes en modo activo

  • Firewall bloqueando la conexión al puerto 21.
  • Configuración incorrecta de permisos de usuario.

Errores comunes en modo pasivo

  • Firewall bloqueando la conexión al puerto especificado por el servidor.
  • Configuración incorrecta de permisos de usuario.

Comparativa: Modo activo vs pasivo

Modo Activo Modo Pasivo
Control de conexión Cliente inicia la conexión. Servidor inicia la conexión.
Firewall Puede requerir abrir el puerto 21. Puede requerir abrir un puerto específico.

Mini-quiz: Modos activo y pasivo

¿Cuál es la diferencia principal entre el modo activo y el modo pasivo en FTP?

  1. El cliente inicia la conexión en ambos modos.
  2. El servidor inicia la conexión en ambos modos.
  3. Correcto: El cliente inicia la conexión en el modo activo y el servidor en el modo pasivo.

Checklist: Instalación FTP activo

  • Iniciar el servicio del servidor FTP.
  • Verificar la conexión desde un cliente FTP.

  • Iniciar el servicio del servidor FTP.
  • Verificar la conexión desde un cliente FTP.
