Introducción a los Comandos de Transferencia de Ficheros

  • Comprenden herramientas para enviar, recibir y gestionar archivos entre sistemas.
  • Ejemplos populares: SCP, SFTP, FTP, RSYNC.

SCP (Secure Copy Protocol)

Copia segura de ficheros entre sistemas Linux/Unix.

scp /ruta/local/archivo usuario@servidor:/ruta/remota/

SFTP (SSH File Transfer Protocol)

Transferencia de archivos a través de SSH, proporcionando seguridad.

sftp usuario@servidor

FTP (File Transfer Protocol)

Protocolo estándar para transferencias de ficheros en Internet.

ftp servidor

RSYNC

Sincronización bidireccional de archivos y directorios.

rsync -avz /ruta/local/ usuario@servidor:/ruta/remota/

Ejemplo Práctico: Transferencia de Ficheros con SCP

  1. Conectarse al servidor remoto.
  2. Copiar un archivo local a un directorio remoto.
  3. Verificar la copia en el servidor.

Buenas Prácticas con SCP

  • Usar SSH para autenticación segura.
  • Especificar rutas absolutas para evitar problemas.
  • Comprimir datos con la opción -z.

Errores Comunes al Usar SCP

  • No especificar la ruta remota correctamente.
  • No tener permisos suficientes en el servidor.
  • Olvidar la opción de compresión (-z).

Comparativa entre SCP y SFTP

SCP SFTP
Copia segura de ficheros. Transferencia de archivos a través de SSH.
No requiere servidor FTP. Necesita un servidor SSH.

Resumen: Comandos de Transferencia de Ficheros

  • SCP, SFTP y FTP son herramientas comunes para transferencias de ficheros.
  • Cada uno tiene sus ventajas y desventajas dependiendo del escenario.
  • Buenas prácticas incluyen seguridad, permisos y compresión.

Normativa General sobre Transferencia de Ficheros

Seguridad en transferencias de datos es crucial para proteger información confidencial.

  • Usar protocolos seguros como SCP y SFTP.
  • Implementar autenticación fuerte.
  • Limitar los permisos de acceso a ficheros sensibles.

Caso Práctico: Transferencia Segura de Datos

Una empresa necesita transferir datos confidenciales a un proveedor remoto.

  1. Usar SCP para copiar archivos.
  2. Configurar SSH con claves públicas y privadas.
  3. Comprimir los datos antes de la transferencia.
