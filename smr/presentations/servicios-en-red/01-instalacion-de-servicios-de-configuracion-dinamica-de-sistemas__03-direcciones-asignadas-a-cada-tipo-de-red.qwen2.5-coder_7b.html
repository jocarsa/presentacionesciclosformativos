Direcciones asignadas a cada tipo de red

Las direcciones IP se asignan automáticamente a los dispositivos en una red mediante protocolos como DHCP. Esto facilita la gestión y el escalado de redes.

Dirección IPv4 estándar

  • Formato: 4 octetos separados por puntos (ej: 192.168.1.1)
  • Cada octeto puede tener valores de 0 a 255
  • Direcciones reservadas para redes y enrutadores

Clases de direcciones IPv4

  • A: 1.0.0.0 - 126.255.255.255 (127.0.0.0 reservada)
  • B: 128.0.0.0 - 191.255.255.255
  • C: 192.0.0.0 - 223.255.255.255

Direcciones de red y broadcast

Para cada dirección IP se asigna una dirección de red y una dirección de broadcast.

  • Dirección de red: Todos los bits del host soneros
  • Dirección de broadcast: Todos los bits del host son unos

Subredes

Las subredes se crean al dividir una red en segmentos más pequeños.

  • Cálculo de la máscara de subred
  • Determinación del número de direcciones disponibles

Tipos de redes

  • Red local (LAN): Red pequeña dentro de una organización
  • Red metropolitana (MAN): Red que cubre una ciudad o área urbana
  • Red nacional (WAN): Red que conecta diferentes redes a gran distancia

Direcciones reservadas

  • 10.0.0.0 - 10.255.255.255 (Clase A)
  • 172.16.0.0 - 172.31.255.255 (Clase B)
  • 192.168.0.0 - 192.168.255.255 (Clase)

Direcciones de enrutamiento

Las direcciones IP se utilizan para enrutar paquetes entre diferentes redes.

  • Dirección default (0.0.0.0)
  • Direcciones de red y broadcast dentro de la misma red

Configuración dinámica con DHCP

DHCP asigna automáticamente direcciones IP a los dispositivos en una red.

  • Beneficios: Facilita la gestión y el escalado de redes
  • Potencial problemas: Asignación incorrecta o conflictos de IP

Ejemplo de configuración DHCP

Configuración básica en un servidor DHCP:

  • Dirección IP del servidor DHCP
  • Máscara de subred
  • Rango de direcciones IP asignables

Caso práctico: Implementación de DHCP

Implementar DHCP en una red pequeña para gestionar las direcciones IP automáticamente.

  • Pasos básicos: Configurar el servidor DHCP, asignar rango de IPs
  • Verificar la configuración y solucionar problemas comunes

Buenas prácticas en DHCP

  • Configurar tiempos de renovación y reasignación adecuados
  • Monitorear el uso de las direcciones IP asignadas
  • Evitar conflictos de IP mediante la gestión correcta del rango

Errores comunes en DHCP

  • Dirección IP asignada incorrectamente
  • Tiempo de renovación excesivo o insuficiente
  • Conflictos entre dispositivos
