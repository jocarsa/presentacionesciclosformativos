Adopción de pautas de seguridad informática: Seguridad lógica

Objetivo: Comprender las pautas de seguridad lógica y su importancia en la protección de sistemas informáticos.

Criptografía

  • Definición: Transformación de datos legibles en formato ininteligible.
  • Tipo de criptografía: Simétrica y Asimétrica.
  • Ejemplo: AES (Advanced Encryption Standard).
  • Caso práctico: Protección de transacciones financieras.

Listas de control de acceso

  • Definición: Sistema que gestiona quiénes pueden acceder a qué recursos.
  • Tipo de listas: ACL (Access Control List) y RBAC (Role-Based Access Control).
  • Ejemplo: Sistema operativo Linux con permisos de usuario.
  • Caso práctico: Seguridad en redes empresariales.

Establecimiento de políticas de contraseñas
  • Definición: Reglas que rigen la creación y uso de contraseñas.
  • Ejemplos: Longitud mínima, caracteres especiales, cambios periódicos.
  • Caso práctico: Políticas internas de una empresa tecnológica.

Políticas de almacenamiento
  • Definición: Reglas que rigen la gestión y protección de datos en almacenamiento.
  • Ejemplos: Seguridad física, acceso controlado, encriptación.
  • Caso práctico: Almacenamiento de datos sensibles en un centro de datos.

Copias de seguridad e imágenes de respaldo
  • Definición: Proceso de creación de copias exactas de los datos para su recuperación en caso de pérdida.
  • Ejemplos: Software de copia de seguridad, almacenamiento en la nube.
  • Caso práctico: Plan de recuperación ante desastres (DRP).

Medios de almacenamiento
  • Definición: Dispositivos físicos donde se almacenan los datos.
  • Ejemplos: Discos duros, SSDs, discos magnéticos.
  • Caso práctico: Selección de medios de almacenamiento para un servidor web.

Buenas prácticas en seguridad lógica
  • Uso de software actualizado y parcheado.
  • Implementación de firewalls y sistemas antivirus.
  • Monitoreo constante del sistema.
  • Ejemplo: Uso de herramientas de análisis de vulnerabilidades.

Errores comunes en seguridad lógica
  • No actualización regular del software.
  • Falta de implementación de políticas de contraseñas fuertes.
  • No uso de firewalls y sistemas antivirus.
  • Ejemplo: Exposición de puertos no necesarios en un servidor web.

Checklist de seguridad lógica
  1. Implementar políticas de contraseñas fuertes.
  2. Configurar firewalls y sistemas antivirus.
  3. Realizar copias de seguridad periódicas.
  4. Monitorear el sistema constantemente.

Resumen
  • Criptografía: Protección de datos mediante transformación.
  • Listas de control de acceso: Gestión del acceso a recursos.
  • Políticas de contraseñas, almacenamiento y copias de seguridad: Protección integral de los datos.
