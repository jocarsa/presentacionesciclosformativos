Tipos de «proxy». Características y funciones.

Objetivos: comprender los conceptos clave y su aplicación práctica.

Tipo 1: Proxy Inverso

El proxy inverso oculta el servidor real y expone una dirección IP pública. Ejemplos: Nginx, Apache HTTP Server.

Tipo 2: Proxy Directo

El cliente se conecta directamente al servidor. No oculta la dirección del servidor. Ejemplo: SOCKS5.

Tipo 3: Proxy Transparente

No requiere configuración especial en el cliente. El tráfico pasa silenciosamente a través del proxy. Ejemplos: Squid, Fiddler.

Tipo 4: Proxy Anónimo

El servidor no registra la dirección IP del cliente. Protege la privacidad pero revela el tipo de proxy. Ejemplo: Tor.

Tipo 5: Proxy Barrera

El cliente se conecta a través del proxy para acceder a Internet, pero no puede ser utilizado por otros dispositivos en la red. Ejemplo: OpenVPN.

Característica 1:aching

  • Almacena copias de los recursos solicitados para acelerar el acceso futuro.
  • Reduce la carga en servidores originales y mejora la velocidad del usuario.

Característica 2: Filtrado de Contenido

  • Permite o deniega el acceso a ciertos tipos de contenido basándose en políticas estableidas.
  • Aumenta la seguridad y control sobre el tráfico web.

Característica 3: Autenticación

  • Solicita credenciales de los usuarios antes de permitir el acceso al proxy.
  • Aumenta la seguridad y control sobre quién puede usar el proxy.

Característica 4: Balanceo dearga

  • Distribuye la carga entre múltiples servidores para mejorar la disponibilidad y rendimiento.
  • Aumenta la capacidad del sistema al manejar más solicitudes simultáneas.

Característica 5: Registro y Monitoreo

  • Almacena información sobre el tráfico pasante para análisis y auditoría.
  • Aumenta la visibilidad y control sobre el uso del proxy.

Proceso: Configuración Básica de un Proxy Anónimo con Tor

  1. Descargar e instalar Tor Browser.
  2. Iniciar el navegador y navegar por Internet.
  3. Verificar la conexión anónima mediante servicios como WhatIsMyIP.com.

Buena Práctica: Uso de Certificados SSL/TLS en Proxies

Para garantizar la seguridad del tráfico entre el cliente y el servidor, utilizar certificados SSL/TLS firmados por autoridades confiables.

Error Común: Exposición de Direcciones IP Internas

No exponer direcciones IP internas en la configuración del proxy para evitar vulnerabilidades de seguridad.
