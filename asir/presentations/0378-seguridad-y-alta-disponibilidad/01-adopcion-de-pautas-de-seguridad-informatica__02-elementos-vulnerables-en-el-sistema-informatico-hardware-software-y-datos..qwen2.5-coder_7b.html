Adopción de pautas de seguridad informática

Objetivo: Identificar y mitigar elementos vulnerables en el sistema informático.

Elementos Vulnerables en el Sistema Informático

  • Hardware: Componentes físicos que pueden fallar o ser explotados.
  • Software: Programas y sistemas operativos susceptibles a vulnerabilidades.
  • Datos: Información sensible que puede ser comprometida.

Vulnerabilidad de Hardware

Ejemplo: Falta de actualizaciones del firmware en dispositivos de red.

Vulnerabilidad de Software

Errores comunes: Uso de bibliotecas obsoletas o vulnerables.

Vulnerabilidad de Datos

Caso: Brecha de seguridad en una base de datos sin encriptación adecuada.

Proceso de Identificación de Vulnerabilidades

  1. Evaluación del hardware actual.
  2. Análisis de la seguridad del software instalado.
  3. Examen de políticas y procedimientos de manejo de datos.

Buenas Prácticas para el Hardware

  • Mantenimiento regular del firmware.
  • Uso de componentes de alta calidad y confiables.
  • Implementación de medidas físicas de seguridad.

Buenas Prácticas para el Software

  • Actualizaciones regulares del sistema operativo y aplicaciones.
  • Uso de software de código abierto con fuertes comunidades de soporte.
  • Implementación de parches y actualizaciones de seguridad.

Buenas Prácticas para los Datos

  • Encriptación de datos sensibles en reposo y en tránsito.
  • Control de acceso basado en roles.
  • Copias de seguridad regulares y almacenamiento seguro.

Errores Comunes en el Hardware

  • No realizar actualizaciones del firmware.
  • Uso de componentes obsoletos o de baja calidad.
  • Ausencia de medidas físicas de seguridad.

Errores Comunes en el Software

  • No realizar actualizaciones regulares del sistema operativo y aplicaciones.
  • Uso de software de código cerrado con poca o ninguna comunidad de soporte.
  • Ausencia de parches y actualizaciones de seguridad.

Errores Comunes en los Datos

  • No realizar encriptación de datos sensibles.
  • Ausencia de control de acceso basado en roles.
  • No realizar copias de seguridad regulares y almacenamiento inseguro.

Comparativa: Vulnerabilidades vs. Seguridad

Vulnerabilidad Seguridad
Falla en hardware Mantenimiento regular del firmware
Error en software Actualizaciones regulares del sistema operativo y aplicaciones
Compromiso de datos Encriptación de datos sensibles
