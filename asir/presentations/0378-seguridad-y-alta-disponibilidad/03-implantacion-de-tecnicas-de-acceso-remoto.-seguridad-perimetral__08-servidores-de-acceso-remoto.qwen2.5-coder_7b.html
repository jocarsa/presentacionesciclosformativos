Objetivo: Aprender los protocolos de autenticación, configurar parámetros de acceso y entender servidores de autenticación.

Protocolos de autenticación

  • SSH (Secure Shell)
  • Telnet
  • RDP (Remote Desktop Protocol)
  • VNC (Virtual Network Computing)
  • Kerberos

Ejemplo de protocolo seguro: SSH

SSH utiliza cifrado asimétrico para autenticar tanto el cliente como el servidor.

Caso práctico: Configuración de SSH

  • Generar claves RSA
  • Configurar archivo /etc/ssh/sshd_config
  • Reiniciar servicio SSH

Buenas prácticas en autenticación

  • Usar claves públicas y privadas
  • Desactivar el acceso root remoto
  • Utilizar autenticación multifactor

Errores comunes en autenticación

  • Usar contraseñas simples
  • Habilitar el acceso root remoto
  • No utilizar autenticación multifactor

Comparativa de protocolos

ProtocoloCifradoAutenticación
SSHAsimétricoClaves públicas/privadas
TelnetNo cifradoContraseñas planas

Servidores de autenticación

  • LDAP (Lightweight Directory Access Protocol)
  • AD (Active Directory)
  • PAM (Pluggable Authentication Modules)

Ejemplo de servidor de autenticación: LDAP

LDAP es un protocolo de directorio que permite almacenar y gestionar información de usuarios y grupos.

Caso práctico: Configuración de LDAP en un servidor

  • Instalar e instanciar el servicio LDAP
  • Configurar clientes para usar el servidor LDAP

Buenas prácticas en servidores de autenticación

  • Usar un servidor centralizado
  • Cifrar la comunicación entre el cliente y el servidor
  • Asegurar la seguridad del directorio

Errores comunes en servidores de autenticación

  • No usar un servidor centralizado
  • No cifrar la comunicación
  • No asegurar el directorio
