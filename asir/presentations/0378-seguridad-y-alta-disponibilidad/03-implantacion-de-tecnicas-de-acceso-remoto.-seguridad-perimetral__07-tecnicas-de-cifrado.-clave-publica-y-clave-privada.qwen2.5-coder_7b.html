Tecniques de cifrado. Clave pública y clave privada

Objetivo: Comprender los conceptos básicos de cifrado, enfocándose en el uso de claves públicas y privadas.

Concepto de cifrado

  • Cifrado es el proceso de convertir información legible (clara) en una forma que solo pueda ser leída por alguien con la clave adecuada.
  • El cifrado asegura la confidencialidad, integridad y autenticidad de los datos transmitidos o almacenados.

Clave pública y clave privada

Las claves públicas y privadas son pares de números grandes que se generan juntas.ada par consta de dos claves:

  • Clave pública: Se puede compartir libremente con cualquier persona.
  • Clave privada: Debe mantenerse en secreto y nunca compartirse.

Funcionamiento del cifrado asimétrico

El cifrado asimétrico utiliza tanto la clave pública como la clave privada:

  • Cifrado con la clave pública: El remitente cifra el mensaje con la clave pública del destinatario.
  • Descifrado con la clave privada: El destinatario descifra el mensaje con su propia clave privada.

Ejemplo de uso en SSH

SSH (Secure Shell) utiliza claves públicas y privadas para autenticar usuarios:

  • El cliente genera una clave pública y una clave privada.
  • La clave pública se envía al servidor durante la conexión.
  • El servidor verifica la firma digital con la clave pública correspondiente a la clave privada del cliente.

Ejemplo de uso en SSL/TLS

SSL/TLS (Secure Sockets Layer/Transport Layer Security) utiliza claves públicas y privadas para cifrar las comunicaciones:

  • El cliente envía una solicitud de conexión al servidor.
  • El servidor responde con su certificado digital, que contiene su clave pública.
  • El cliente verifica el certificado y genera un par temporal de claves públicas y privadas para la sesión.

Ventajas del cifrado asimétrico

  • Cifrado de alta seguridad: Aunque la clave pública es compartida, solo la clave privada puede descifrar los datos.
  • Autenticación digital: Permite verificar la identidad de las partes involucradas.

Desventajas del cifrado asimétrico

  • Cálculos intensivos: El cifrado y descifrado con claves largas son computacionalmente intensivos.
  • Requisitos de almacenamiento: Se requieren almacenes grandes para almacenar las claves privadas.

Comparación VPN a nivel de red y SSL/TLS

Tecnología Ventajas Desventajas
VPN a nivel de red (IKEv2/IPSec) Seguridad robusta, autenticación múltiple, soporte para múltiples protocolos. Configuración compleja, requerimientos de hardware y software adicionales.
SSL/TLS Cifrado de datos en tránsito, autenticación digital, soporte para aplicaciones web. Dependencia del protocolo HTTP/HTTPS, limitado a aplicaciones web.

Buenas prácticas en el uso de claves públicas y privadas

  • Generar claves de tamaño adecuado (al menos 2048 bits).
  • Utilizar certificados digitales emitidos por autoridades confiables.
  • Regularmente actualizar las claves y certificados.

Errores comunes en el uso de claves públicas y privadas

  • Compartir la clave privada con terceros.
  • No actualizar regularmente las claves y certificados.
  • Usar claves de tamaño insuficientemente grande.

Caso práctico: Implementación de SSH con claves públicas y privadas

Para implementar SSH con claves públicas y privadas:

  1. Generar una pareja de claves (publica.pem, private.pem).
  2. Copiar la clave pública al servidor remoto.
  3. Configurar el cliente para usar la clave privada.
  4. Conectar al servidor utilizando SSH.

Mini-quiz: ¿Cuál es la diferencia entre VPN a nivel de red y SSL/TLS?

Elija la respuesta correcta:

  • VPN a nivel de red cifra datos en tránsito.
  • SSL/TLS autentica identidades digitales.
  • VPN a nivel de red es más adecuada para aplicaciones web.

Checklist: Implementación segura de SSH con claves públicas y privadas

  1. Generar una pareja de claves (publica.pem, private.pem).
  2. Copiar la clave pública al servidor remoto.
  3. Configurar el cliente para usar la clave privada.
  4. Verificar los permisos de las claves (solo lectura para la clave privada).

Actividad: Configuración de una conexión SSH segura con claves públicas y privadas

Siga los pasos descritos en el caso práctico para configurar una conexión SSH segura.

Resumen

  • Cifrado asimétrico utiliza claves públicas y privadas para garantizar la seguridad de los datos.
  • SSH y SSL/TLS son ejemplos comunes de aplicaciones que utilizan este tipo de cifrado.
  • Buenas prácticas incluyen generar claves seguras, mantenerlas actualizadas y proteger las claves privadas.

Normativa general

La normativa vigente establece requisitos mínimos para la implementación de sistemas de seguridad:

  • Norma ISO/IE 27001: Norma internacional para el manejo seguro de la información.
  • Reglamento General de Protección de Datos (RGPD): Reglamento europeo sobre protección de datos personales.
