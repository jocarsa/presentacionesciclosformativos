Reglas de filtrado de cortafuegos

Objetivo: Comprender y configurar las reglas de filtrado en un cortafuego.

Concepto básico

  • Las reglas de filtrado definen qué tráfico es permitido o denegado por el cortafuego.
  • Estructuran las políticas de seguridad del sistema.
  • Incluyen criterios como origen, destino y protocolo.

Tipos de reglas

  • Permitir (Allow): Acepta el tráfico que cumple con los criterios especificados.
  • Denegar (Deny): Bloquea el tráfico que no cumple con los criterios.
  • Rechazar (Reject): Similar a Deny, pero envía un mensaje de rechazo al origen del paquete.

Estructura de una regla

  • Protocolo: TCP, UDP, ICMP, etc.
  • Origen: Dirección IP o rango permitido.
  • Destino: Dirección IP o rango permitido.
  • Puerto: Número de puerto específico o rango.
  • Accion: Permitir, Denegar o Rechazar.

Ejemplo de regla

<rule family="ipv4" action="accept">
    <source address="192.168.1.0/24"/>
    <destination address="10.0.0.0/8"/>
    <protocol name="tcp" port="80"/>
</rule>

Orden de las reglas

  • Las reglas se evalúan en el orden en que aparecen.
  • La primera regla que coincide determina la acción.
  • Es crucial tener cuidado con el orden para evitar cortar tráfico innecesario.

Caso práctico: Reglas de entrada y salida

  • Entrada (Inbound): Controla el tráfico entrante al sistema.
  • Salida (Outbound): Controla el tráfico saliente del sistema.
  • Ejemplo: Permitir SSH (puerto 22) solo desde una red interna específica.

Buenas prácticas

  • Minimizar el número de reglas activas para mejorar la eficiencia.
  • Usar grupos de direcciones y puertos para simplificar las reglas.
  • Mantener un registro y auditoría de las reglas configuradas.

Errores comunes

  • No permitir tráfico necesario (por ejemplo, SSH).
  • Denegar tráfico innecesario (por ejemplo, tráfico de aplicaciones legítimas).
  • No tener en cuenta el orden de las reglas.

Comparativa: Reglas vs. Políticas

  • Reglas: Definen acciones específicas para tráfico individual.
  • Políticas: Define comportamientos generales (por ejemplo, permitir todo el tráfico entrante).
  • Las reglas suelen ser más detalladas y específicas que las políticas.

Mini-quiz

¿Cuál de las siguientes acciones es una acción válida en una regla de cortafuegos?

  1. Aceptar (Accept)
  2. Denegar (Deny)
  3. Rechazar (Reject)
  4. Ignorar (Ignore)

Checklist de configuración

  • Identificar los servicios necesarios.
  • Determinar las direcciones IP y puertos correspondientes.
  • Configurar reglas de entrada y salida.
  • Revisar el orden de las reglas.
  • Probar la configuración con herramientas como ping o telnet.

Actividad práctica

Configurar una regla de cortafuegos en un sistema Linux utilizando iptables para permitir el tráfico SSH desde una red interna específica.

Resumen

  • Las reglas de filtrado definen las políticas de seguridad del cortafuego.
  • Incluyen criterios como origen, destino y protocolo.
  • Es crucial tener cuidado con el orden y la eficiencia de las reglas.

Normativa general

Seguir normativas como NIST SP 800-53 para configurar políticas de seguridad efectivas.
