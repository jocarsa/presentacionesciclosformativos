Objetivo: Configuración y utilización de máquinas virtuales para mejorar la seguridad y disponibilidad.

Definición de Alta Disponibilidad

  • Sistema operativo disponible 99,9% del tiempo
  • No interrupiones programadas ni no programadas
  • Redundancia y failover automático

Máquinas Virtuales (VM)

Software que emula un hardware virtual en un sistema operativo.

  • Hypervisor: software que gestiona las VMs
  • VMM (Virtual Machine Monitor): controla la ejecución de las VMs

Beneficios de las Máquinas Virtuales

  • Escalabilidad horizontal y vertical
  • Costo reducido al compartir hardware
  • Facilidad de migración y copia de seguridad

Ejemplo: Configuración de una VM

  1. Instalar el software del hipervisor (Ej: VMware ESXi)
  2. Crear un nuevo recurso virtual
  3. Instalar el sistema operativo en la VM

Caso Práctico: Implementación de una VM para una base de datos

Configuración de un servidor SQL Server en una VM.

Proceso de Configuración

  1. Preparar el hardware y software del hipervisor
  2. Crear la configuración virtual
  3. Instalar el sistema operativo en la VM
  4. Configurar el servicio de base de datos

Buenas Prácticas para las VMs

  • Uso de discos virtuales separados para sistemas operativos y datos
  • Configuración de red dedicada para la VM
  • Monitoreo continuo del rendimiento y salud de las VMs

Errores Comunes en la Configuración de VMs

  • No asignar suficiente memoria a la VM
  • No configurar correctamente el almacenamiento virtual
  • No realizar copias de seguridad regulares

Comparativa: Máquinas Virtuales vs. Servidores Físicos

Máquinas Virtuales Servidores Físicos
Economía de costos Costo fijo por hardware
Facilidad de escalado Requiere adquisición de nuevos equipos

Mini-Quiz: Conceptos Básicos

¿Qué es una máquina virtual?

  1. Sistema operativo real ejecutándose directamente en el hardware
  2. Software que emula un hardware virtual en un sistema operativo
  3. Dispositivo de almacenamiento externo virtualizado

Checklist: Configuración de una VM

  • Instalar el hipervisor
  • Crear la configuración virtual
  • Instalar sistema operativo
  • Configurar red y almacenamiento
  • Realizar pruebas de funcionamiento
