Objetivo: Comprender los conceptos básicos del filtrado de paquetes de datos en cortafuegos.

Concepto de filtrado de paquetes

El filtrado de paquetes es una técnica utilizada por los cortafuegos para decidir si permitir o denegar el paso de tráfico de red.

Cómo funciona el filtrado de paquetes

  • Los cortafuegos examinan cada paquete de datos que intenta pasar por él.
  • Aplican reglas predefinidas para determinar si el paquete debe ser permitido o denegado.
  • Las reglas pueden basarse en criterios como origen, destino, protocolo y puerto.

Ejemplo de regla de filtrado

Ejemplo: Permitir todos los tráfico entrante del puerto 80 (HTTP).

Caso práctico: Implementación de un cortafuegos

Se debe configurar un cortafuegos para permitir el acceso a una base de datos solo desde la red interna.

Proceso de configuración

  1. Identificar las interfaces de red del cortafuegos.
  2. Configurar reglas para permitir el tráfico necesario entre interfaces.
  3. Aplicar reglas adicionales para denegar todo el resto del tráfico entrante y saliente.

Buenas prácticas en filtrado de paquetes

  • Minimizar las reglas activas para evitar problemas de rendimiento.
  • Usar listas blancas (permitir lo necesario) en lugar de listas negras (denegar lo innecesario).
  • Mantener actualizadas las reglas según los cambios en la infraestructura y las políticas de seguridad.

Errores comunes en filtrado de paquetes

  • No configurar suficientemente las reglas para permitir el tráfico necesario.
  • Configurar reglas demasiado restrictivas que impidan el acceso a recursos necesarios.
  • No mantener actualizadas las reglas con los cambios en la infraestructura o políticas de seguridad.

Comparativa: Filtrado por estado vs. filtrado por paquete

Filtrado por estado: Permite el paso de tráfico basándose en la conexión existente.

Filtrado por paquete: Examina cada paquete individualmente, independientemente de las conexiones existentes.
