Sentencias para la realización y recuperación de copias de seguridad

Objetivo: Aprender las sentencias SQL necesarias para crear, realizar y restaurar copias de seguridad en una base de datos.

Concepto de Copia de Seguridad

Copia de seguridad es un proceso que crea una copia exacta de los datos almacenados en una base de datos.

  • Propósito: Protección contra pérdidas de datos y recuperación ante desastres.
  • Tipo de Copias:
    • Copia completa
    • Copia diferencial
    • Copia incremental

Comando SQL para Crear una Copia de Seguridad

Se utiliza el comando BACKUP DATABASE.

BACKUP DATABASE nombre_base_de_datos TO DISK = 'ruta/del/archivo.bak'

Ejemplo de Creación de Copia de Seguridad

BACKUP DATABASE MiBaseDeDatos TO DISK = ':\Backups\MiBaseDeDatos.bak'

Este comando crea una copia completa de la base de datos "MiBaseDeDatos" en el archivo "MiBaseDeDatos.bak" ubicado en:\Backups.

Comando SQL para Realizar una Recuperación de Seguridad

Se utiliza el comando RESTORE DATABASE.

RESTORE DATABASE nombre_base_de_datos FROM DISK = 'ruta/del/archivo.bak'

Ejemplo de Recuperación de Seguridad

RESTORE DATABASE MiBaseDeDatos FROM DISK = ':\Backups\MiBaseDeDatos.bak'

Este comando restaura la base de datos "MiBaseDeDatos" desde el archivo "MiBaseDeDatos.bak" ubicado en:\Backups.

Buenas Prácticas para Copias de Seguridad

  • Realizar copias de seguridad periódicamente (diariamente, semanalmente).
  • Almacenar copias de seguridad en ubicaciones seguras y separadas.
  • Probar regularmente la recuperación para asegurar su funcionamiento.

Errores Comunes al Manejar Copias de Seguridad

  • No realizar copias de seguridad periódicamente.
  • Almacenar copias de seguridad en la misma ubicación que los datos originales.
  • No probar regularmente la recuperación.
