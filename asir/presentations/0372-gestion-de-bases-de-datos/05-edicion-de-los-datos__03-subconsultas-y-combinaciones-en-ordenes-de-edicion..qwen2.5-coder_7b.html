Subconsultas en órdenes de edición

  • Definición: Consulta que se ejecuta dentro de otra consulta.
  • Tipos principales: Correlacionadas y no correlacionadas.
  • Ventajas: Facilitan la realización de operaciones complejas en una sola consulta.

Ejemplo de subconsulta

UPDATE empleados
SET salario = salario * 1.10
WHERE departamento_id IN (SELECT id FROM departamentos WHERE ubicacion = 'Nueva York');

Combinaciones en órdenes de edición

  • Definición: Combinación de dos o más operaciones de edición (INSERT, UPDATE, DELETE).
  • Ventajas: Permiten realizar múltiples cambios en una sola transacción.

Ejemplo de combinación de órdenes de edición

BEGIN TRANSACTION;
UPDATE empleados SET salario = salario * 1.05 WHERE departamento_id = 1;
DELETE FROM proyectos WHERE estado = 'Terminado';
COMMIT;

Buenas prácticas en subconsultas y combinaciones

  • Optimización: Usar índices y evitar subconsultas complejas.
  • Transacciones: Utilizar transacciones para asegurar la integridad de los datos.
  • Seguridad: Limitar el acceso a las consultas y operaciones de edición.

Errores comunes en subconsultas y combinaciones

  • Sintaxis incorrecta: Verificar la estructura de la consulta.
  • Subconsulta no correlacionada: Asegurarse de que la subconsulta dependa de los datos del exterior.
  • Problemas de rendimiento: Optimizar las consultas y evitar bucles innecesarios.

  1. Muestra el número de empleados con un salario mayor que la media.

Checklist: Subconsultas y combinaciones

  • Verificar la sintaxis de las consultas.

  • Optimizar las consultas para mejorar el rendimiento.
