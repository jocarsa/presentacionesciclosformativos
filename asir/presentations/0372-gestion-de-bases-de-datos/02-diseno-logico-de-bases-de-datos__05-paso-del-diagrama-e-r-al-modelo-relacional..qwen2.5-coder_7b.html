Diseño lógico de bases de datos

Objetivo: Convertir diagramas E/R en modelos relacionales.

Paso del diagrama E/R al modelo relacional

  • Identificar entidades y atributos
  • Determinar relaciones entre entidades
  • Definir cardinalidad de las relaciones
  • Convertir atributos simples en columnas
  • Crear claves primarias y foráneas

Ejemplo: Conversión de diagrama E/R a modelo relacional

Entidad "Alumno"

  • ID_Alumno (PK)
  • Nombre
  • Apellido
  • DNI
  • Email

Entidad "Materia"

  • ID_Materia (PK)
  • Nombre
  • Creditos

Relación "Matricula"

  • ID_Alumno (FK)
  • ID_Materia (FK)
  • Fecha

Caso: Normalización

Convertir el modelo relacional en uno normalizado para evitar redundancias y anomalías.

Buenas prácticas

  • Usar nombres descriptivos para tablas y columnas
  • Definir tipos de datos adecuados
  • Asegurar integridad referencial
  • Optimizar el diseño para la eficiencia

Errores comunes

  • No definir claves primarias
  • Usar tipos de datos incorrectos
  • No normalizar el modelo
  • No considerar la eficiencia en el diseño
