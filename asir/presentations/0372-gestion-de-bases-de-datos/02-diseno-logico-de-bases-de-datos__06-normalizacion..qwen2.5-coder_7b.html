Normalización

Objetivo: Comprender los conceptos básicos y principios de normalización en la gestión de bases de datos.

Definición de Normalización

La normalización es el proceso de diseñar una base de datos para eliminar redundancia y dependencias no funcionales, asegurando que cada dato esté almacenado en un lugar único.

Principios de Normalización

  • 1NF (First Normal Form): Elimina duplicados y asegura que todos los valores sean atómicos.
  • 2NF (Second Normal Form): Cumple con 1NF y elimina dependencias parciales.
  • 3NF (Third Normal Form): Cumple con 2NF y evita dependencias transitivas.

Ejemplo de 1NF

ID Nombre Dirección
1 Juan Pérez Calle 1, Ciudad A

Problema: La dirección está dividida en dos columnas.

Ejemplo de 2NF

ID Nombre Calle Ciudad
1 Juan Pérez Calle 1 Ciudad A

Problema: La ciudad está repetida.

Ejemplo de 3NF

ID Nombre Calle Ciudad
1 Juan Pérez Calle 1 Ciudad A

Problema: No hay dependencias transitivas.

Dependencias Funcionales

Una dependencia funcional es una relación entre un atributo (o conjunto de atributos) y otro atributo en la misma tabla. Se expresa como A → B, donde A es el conjunto de atributos determinantes y B es el atributo determinado.

Dependencias No Funcionales

  • Integridad Referencial: Relación entre tablas para mantener consistencia de datos.
  • Unicidad:ada valor en una columna debe ser único.
  • No Nulos: Atributos no pueden tener valores nulos.

Ejemplo de Dependencia Funcional

ID Nombre Calle Ciudad
1 Juan Pérez Calle 1 Ciudad A

Ejemplo: ID → Nombre, Calle, Ciudad.

Dependencias Transitivas

Una dependencia transitiva ocurre cuando A → B y B →, pero no existe una relación directa entre A y. Esto puede causar redundancia y problemas de integridad.

Ejemplo de Dependencia Transitiva

ID Nombre Calle Ciudad
1 Juan Pérez Calle 1 Ciudad A

Ejemplo: ID → Nombre, Nombre → Ciudad.

Buenas Prácticas de Normalización

  • Minimizar la redundancia de datos.
  • Asegurar la integridad referencial.
  • Evitar dependencias no funcionales.

Errores Comunes en Normalización

  • No eliminar completamente las redundancias.
  • No considerar todas las dependencias funcionales.
  • No aplicar el nivel adecuado de normalización.

Mini-Quiz

Pregunta: ¿Cuál es la principal ventaja de la normalización en una base de datos?

  1. Aumenta el tamaño del archivo.
  2. Reduce la redundancia y mejora la integridad de los datos.
  3. Permite almacenar más información en una sola tabla.

Checklist de Normalización

  • Verificar que la base de datos cumpla con 1NF, 2NF y 3NF.
  • Asegurar que no haya dependencias transitivas.
  • Comprobar la integridad referencial entre tablas.

Ejercicio Práctico

Diseña una base de datos para un sistema de gestión de bibliotecas que incluya las entidades: Libro, Autor y Prestamo. Aplica los principios de normalización hasta el nivel 3NF.

Resumen

  • La normalización es crucial para la gestión eficiente de bases de datos.
  • Se basa en principios como 1NF, 2NF y 3NF.
  • Asegura la integridad y reducción de redundancia.

Normativa Relacionada

No hay normativas específicas, pero se aplican principios de diseño de sistemas de información y bases de datos.
