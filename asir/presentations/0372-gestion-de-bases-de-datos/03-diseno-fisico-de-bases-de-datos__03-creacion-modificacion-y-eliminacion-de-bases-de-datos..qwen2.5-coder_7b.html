Diseño físico de bases de datos

Objetivo: Comprender los procesos de creación, modificación y eliminación de bases de datos.

Creación de Bases de Datos

  • Especificar el nombre de la base de datos.
  • Definir las características del motor de base de datos (SQL, NoSQL, etc.).
  • Establecer los permisos y usuarios.
  • Cargar datos iniciales si es necesario.

Ejemplo de Creación de Base de Datos


CREATE DATABASE MiBaseDeDatos;
USE MiBaseDeDatos;
CREATE TABLE Usuarios (
    ID INT PRIMARY KEY,
    Nombre VARCHAR(100),
    Email VARCHAR(100)
);
INSERT INTO Usuarios (ID, Nombre, Email) VALUES (1, 'Juan', 'juan@example.com');

Modificación de Bases de Datos

  • Añadir nuevas tablas.
  • Modificar estructura existente (campos, índices).
  • Agregar o eliminar usuarios y permisos.
  • Realizar copias de seguridad periódicas.

Ejemplo de Modificación de Base de Datos


ALTER TABLE Usuarios ADD COLUMN Edad INT;
CREATE INDEX idx_email ON Usuarios(Email);
GRANT SELECT, INSERT ON MiBaseDeDatos TO 'usuario'@'localhost';

Eliminación de Bases de Datos

  • Eliminar tablas.
  • Borrar completamente la base de datos.
  • Revertir cambios en caso de error.
  • Documentar todos los cambios realizados.

Ejemplo de Eliminación de Base de Datos


DROP TABLE Usuarios;
DROP DATABASE MiBaseDeDatos;

Buenas Prácticas en la Gestión de Bases de Datos

  • Seguridad: Proteger contra accesos no autorizados.
  • Mantenimiento: Realizar actualizaciones y optimizaciones periódicas.
  • Backup: Mantener copias de seguridad regulares.
  • Documentación: Registrar todos los cambios y procedimientos.

Errores Comunes en la Gestión de Bases de Datos

  • No realizar copias de seguridad.
  • No documentar cambios importantes.
  • No tener permisos adecuados.
  • No optimizar regularmente las bases de datos.

Comparativa: SQL vs NoSQL

SQL NoSQL
Estructura de datos Relacional (tablas) No relacional (colecciones, documentos)
Escalabilidad Vertical (hardware) Horizontal (nodos adicionales)
Consistencia ACID CAP (Consistencia, Disponibilidad, Partición)

Checklist: Gestión de Bases de Datos

  • Crear base de datos con nombre y motor adecuado.
  • Añadir tablas y definir sus campos.
  • Establecer permisos y usuarios seguros.
  • Realizar copias de seguridad periódicas.
  • Mantener documentación actualizada.

Resumen: Creación, Modificación y Eliminación de Bases de Datos

Se han aprendido los procesos básicos para gestionar una base de datos:

  • Creación: Nombre, motor, usuarios.
  • Modificación: Añadir tablas, modificar estructura, permisos.
  • Eliminación: Tablas, base de datos completa.

Normativa en la Gestión de Bases de Datos

Es importante cumplir con las leyes de protección de datos (GDPR, CCPA, etc.) al gestionar bases de datos.

  • Consentimiento explícito de los usuarios.
  • Seguridad de la información personal.
  • Acceso limitado a los datos.
