Operadores y Estructuras de Control en Bases de Datos

Objetivo: Comprender los operadores y estructuras de control utilizados en la construcción de guiones para bases de datos.

Introducción a los Operadores

  • Operadores aritméticos: +, -, *, /, %
  • Operadores relacionales: =, !=, >, <, >=, <=
  • Operadores lógicos: AND, OR, NOT

Ejemplo de Operador Aritmético

SELECT precio * 0.95 AS descuento FROM productos;

Ejemplo de Operador Relacional

SELECT nombre FROM clientes WHERE edad > 18;

Ejemplo de Operador Lógico

SELECT * FROM empleados WHERE departamento = 'Ventas' AND activo = TRUE;

Operadores de Comparación

  • LIKE: Busca patrones en cadenas
  • BETWEEN: Busca valores dentro de un rango
  • IN: Busca múltiples valores

Ejemplo de Operador LIKE

SELECT nombre FROM clientes WHERE nombre LIKE 'J%';

Ejemplo de Operador BETWEEN

SELECT fecha FROM ventas WHERE fecha BETWEEN '2023-01-01' AND '2023-12-31';

Ejemplo de Operador IN

SELECT * FROM productos WHERE categoria IN ('Electrónica', 'Computadoras');

Estructuras de Control en Bases de Datos

  • IF...ELSE
  • CASE
  • LOOP

Ejemplo de IF...ELSE

SELECT nombre, CASE WHEN edad >= 18 THEN 'Mayor de edad' ELSE 'Menor de edad' END AS estado FROM clientes;

Ejemplo de CASE

SELECT categoria, precio, CASE WHEN precio < 50 THEN 'Bajo' WHEN precio BETWEEN 50 AND 100 THEN 'Medio' ELSE 'Alto' END AS rango_precio FROM productos;

Ejemplo de LOOP

CREATE PROCEDURE actualizar_precios() BEGIN DECLARE i INT DEFAULT 1; WHILE i <= 10 DO UPDATE productos SET precio = precio * 1.1 WHERE id = i; SET i = i + 1; END WHILE; END;

Buenas Prácticas

  • Usar nombres descriptivos para columnas y tablas
  • Evitar operadores complejos en consultas
  • Optimizar bucles y estructuras de control

Errores Comunes

  • Olvidar el uso de comillas alrededor de cadenas
  • No usar paréntesis en operaciones aritméticas
  • Bucles infinitos con estructuras de control

Mini-Quiz

¿Cuál es el resultado de la siguiente consulta?

SELECT nombre FROM clientes WHERE edad > 18 AND ciudad = 'Madrid';

  1. Todos los clientes mayores de 18 años en Madrid
  2. Solo los clientes de Madrid
  3. Solo los clientes mayores de 18 años
  4. Error de sintaxis

Checklist

  • Verificar la sintaxis de las consultas SQL
  • Asegurarse de que los operadores sean correctos
  • Optimizar el uso de estructuras de control
  • Probar las consultas en un entorno de desarrollo
