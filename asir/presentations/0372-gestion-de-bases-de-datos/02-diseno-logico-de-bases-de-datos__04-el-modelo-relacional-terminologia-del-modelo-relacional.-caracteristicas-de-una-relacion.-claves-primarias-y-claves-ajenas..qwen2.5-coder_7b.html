Objetivo: Comprender el modelo relacional, su terminología y características.

Modelo Relacional: Terminología

  • Relación (Tabla)
  • Tupla (Fila)
  • Atributo (Columna)
  • Domino (Tipo de dato)
  • Cardinalidad (Número de tuplas)

Modelo Relacional: Características

  • No redundancia de datos
  • Integridad referencial
  • Independencia física y lógica
  • Consistencia semántica
  • Uniformidad en la organización de los datos

Claves Primarias

Definición: Identifica de manera única cada tupla en una relación.

  • No puede ser nula
  • No puede repetirse
  • Única por relación

Claves Ajenas

Definición: Referencia a una clave primaria de otra relación.

  • No puede ser nula (excepto en relaciones que permitan)
  • Puede repetirse
  • Referencia única a una tupla en la relación referenciada

Ejemplo de Relación

ID EmpleadoNombreDepartamento ID
101Juan5
102Maria3

ID Empleado es clave primaria, Departamento ID es clave ajena.

Caso Práctico: Diseño de una Relación

  • Relación: Alumnos
  • Atributos: ID, Nombre, Edad, Carrera
  • Clave Primaria: ID

Buenas Prácticas en Claves

  • Usar tipos de datos adecuados (numéricos, texto)
  • Asegurar la unicidad con índices
  • No permitir claves nulas donde no sea necesario

Errores Comunes en Claves

  • Usar claves que pueden repetirse (por ejemplo, números de teléfono)
  • No definir claves primarias adecuadas
  • No considerar la integridad referencial al crear claves ajenas

Comparativa: Clave Primaria vs Clave Ajena
