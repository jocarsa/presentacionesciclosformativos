Diseño físico de bases de datos

Objetivo: Comprender las herramientas gráficas proporcionadas por los SGBD para la implementación de una base de datos.

Herramientas gráficas en SGBD

  • Diagrama Entidad-Relación (ER)
  • Modelo Conceptual de Datos (MCD)
  • Modelo Lógico de Datos (MLD)
  • Modelo Físico de Datos (MFD)

Diagrama Entidad-Relación (ER)

Representa la estructura lógica de una base de datos.

Modelo Conceptual de Datos (MCD)

Describe las entidades, atributos y relaciones sin considerar detalles técnicos.

Modelo Lógico de Datos (MLD)

Especifica cómo se organizarán los datos en el almacenamiento físico.

Modelo Físico de Datos (MFD)

Determina cómo se almacenan los datos en el sistema de archivos.

Uso de herramientas gráficas

  • Facilitan la visualización y comprensión del diseño.
  • Aumentan la eficiencia en el proceso de modelado.
  • Permiten una comunicación más clara entre los stakeholders.

Caso de estudio

Desarrollo del MCD, MLD y MFD para una base de datos escolar.

Buenas prácticas

  • Consistencia en el uso de símbolos y notaciones.
  • Documentación detallada del diseño.
  • Revisión y aprobación por parte de los stakeholders.

Errores comunes

  • Olvidar atributos clave en las entidades.
  • No considerar la normalización adecuada.
  • Ignorar los requisitos de rendimiento y seguridad.
