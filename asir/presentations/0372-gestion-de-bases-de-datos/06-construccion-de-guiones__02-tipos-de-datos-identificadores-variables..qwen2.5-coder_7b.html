Tipos de datos, identificadores y variables en bases de datos

Objetivo: Comprender los tipos de datos, cómo definir identificadores y variables en el contexto de las bases de datos.

Tipos de Datos

  • Númericos: Enteros (INT), Reales (FLOAT, DOUBLE)
  • Cadenas: Caracteres (CHAR, VARCHAR), Texto (TEXT)
  • Lógicos: Verdadero/Falso (BOOLEAN)
  • Fecha y Hora: Fecha (DATE), Tiempo (TIME), Fecha y Hora (DATETIME)

Ejemplo de Tipos de Datos

Tabla "Empleados":

IDNombreEdadSueldo
1Juan305000.00

Identificadores

Un identificador es un nombre único que se utiliza para referirse a una tabla, columna o otro objeto en la base de datos.

Ejemplo: "TablaEmpleados"

Reglas para Identificadores

  • No puede comenzar con un número
  • No puede contener espacios
  • No puede ser una palabra reservada de SQL
  • Debe seguir las convenciones de nomenclatura (por ejemplo, CamelCase)

Ejemplo de Identificador

Tabla "Producto":

IDProductoNombrePrecio
101Laptop999.99

Variables en Bases de Datos

Las variables son contenedores que almacenan valores durante la ejecución de un programa o consulta.

Ejemplo: @precioUnitario

Tipos de Variables en SQL Server

  • Numeric: INT, FLOAT, DECIMAL
  • String: VARCHAR, NVARCHAR
  • Date: DATE, DATETIME

Ejemplo de Uso de Variables en SQL Server


DECLARE @precioUnitario DECIMAL(10, 2)
SET @precioUnitario = 99.99
SELECT * FROM Producto WHERE Precio > @precioUnitario

Buenas Prácticas

  • Usar nombres descriptivos para identificadores y variables
  • Definir tipos de datos adecuados para evitar problemas de rendimiento
  • Evitar el uso de caracteres especiales en los nombres

Errores Comunes

  • No definir un tipo de dato para una columna
  • Usar nombres de identificadores que son palabras reservadas
  • No inicializar variables antes de usarlas

Comparativa: Variables vs. Columnas
