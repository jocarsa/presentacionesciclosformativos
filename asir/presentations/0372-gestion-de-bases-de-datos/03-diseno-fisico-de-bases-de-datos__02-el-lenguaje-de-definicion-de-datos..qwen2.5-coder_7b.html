Diseño físico de bases de datos

Objetivo: Comprender el lenguaje de definición de datos (DDL) y su uso en la creación y gestión de esquemas de base de datos.

Introducción al Lenguaje de Definición de Datos (DDL)

  • DDL es un conjunto de comandos utilizados para definir y gestionar la estructura de una base de datos.
  • Incluye operaciones como crear, modificar y eliminar objetos de la base de datos.
  • Ejemplos: CREATE TABLE, ALTER TABLE, DROP TABLE.

Comando CREATE TABLE

Crea una nueva tabla en la base de datos.

CREATE TABLE empleados (
        id INT PRIMARY KEY,
        nombre VARCHAR(100),
        apellido VARCHAR(100),
        email VARCHAR(100)
    );

Definición de Tipos de Datos

  • INT: Entero.
  • VARCHAR(n): Cadena de caracteres con longitud máxima n.
  • DATE: Fecha.
  • DECIMAL(p, s): Número decimal con p dígitos en total y s decimales.

Definición de Restriciones

  • PRIMARY KEY: Identifica únicamente cada fila.
  • FOREIGN KEY: Relaciona dos tablas.
  • UNIQUE: Asegura que los valores en una columna sean únicos.
  • NOT NULL: Impide que una columna tenga valores nulos.

Ejemplo de Tabla con Restriciones

CREATE TABLE departamentos (
        id INT PRIMARY KEY,
        nombre VARCHAR(100) NOT NULL
    );

    CREATE TABLE empleados (
        id INT PRIMARY KEY,
        nombre VARCHAR(100),
        apellido VARCHAR(100),
        email VARCHAR(100),
        departamento_id INT,
        FOREIGN KEY (departamento_id) REFERENCES departamentos(id)
    );

Comando ALTER TABLE

Modifica la estructura de una tabla existente.

  • Añadir columna: ALTER TABLE table_name ADD column_name datatype.
  • Modificar columna: ALTER TABLE table_name MODIFY COLUMN column_name datatype.
  • Borrar columna: ALTER TABLE table_name DROP COLUMN column_name.

Ejemplo de ALTER TABLE

ALTER TABLE empleados ADD COLUMN edad INT;
    ALTER TABLE empleados MODIFY COLUMN email VARCHAR(150);
    ALTER TABLE empleados DROP COLUMN edad;

Comando DROP TABLE

Borra una tabla completamente de la base de datos.

DROP TABLE table_name;

Ejemplo de DROP TABLE

DROP TABLE empleados;

Buenas Prácticas en DDL

  • Usar nombres descriptivos para tablas y columnas.
  • Definir claves primarias y foráneas adecuadamente.
  • Utilizar tipos de datos apropiados.
  • Añadir restricciones como NOT NULL y UNIQUE cuando sea necesario.

Errores Comunes en DDL

  • No definir claves primarias o foráneas.
  • Usar tipos de datos incorrectos.
  • No añadir restricciones necesarias.
  • Borrar tablas sin antes hacer una copia de seguridad.

Ejemplo de Caso Práctico

Se necesita crear una base de datos para un sistema de gestión de tiendas en línea. Diseña la estructura inicial de las tablas.

CREATE TABLE productos (
        id INT PRIMARY KEY,
        nombre VARCHAR(100),
        precio DECIMAL(10, 2)
    );

    CREATE TABLE clientes (
        id INT PRIMARY KEY,
        nombre VARCHAR(100),
        email VARCHAR(100) UNIQUE
    );

    CREATE TABLE pedidos (
        id INT PRIMARY KEY,
        cliente_id INT,
        fecha DATE,
        FOREIGN KEY (cliente_id) REFERENCES clientes(id)
    );

    CREATE TABLE pedido_items (
        id INT PRIMARY KEY,
        pedido_id INT,
        producto_id INT,
        cantidad INT,
        FOREIGN KEY (pedido_id) REFERENCES pedidos(id),
        FOREIGN KEY (producto_id) REFERENCES productos(id)
    );

  • A) CREATE TABLE table_name (id INT);
  • B) CREATE TABLE table_name (id INT PRIMARY KEY);
  • C) CREATE TABLE table_name (id INT UNIQUE);
  • D) CREATE TABLE table_name (id INT NOT NULL);

Resumen

  • DDL es el lenguaje utilizado para definir y gestionar la estructura de una base de datos.
  • Comandos principales: CREATE TABLE, ALTER TABLE, DROP TABLE.
  • Buenas prácticas y errores comunes en DDL.

Normativa General

No hay normativas específicas para DDL. Sin embargo, es importante seguir buenas prácticas de diseño y seguridad al definir la estructura de las bases de datos.
