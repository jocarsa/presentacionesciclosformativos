Unión de consultas en SQL

Objetivo: Aprender a combinar múltiples consultas SQL utilizando UNION y UNION ALL.

Concepto básico

  • UNION: Combina los resultados de dos o más SELECTs, eliminando duplicados.
  • UNION ALL: Combina los resultados de dos o más SELECTs, incluyendo duplicados.

Sintaxis básica


SELECT columna1, columna2 FROM tabla1
UNION [ALL]
SELECT columna1, columna2 FROM tabla2;

Requisitos para la unión

  • Número de columnas debe ser el mismo en ambos SELECTs.
  • Tipo de datos correspondientes en las columnas.

Ejemplo simple


SELECT nombre, edad FROM empleados
UNION ALL
SELECT nombre, edad FROM clientes;

Ordenamiento de resultados

Puedes ordenar los resultados combinados con ORDER BY al final:


(SELECT nombre, edad FROM empleados
UNION ALL
SELECT nombre, edad FROM clientes)
ORDER BY edad DESC;

Uso de WHERE en UNION

No puedes usar WHERE directamente en cada SELECT dentro de UNION. Filtra los datos antes de la unión:


SELECT nombre, edad FROM empleados WHERE edad > 30
UNION ALL
SELECT nombre, edad FROM clientes WHERE edad > 30;

Mini-quiz

¿Cuál de las siguientes consultas es correcta?

  • SELECT nombre FROM empleados UNION SELECT apellido FROM clientes;
  • SELECT nombre, edad FROM empleados WHERE edad > 30 UNION ALL SELECT nombre, edad FROM clientes WHERE edad > 30;

Buenas prácticas

  • Usa UNION ALL si es posible para evitar el costo de eliminación de duplicados.
  • Ordena los resultados al final si es necesario.

Errores comunes

  • No tener el mismo número de columnas en ambos SELECTs.
  • Diferentes tipos de datos entre las columnas correspondientes.

Comparativa UNION vs. UNION ALL

UNIONUNION ALL
DuplicadosNo incluyeIncluye
Costo computacionalMás alto debido a la eliminación de duplicadosBajo, solo combina los resultados

Caso práctico

Un banco necesita una lista de todos los clientes y empleados. ¿Cómo usar UNION?


SELECT nombre, apellido FROM clientes
UNION ALL
SELECT nombre, apellido FROM empleados;

Checklist

  • Verifica que el número de columnas sea igual en ambos SELECTs.
  • Asegúrate de que los tipos de datos sean compatibles.
  • Ordena los resultados si es necesario al final.

Actividad práctica

Combina las siguientes consultas usando UNION y UNION ALL:


SELECT producto, precio FROM productos
WHERE categoria = 'Electrónica';
SELECT producto, precio FROM productos
WHERE categoria = 'Ropa';

Resumen

  • UNION combina resultados eliminando duplicados.
  • UNION ALL combina resultados incluyendo duplicados.
  • Requiere el mismo número de columnas y tipos de datos compatibles.

Normativa general

No hay normativas específicas, pero sigue las mejores prácticas de SQL para mejorar la eficiencia y legibilidad del código.
