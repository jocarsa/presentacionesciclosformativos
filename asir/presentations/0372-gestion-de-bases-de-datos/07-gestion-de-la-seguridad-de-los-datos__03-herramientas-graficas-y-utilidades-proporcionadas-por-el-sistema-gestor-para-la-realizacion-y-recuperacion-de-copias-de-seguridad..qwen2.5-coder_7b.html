Ejemplo de uso en PostgreSQL con pgAdmin

Crea una copia de seguridad:

  1. Conéctate a la base de datos.
  2. Navega hasta "Backup" y selecciona la base de datos.
  3. Especifica el tipo de backup (pg_dump, pg_dumpall).
  4. Selecciona el archivo de salida y realiza el backup.

Ejemplo de uso en MySQL con MySQL Workbench

Crea una copia de seguridad:

  1. Conéctate a la base de datos.
  2. Navega hasta "Server" y selecciona "Data Export".
  3. Especifica las bases de datos a exportar.
  4. Selecciona el archivo de salida y realiza la exportación.

Caso práctico: Recuperación de copia de seguridad en SQL Server con SSMS

Pasos:

  1. Conéctate a la instancia de SQL Server.
  2. Navega hasta "Management" y selecciona "Backup and Restore".
  3. Especifica el archivo de copia de seguridad.
  4. Selecciona las opciones de restauración (reemplazar, aplicar parches).

Buenas prácticas para la gestión de copias de seguridad

  • Frecuencia: Diaria, semanal o mensual según riesgo.
  • Almacenamiento seguro: Diferentes ubicaciones geográficas.
  • Pruebas periódicas: Simular recuperación para validar copias de seguridad.
  • Caducidad: Eliminar copias de seguridad antiguas.

Errores comunes en la gestión de copias de seguridad

  • No realizar copias de seguridad frecuentes.
  • Almacenar todas las copias de seguridad en el mismo lugar.
  • No probar regularmente las copias de seguridad.
  • Copiar y almacenar solo la estructura sin datos.
