Bases de Datos

Conceptos, usos y tipos según el modelo de datos, la ubicación de la información.

Definición de Bases de Datos

  • Conjunto organizado de datos que se almacenan y gestionan en un sistema informático.
  • Permite el acceso, almacenamiento, recuperación y modificación de los datos de manera eficiente.

Tipos de Bases de Datos según el Modelo de Datos

  • Relacional: Utiliza tablas y relaciones entre ellas (SQL).
  • No Relacional: No utiliza tablas, permite almacenamiento flexible (NoSQL).
  • Híbrida: Combina características de bases de datos relacional y no relacional.

Tipos de Bases de Datos según la Ubicación de los Datos

  • Centralizadas: Todos los datos se almacenan en un solo servidor.
  • Distribuidas: Los datos están dispersos entre varios servidores.
  • Móviles: Datos almacenados en dispositivos móviles (smartphones, tablets).

Usos Comunes de las Bases de Datos

  • Gestión empresarial: Ventas, inventario, recursos humanos.
  • Servicios web: Autenticación, almacenamiento de usuarios.
  • Educación: Almacenamiento de calificaciones, notas.

Modelos Conceptuales de Bases de Datos

  • Entidad-Relación (ER): Representa entidades, atributos y relaciones entre ellas.
  • Objeto-Orientado (OO): Modela objetos con propiedades y métodos.
  • Híbrido: Combina elementos de ER y OO.

Ejemplo de Bases de Datos Relacional

Una base de datos relacional para una tienda online podría tener tablas como Usuarios, Productos, Ventas y DetallesVentas.

Ejemplo de Bases de Datos No Relacional

Una base de datos no relacional para un sistema de recomendaciones podría almacenar usuarios y sus preferencias en documentos.

Caso Práctico: Sistema de Gestión de Biblioteca

  • Almacena información sobre libros, autores, prestamos.
  • Utiliza modelo relacional para organizar datos.

Buenas Prácticas en la Gestión de Bases de Datos

  • Seguridad: Implementar medidas de seguridad para proteger los datos.
  • Optimización: Asegurar que las consultas y operaciones sean eficientes.
  • Mantenimiento: Realizar regularmente backups y mantenimiento preventivo.

Errores Comunes en la Gestión de Bases de Datos

  • No implementar medidas de seguridad adecuadas.
  • No optimizar consultas y operaciones.
  • No realizar backups regulares.
