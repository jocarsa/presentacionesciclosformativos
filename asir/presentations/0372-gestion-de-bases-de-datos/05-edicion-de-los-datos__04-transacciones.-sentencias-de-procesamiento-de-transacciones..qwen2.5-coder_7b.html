Transacciones. Sentencias de procesamiento de transacciones

Objetivo: Comprender los conceptos básicos y sentencias utilizadas para gestionar las transacciones en bases de datos.

Definición de Transacción

Una transacción es una unidad de trabajo que realiza un conjunto de operaciones relacionadas. Debe cumplir cuatro propiedades fundamentales: ACID (Atomicidad, Coherencia, Isolación y Durabilidad).

Tipos de Transacciones

  • Transacciones Locales: Realizadas dentro de una sola base de datos.
  • Transacciones Distribuidas: Involucran múltiples bases de datos o sistemas.

Sentencias SQL para Transacciones

  • BEGIN TRANSACTION;
  • COMMIT;
  • ROLLBACK;
  • SAVEPOINT;

Atomicidad (A)

Cada transacción debe ser una unidad indivisible. Si ocurre un error, toda la operación debe revertirse.

Coherencia ()

Una transacción debe llevar a la base de datos de un estado válido a otro. No puede dejar el sistema en un estado inconsistente.

Isolación (I)

Cada transacción debe ejecutarse como si fuera la única operación en el sistema, evitando interferencias entre ellas.

Durabilidad (D)

Una vez que una transacción ha sido confirmada, sus cambios deben persistir incluso ante fallas del sistema.

Ejemplo de Transacción


BEGIN TRANSACTION;
UPDATE cuentas SET saldo = saldo - 100 WHERE id = 1;
UPDATE cuentas SET saldo = saldo + 100 WHERE id = 2;
COMMIT;

Manejo de Errores

Usar ROLLBACK; para revertir cambios en caso de error. Usar SAVEPOINT; para marcar puntos de recuperación.

Proceso de Transacción

  1. Iniciar la transacción con BEGIN TRANSACTION;
  2. Ejecutar las operaciones necesarias.
  3. Confirmar los cambios con COMMIT;
  4. Opcionalmente, usar SAVEPOINT; para puntos de recuperación.

Buenas Prácticas

  • Usar transacciones cuando sea necesario mantener la integridad de los datos.
  • Minimizar el tiempo de bloqueo para evitar problemas de concurrencia.
  • Manejar adecuadamente errores y excepciones.

Errores Comunes en Transacciones

  • Olvidar el uso de BEGIN TRANSACTION;
  • No confirmar cambios con COMMIT;
  • No manejar adecuadamente errores y excepciones.

Comparativa: Transacciones vs. Consultas

Transacción Consulta
Efectúa múltiples operaciones. Ejecuta una sola consulta.
Requiere confirmación con COMMIT; No requiere confirmación explícita.

Resumen

  • Transacciones son unidades de trabajo que realizan operaciones relacionadas.
  • Deben cumplir las propiedades ACID para garantizar la integridad de los datos.
  • Se manejan con sentencias como BEGIN TRANSACTION;, COMMIT; y ROLLBACK;.
