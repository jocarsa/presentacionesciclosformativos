Configuración básica de un servidor Web

Objetivo: Aprender los pasos fundamentales para configurar y administrar un servidor Web básico.

Definición de Servidor Web

Un servidor Web es un programa que atiende solicitudes HTTP (Hypertext Transfer Protocol) y devuelve páginas web al cliente.

Ejemplos de Servidores Web

  • Apache HTTP Server
  • Nginx
  • Microsoft IIS (Internet Information Services)
  • Lighttpd
  • LiteSpeed

Requisitos Hardware para un Servidor Web

  • CPU con al menos 1 GHz
  • Al menos 512 MB de RAM
  • Espacio en disco: Al menos 10 GB (dependiendo del tamaño del sitio)
  • Conexión a Internet

Requisitos Software para un Servidor Web

  • Sistema operativo compatible (Linux, Windows Server)
  • Software de servidor web instalado
  • Lenguajes de programación y frameworks necesarios (,, Ruby, etc.)
  • Bases de datos (MySQL, PostgreSQL, MongoDB)

Configuración del Document Root

El Document Root es la carpeta donde se almacenan las páginas web que el servidor sirve a los clientes.

Ejemplo: /var/www/html (Linux) o:\inetpub\wwwroot (Windows)

Configuración del Servidor Virtual

Un servidor virtual permite alojar múltiples sitios web en un solo servidor.

Ejemplo de configuración en Apache:

<VirtualHost *:80>
        ServerName www.ejemplo.com
        DocumentRoot /var/www/ejemplo
</VirtualHost>

Configuración del Servidor HTTPS

HTTPS es una versión segura de HTTP que utiliza SSL/TLS para cifrar la comunicación.

Ejemplo de configuración en Apache:

<VirtualHost *:443>
        ServerName www.ejemplo.com
        DocumentRoot /var/www/ejemplo
        SSLEngine on
        SSLCertificateFile /etc/ssl/certs/example.crt
        SSLCertificateKeyFile /etc/ssl/private/example.key
</VirtualHost>

Configuración del Servidor por Defecto

El servidor por defecto es el que se ejecuta cuando no se especifica un nombre de dominio.

Ejemplo en Apache:

<VirtualHost *:80>
        ServerName localhost
        DocumentRoot /var/www/html
</VirtualHost>

Configuración del Servidor de Control de Acceso

Puedes limitar el acceso a ciertas partes del sitio web.

Ejemplo en Apache:

<Directory /var/www/ejemplo>
        Require all granted
</Directory>

Configuración del Servidor deaché

Puedes configurar el servidor para almacenar en caché las páginas web para mejorar el rendimiento.

Ejemplo en Apache:

<IfModule mod_cache.c>
        CacheEnable disk /
</IfModule>

Configuración del Servidor de Redirecciones

Puedes configurar el servidor para redirigir las solicitudes a otras páginas o dominios.

Ejemplo en Apache:

<VirtualHost *:80>
        ServerName www.ejemplo.com
        Redirect permanent / http://www.nuevo-ejemplo.com/
</VirtualHost>

Configuración del Servidor de Alias

Puedes configurar el servidor para servir archivos desde ubicaciones diferentes.

Ejemplo en Apache:

<VirtualHost *:80>
        ServerName www.ejemplo.com
        Alias /images /var/www/otra-carpeta/images
</VirtualHost>

Configuración del Servidor de Errores Personalizados

Puedes configurar el servidor para mostrar mensajes de error personalizados.

Ejemplo en Apache:

<VirtualHost *:80>
        ServerName www.ejemplo.com
        ErrorDocument 404 /error404.html
</VirtualHost>

Configuración del Servidor de Logs

Puedes configurar el servidor para registrar las solicitudes y errores.

Ejemplo en Apache:

<VirtualHost *:80>
        ServerName www.ejemplo.com
        ErrorLog /var/log/apache2/error.log
        CustomLog /var/log/apache2/access.log combined
</VirtualHost>
