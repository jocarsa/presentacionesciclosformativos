Configuración del servicio de transferencia de archivos. Permisos y cuotas.

Objetivo: Aprender a configurar servicios de transferencia de archivos, establecer permisos y asignar cuotas.

Concepto básico

El servicio de transferencia de archivos (FTP) permite la comunicación entre dos computadoras para el interambio de datos.

Ejemplo práctico

  • Se desea configurar un servidor FTP para compartir documentos entre los empleados de una empresa.
  • El servicio debe permitir la subida y descarga de archivos, pero no la eliminación.

Caso real

Una institución educativa necesita un sistema seguro para compartir recursos entre los docentes y estudiantes.

Proceso de configuración

  1. Instalar el software FTP en el servidor.
  2. Configurar las opciones básicas (puerto, modo seguro).
  3. Crear usuarios y asignar permisos.

Buenas prácticas

  • Usar contraseñas fuertes para los usuarios FTP.
  • Limitar el acceso a solo las carpetas necesarias.
  • Realizar copias de seguridad periódicas del servidor.

Errores comunes

  • No establecer permisos adecuados, lo que puede causar problemas de acceso.
  • No usar un puerto seguro (FTPS), lo que expone los datos a riesgos de interceptación.
  • No realizar copias de seguridad, lo que puede resultar en pérdida de datos importantes.

Comparativa FTP vs SFTP

FTP SFTP
No cifrado de datos. Cifrado de datos mediante SSH.
Puede ser inseguro si no se usa FTPS. Más seguro, ya que utiliza el protocolo SSH.

Checklist

  2. Configurar opciones básicas (puerto, modo seguro).
  3. Crear usuarios y asignar permisos adecuados.
  4. Usar contraseñas fuertes para los usuarios.
  5. Limitar el acceso a solo las carpetas necesarias.
  6. Realizar copias de seguridad periódicas del servidor.

Actividad práctica

Configurar un servicio FTP en un servidor local y asignar permisos a diferentes usuarios.

La normativa vigente en materia de seguridad informática requiere el uso de protocolos seguros como SFTP para servicios FTP.
