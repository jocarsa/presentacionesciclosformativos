Portada

Resolutores de nombres. Proceso de resolución de un nombre de dominio.

Objetivo: Entender el proceso detallado de cómo los resolutores de nombres convierten una dirección URL en una dirección IP.

¿Qué son los Resolutores de Nombres?

  • Servicios que traducen nombres de dominio (como www.example.com) a direcciones IP (como 192.0.2.1).
  • Utilizan una base de datos distribuida para almacenar y compartir esta información.

Estructura del Sistema DNS

  • Root Servers: Los servidores más altos en la jerarquía.
  • TLD Servers (Top-Level Domain): Servidores para dominios de nivel superior (.com, .org, etc.).
  • Authoritative Servers: Servidores que tienen la autoridad sobre un dominio específico.

Proceso de Resolución

  1. Digito o escribo una URL en el navegador.
  2. Navegador envía una solicitud DNS al resolver local del usuario.
  3. Resolver local consulta los servidores root para encontrar el TLD server correspondiente.
  4. TLD server responde con la dirección del servidor autoritativo para el dominio específico.
  5. Resolver local consulta el servidor autoritativo, que devuelve la dirección IP.
  6. Resolver local almacena esta información en su caché para futuras solicitudes.
  7. Navegador usa la dirección IP para establecer una conexión con el servidor web del dominio.

Casos de Uso Comunes

  • Acceso a sitios web.
  • Email.
  • Servicios en la nube (Google, Amazon).

Buenas Prácticas

  • Usar resolutores de nombres confiables y actualizados.
  • Mantener la configuración DNS segura.
  • Realizar pruebas periódicas de conectividad.

Errores Comunes

  • DNS no encontrado (NXDOMAIN).
  • Tiempo de espera excedido (Timeout).
  • Error de autorización.

Comparativa con otros sistemas

Sistema Características
DNS (Domain Name System) Base de datos distribuida, escalable.
ARP (Address Resolution Protocol) Traduce direcciones MAC a IP en una red local.

Resumen

  • Los resolutores de nombres traducen nombres de dominio a direcciones IP.
  • Involucran múltiples servidores en una jerarquía distribuida.
  • El proceso incluye consultas a root servers, TLD servers y servidores autoritativos.

Normativa General

La normativa sobre seguridad en la red impone que los resolutores de nombres sean confiables y actualizados para prevenir ataques como el DNS Spoofing.
