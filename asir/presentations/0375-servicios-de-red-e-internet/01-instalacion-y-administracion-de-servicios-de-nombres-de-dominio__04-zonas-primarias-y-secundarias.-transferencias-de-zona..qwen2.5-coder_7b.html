Zonas primarias y secundarias. Transferencias de zona.

Objetivo: Comprender la diferencia entre zonas primarias y secundarias, así como los procesos de transferencia de zona.

Zona Primaria

  • Contiene toda la información del dominio.
  • Administrada por el propietario del dominio.
  • Almacena los registros DNS completos.
  • Ejemplo: "example.com" es una zona primaria.

Zona Secundaria

  • Copia parcial de la información de la zona primaria.
  • Administrada por un servidor DNS secundario.
  • Actualizadas automáticamente desde la zona primaria.
  • Ejemplo: Un servidor DNS que replica "example.com" es una zona secundaria.

Transferencia de Zona (Zone Transfer)

Proceso por el cual una zona primaria envía su información a un servidor DNS secundario.

Métodos de Transferencia de Zona

  • Axfr (Authoritative Zone Transfer): Transferencia completa.
  • IXFR (Incremental Zone Transfer): Transferencia incremental basada en cambios.
  • Dynamic Update: Actualizaciones en tiempo real.

Buenas Prácticas de Transferencia de Zona

  • Usar IXFR para minimizar el tráfico.
  • Implementar autenticación para transferencias seguras.
  • Configurar tiempos de actualización adecuados.

Errores Comunes en Transferencia de Zona

  • No autorizado: Falta la configuración correcta del servidor secundario.
  • Tiempo de actualización incorrecto: Permite transferencias frecuentes o tardías.
  • Problemas de red: Interrupiones en las transferencias.

Caso Práctico: Configuración de Zona Secundaria

Configurar un servidor DNS secundario para replicar una zona primaria "example.com".

Ejemplo de Transferencia de Zona


dig @primary_server axfr example.com

Proceso de Transferencia de Zona

  1. El servidor secundario solicita una transferencia.
  2. El servidor primario responde con la información completa o incremental.
  3. El servidor secundario actualiza sus registros DNS.

Mini-Quiz: Transferencias de Zona

¿Cuál es el propósito principal de una zona secundaria?

  • Aumentar la velocidad de respuesta.
  • Reducir el tráfico en la red primaria.
  • Proporcionar redundancia y disponibilidad.

Checklist: Configuración de Transferencia de Zona

  • Configurar el servidor primario para permitir transferencias.
  • Configurar el servidor secundario con la dirección del servidor primario.
  • Verificar los tiempos de actualización y autenticación.

Actividad: Práctica de Transferencia de Zona

Resumen

  • Zonas primarias y secundarias son fundamentales para la distribución de DNS.
  • Transferencias de zona permiten mantener la consistencia entre servidores DNS.
  • Buenas prácticas y atención a los errores son cruciales para una configuración exitosa.
