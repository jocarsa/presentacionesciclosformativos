Parámetros y declaraciones de configuración

Objetivo: Comprender los parámetros y declaraciones clave en la configuración automática de red.

Parámetros de configuración básicos

  • Dirección IP
  • Máscara de subred
  • Puerta de enlace predeterminada
  • Servidores DNS
  • Hostname
  • Zona horaria

Ejemplo de configuración

Dirección IP: 192.168.1.100
Máscara de subred: 255.255.255.0
Puerta de enlace predeterminada: 192.168.1.1
Servidores DNS: 8.8.8.8, 8.8.4.4
Hostname: servidor.local
Zona horaria: UTC+2

Declaraciones de configuración en scripts

Ejemplo de script para configurar una interfaz de red:

network interface eth0 {
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1
}

Parámetros de configuración avanzados

  • DHCP
  • LLMNR
  • mDNS
  • NTP
  • Firewall rules
  • Proxy settings

Ejemplo de configuración DHCP

DHCP server {
    subnet 192.168.1.0 netmask 255.255.255.0 {
        range 192.168.1.10 192.168.1.100;
        option routers 192.168.1.1;
        option subnet-mask 255.255.255.0;
    }
}

Declaraciones de configuración en archivos de configuración

Ejemplo de archivo /etc/network/interfaces:

auto eth0
iface eth0 inet static
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1

Parámetros de configuración en sistemas operativos

  • Windows: Configuración de red a través del panel de control.
  • Linux: Utilización de interfaces como NetworkManager o netplan.
  • macOS: Preferencias de red en el sistema.

Ejemplo de configuración en Windows

Paso a paso para configurar una interfaz de red en Windows:

  1. Ir a "Configuración de la red y las redes".
  2. Hacer clic derecho en la interfaz deseada.
  3. Elegir "Propiedades".
  4. Seleccionar "Internet Protocol Version 4 (TCP/IPv4)".
  5. Configurar dirección IP, máscara de subred y puerta de enlace.

Declaraciones de configuración en scripts de automatización

Ejemplo de script para configurar una interfaz de red:

#!/bin/bash
ifconfig eth0 192.168.1.100 netmask 255.255.255.0
route add default gw 192.168.1.1 eth0

Parámetros de configuración en contenedores Docker

  • Docker network
  • IP address
  • Port mapping
  • Environment variables

Ejemplo de configuración en Docker

docker run -d --name mycontainer \
    --network mynetwork \
    --ip 172.18.0.10 \
    -p 8080:80 \
    -e MY_ENV_VAR=value \
    myimage

Declaraciones de configuración en archivos

Ejemplo de archivo docker-compose.yml:

version: '3'
services:
  web:
    image: myimage
    networks:
      - mynetwork
    ports:
      - "8080:80"
    environment:
      - MY_ENV_VAR=value

networks:
  mynetwork:
    driver: bridge

  • Ansible

  • Chef

  instance_type = "t2.micro"

  network_interface {
    subnet_id       = aws_subnet.example.id
    associate_public_ip_address = true

Parámetros de configuración en sistemas de gestión de configuraciones

  • Puppet manifest
  • Chef recipe

Ejemplo de configuración en Puppet

class { 'network':
  ipaddress      => '192.168.1.100',
  netmask        => '255.255.255.0',
  gateway        => '192.168.1.1',
  dns_servers    => ['8.8.8.8', '8.8.4.4'],
}

node.default['network']['ipaddress'] = '192.168.1.100'
