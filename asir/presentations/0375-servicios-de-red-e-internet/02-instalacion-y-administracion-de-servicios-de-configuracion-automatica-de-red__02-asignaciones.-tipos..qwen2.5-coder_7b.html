Portada

Instalación y administración de servicios de configuración automática de red

Objetivo: Comprender los tipos de asignaciones en la configuración automática de redes.

Tipos de Asignaciones en Configuración Automática de Red

  • Dinámica: Asignadas automáticamente por un servidor DHCP.
  • Estatica: Configuradas manualmente por el usuario.
  • Automática con DNS: Asignación de IP y configuración de DNS.
  • Manual con DNS: Configuración manual de IP y DNS.
  • Asignación basada en MAC: Asignación única a dispositivos según su dirección MAC.

Ejemplo de Asignación Dinámica

Cuando un dispositivo se conecta a una red, solicita una IP al servidor DHCP. El servidor asigna una IP disponible y proporciona otros parámetros como la puerta de enlace y el DNS.

Ejemplo de Asignación Estática

El usuario configura manualmente la dirección IP, la máscara de subred, la puerta de enlace y los servidores DNS. Esto se hace generalmente en dispositivos que no cambian su ubicación.

Caso Práctico: Asignación Automática con DNS

Un dispositivo se conecta a una red y solicita una IP. El servidor DHCP también proporciona la configuración de DNS para que el dispositivo pueda acceder a Internet.

Caso Práctico: Asignación Manual con DNS

Un usuario configura manualmente su dirección IP y los servidores DNS. Esto es útil en entornos donde se requiere una configuración específica para el acceso a Internet.

Buenas Prácticas: Asignaciones Dinámicas

  • Reducción de errores humanos al no tener que asignar manualmente IPs.
  • Facilita la gestión de dispositivos móviles y remotos.
  • Evita conflictos IP ya que las direcciones se asignan automáticamente.

Buenas Prácticas: Asignaciones Estáticas

  • Proporciona una dirección IP constante para dispositivos estáticos.
  • Facilita la configuración de servicios que requieren una IP fija.
  • Permite la creación de políticas de red más específicas.

Errores Comunes en Asignaciones Dinámicas

  • No tener suficientes direcciones IP disponibles en el servidor DHCP.
  • Solicitar una IP que ya está asignada a otro dispositivo.
  • No configurar correctamente los parámetros de puerta de enlace y DNS.

Errores Comunes en Asignaciones Estáticas

  • Duplicar la dirección IP con otro dispositivo.
  • No configurar correctamente la máscara de subred.
  • No tener acceso a Internet debido a errores en la configuración de DNS.

Comparativa: Asignaciones Dinámicas vs Estáticas

Dinámica Estática
Control del usuario Bajo Alto
Facilidad de gestión Alta Baja
Potencial conflictos IP Sí No

Mini-Quiz: Tipos de Asignaciones

¿Cuál es la principal ventaja de las asignaciones dinámicas?

  • Reducción de errores humanos.
  • Facilita la gestión de dispositivos móviles y remotos.
  • Proporciona una dirección IP constante para dispositivos estáticos.

Checklist: Configuración de Asignaciones Dinámicas

  • Configurar el servidor DHCP.
  • Asignar una IP a un dispositivo.
  • Verificar la configuración de puerta de enlace y DNS.
  • Probar el acceso a Internet.

Checklist: Configuración de Asignaciones Estáticas

  • Configurar la dirección IP manualmente.
  • Configurar la máscara de subred.
  • Configurar los servidores DNS.
  • Probar el acceso a Internet.

Actividad: Configuración de Asignaciones Dinámicas

Realizar la configuración de una máquina virtual para que obtenga su dirección IP automáticamente.

Actividad: Configuración de Asignaciones Estáticas

Configurar manualmente la dirección IP, máscara de subred y servidores DNS en una máquina virtual.

Resumen

  • Dinámica: Asignada automáticamente por DHCP.
  • Estática: Configurada manualmente por el usuario.
  • Tipos adicionales: Automática con DNS, Manual con DNS, Basada en MAC.
