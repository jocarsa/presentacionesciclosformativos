Definición de Excepciones

Una excepción es un evento inesperado que ocurre durante la ejecución de un programa, interrumpiendo el flujo normal del programa.

Causas comunes de excepciones

  • Errores en los datos de entrada
  • Problemas de conexión a bases de datos
  • Fallos en la autenticación
  • Desbordamientos de memoria
  • Divisiones porero

Estructura de un bloque try-catch


try {
    // Código que puede generar una excepción
} catch (TipoDeExcepcion e) {
    // Código para manejar la excepción
}

Tipos de excepciones en

  • Checked Exception: Excepciones que deben ser declaradas o manejadas.
  • Unchecked Exception (RuntimeException): Excepciones no comprobadas, generalmente errores del programador.
  • Error: Errores graves que no pueden ser recuperados por el programa.

Manejo de excepciones personalizado

Puedes crear tus propias excepciones extendiendo las clases existentes como `Exception` o `RuntimeException`.

Ejemplo de excepción personalizada


public class MiExcepcion extends Exception {
    public MiExcepcion(String mensaje) {
        super(mensaje);
    }
}

Uso de finally

El bloque `finally` se ejecuta siempre, independientemente de si ocurre una excepción o no.

Ejemplo de uso de finally


try {
    // Código que puede generar una excepción
} catch (Exception e) {
    // Manejo de la excepción
} finally {
    // Código que siempre se ejecuta
}

Mini-Quiz: Excepciones en

¿Cuál es la diferencia entre `Exception` y `RuntimeException`?

  1. Exception: Excepciones comprobadas.
  2. RuntimeException: Excepciones no comprobadas.

Buenas prácticas en el manejo de excepciones

  • Captura las excepciones más específicas primero.
  • Propague las excepciones cuando no puedas manejarlas.
  • Mantén los mensajes de error claros y descriptivos.

Errores comunes en el manejo de excepciones

  • No capturar excepciones que no puedas manejar.
  • No propagar excepciones sin información útil.
  • No usar `catch (Exception e)` sin procesamiento adecuado.

Comparativa: Try-catch vs. If-else

Try-catch: Mejor para manejar errores inesperados y mantener el flujo normal del programa.

If-else: Mejor para verificar condiciones conocidas antes de ejecutar código.

Checklist: Manejo de excepciones

  • Captura las excepciones específicas.
  • Propaga las excepciones cuando no puedas manejarlas.
  • Mantén mensajes de error claros.
  • Usa `finally` para liberar recursos.

Resumen

  • Excepciones son eventos inesperados durante la ejecución del programa.
  • Manejo eficiente de excepciones es crucial para la robustez de los sistemas.
  • Usa `try-catch`, `finally` y propaga excepciones cuando sea necesario.

Normativa (general)

ISO/IE 9126: Modelo de calidad para el software, que incluye aspectos como la robustez y la confiabilidad.
