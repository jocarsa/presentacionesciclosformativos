Portada

Creación, modificación y eliminación de vistas en sistemas gestores de bases de datos

Objetivo: Aprender a crear, modificar y eliminar vistas para mejorar la accesibilidad y eficiencia del acceso a los datos.

¿Qué son las vistas?

  • Vistas son objetos virtuales que representan una consulta SQL predefinida.
  • No almacenan datos físicamente, sino que se generan en tiempo real basándose en consultas subyacentes.
  • Permiten a los usuarios interactuar con los datos de manera más sencilla y segura.

Crea una vista

Sintaxis básica:

CREATE VIEW nombre_vista AS
SELECT columna1, columna2, ...
FROM tabla
WHERE condicion;

Ejemplo de creación de vista

CREATE VIEW empleados_jefe AS
SELECT e.id_empleado, e.nombre, j.nombre AS nombre_jefe
FROM empleados e
JOIN empleados j ON e.jefe_id = j.id_empleado;

Modifica una vista

Sintaxis básica:

ALTER VIEW nombre_vista AS
SELECT columna1, columna2, ...
FROM tabla
WHERE condicion_nueva;

Ejemplo de modificación de vista

ALTER VIEW empleados_jefe AS
SELECT e.id_empleado, e.nombre, j.nombre AS nombre_jefe
FROM empleados e
JOIN empleados j ON e.jefe_id = j.id_empleado
WHERE e.activo = TRUE;

Elimina una vista

Sintaxis básica:

DROP VIEW nombre_vista;

Ejemplo de eliminación de vista

DROP VIEW empleados_jefe;

Ventajas de usar vistas

  • Simplifican las consultas complejas.
  • Protegen los datos ocultando la estructura interna del esquema.
  • Pueden mejorar el rendimiento al cachear resultados frecuentemente solicitados.

Desventajas de usar vistas

  • No almacenan datos, solo la definición de la consulta.
  • Pueden afectar el rendimiento si no están optimizadas adecuadamente.
  • Requieren permisos adicionales para crear, modificar y eliminar vistas.

Buenas prácticas al trabajar con vistas

  • Documenta las vistas con comentarios en la definición SQL.
  • Mantén las vistas actualizadas y optimizadas según los cambios en el esquema de datos.
  • Limita el acceso a las vistas solo para usuarios autorizados.

Errores comunes al trabajar con vistas

  • No considerar la actualización de las vistas cuando cambia el esquema subyacente.
  • No optimizar las consultas que forman parte de las vistas.
  • Dar acceso a vistas sin control adecuado, lo que puede comprometer la seguridad de los datos.

Comparativa: Vistas vs. Tablas Virtuales

Vistas Tablas Virtuales
No almacenan datos físicamente. Almacenan datos en memoria o disco.
Se generan dinámicamente. Son objetos estáticos.
No pueden ser modificadas directamente. Pueden ser modificadas directamente.

Checklist de creación y gestión de vistas

  1. Definir claramente la finalidad de la vista.
  2. Optimizar la consulta subyacente para mejorar el rendimiento.
  3. Documentar la vista con comentarios en la definición SQL.
  4. Mantener los permisos adecuados para acceder a la vista.

Actividad práctica

Crea una vista que muestre el nombre del cliente, su dirección y el número de pedidos realizados por cada cliente. Utiliza las tablas 'clientes' y 'pedidos'.

Resumen

  • Vistas son objetos virtuales que representan consultas SQL predefinidas.
  • Sirven para simplificar consultas, proteger datos y mejorar rendimiento.
  • Se pueden crear, modificar y eliminar usando las sentencias CREATE VIEW, ALTER VIEW y DROP VIEW respectivamente.

Normativa general

La creación y gestión de vistas deben seguir las mejores prácticas estableidas por el sistema gestor de bases de datos y las políticas de seguridad de la organización.
