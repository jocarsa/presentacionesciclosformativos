Acceso a la información: Creación y eliminación de usuarios

Objetivo: Comprender los procesos para crear y eliminar usuarios en sistemas gestores de bases de datos.

Concepto básico

  • Usuario: Entidad que accede a una base de datos.
  • Credenciales: Nombre de usuario y contraseña para autenticarse.
  • Rol: Conjunto de permisos asignados a un usuario.

Proceso de creación de usuarios

  1. Identificar el nombre de usuario y la contraseña.
  2. Especificar los roles o permisos necesarios.
  3. Utilizar comandos SQL como CREATE USER.
  4. Asignar la contraseña mediante un comando como SET PASSWORD.

Ejemplo de creación de usuario

CREATE USER 'usuario1'@'localhost' IDENTIFIED BY 'contraseña1';
GRANT SELECT, INSERT ON database_name.* TO 'usuario1'@'localhost';

Caso práctico: Creación de usuario para un proyecto

Se necesita crear un usuario para el equipo de marketing que solo tenga permisos de lectura en la base de datos.

Buenas prácticas al crear usuarios

  • Usar contraseñas fuertes y únicas.
  • Asignar roles específicos según necesidad.
  • No compartir credenciales.

Errores comunes al crear usuarios

  • Usar contraseñas débiles o fáciles de adivinar.
  • No asignar roles adecuados.
  • Compartir credenciales con otros usuarios.

Proceso de eliminación de usuarios

  1. Identificar el usuario a eliminar.
  2. Utilizar comandos SQL como DROP USER.
  3. Confirmar la eliminación.

Ejemplo de eliminación de usuario

DROP USER 'usuario1'@'localhost';

Caso práctico: Eliminación de un usuario inactivo

Se necesita eliminar un usuario que no ha utilizado la base de datos en el último año.

Buenas prácticas al eliminar usuarios

  • Confirmar la eliminación antes de ejecutar el comando.
  • No eliminar usuarios que aún puedan tener datos importantes.

Errores comunes al eliminar usuarios

  • Eliminar usuarios por error.
  • No confirmar la eliminación antes de ejecutar el comando.

Mini-quiz: Creación y eliminación de usuarios

¿Cuál es el comando SQL para crear un usuario con permisos de lectura en una base de datos?

  1. CREATE USER 'usuario'@'localhost' IDENTIFIED BY 'contraseña'; GRANT SELECT ON database_name.* TO 'usuario'@'localhost';
  2. CREATE USER 'usuario'@'localhost' IDENTIFIED BY 'contraseña'; GRANT ALL PRIVILEGES ON database_name.* TO 'usuario'@'localhost';
  3. CREATE USER 'usuario'@'localhost' IDENTIFIED BY 'contraseña'; REVOKE SELECT ON database_name.* FROM 'usuario'@'localhost';

Checklist: Creación y eliminación de usuarios

  • Utilizar comandos SQL como CREATE USER.
  • Asignar la contraseña mediante un comando como SET PASSWORD.
  • Confirmar la eliminación antes de ejecutar el comando DROP USER.
