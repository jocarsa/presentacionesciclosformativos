Definición de Transacción Distribuida

Una transacción distribuida es un conjunto de operaciones que se realizan en diferentes componentes del sistema y deben ser tratadas como una unidad única.

Ejemplo de Transacción Distribuida

  • Transferencia bancaria entre dos cuentas en diferentes bancos.
  • Pedidos en línea que involucran múltiples servicios (compra, envío, facturación).

Características de las Transacciones Distribuidas

  • Autoridad:ada componente tiene autoridad sobre sus datos.
  • Asincronía: Operaciones pueden realizarse en diferentes momentos.
  • Conurrencia: Pueden existir múltiples transacciones simultáneas.

Proceso de Ejecución de una Transacción Distribuida

  1. Iniciar la transacción: El cliente inicia la transacción en el servidor local.
  2. Propagar la transacción: La transacción se propaga a los servidores remotos involucrados.
  3. Ejecutar la transacción:ada servidor ejecuta las operaciones correspondientes.
  4. Confirmar o abortar la transacción: Se confirma si todas las operaciones son exitosas, o se aborta si ocurre un error.

Buenas Prácticas para Transacciones Distribuidas

  • Usar protocolos de comunicación confiables (TCP/IP).
  • Implementar mecanismos de concurrencia y bloqueo.
  • Monitorear y gestionar los tiempos de respuesta.

Errores Comunes en Transacciones Distribuidas

  • Desacoplamiento: Falta de comunicación entre componentes.
  • Fallo de un componente: Un servidor no responde o falla.
  • Conflicto de concurrencia: Dos transacciones intentan modificar los mismos datos simultáneamente.

Mecanismos para Manejar Errores en Transacciones Distribuidas

  • Afirmación: Confirmar la transacción si todas las operaciones son exitosas.
  • Aborto: Anular la transacción si ocurre un error.
  • Rol de coordinador: Un servidor que coordina y gestiona las transacciones.

Normativa Relacionada con Transacciones Distribuidas

ACID (Atomicidad, Consistencia, Isolamiento, Durabilidad): Propiedades esenciales para garantizar la integridad de las transacciones distribuidas.

Checklist: Ejecución de una Transacción Distribuida

  • Propagar la transacción a los servidores remotos.
  • Ejecutar las operaciones correspondientes en cada servidor.
  • Confirmar o abortar la transacción según el resultado de las operaciones.
