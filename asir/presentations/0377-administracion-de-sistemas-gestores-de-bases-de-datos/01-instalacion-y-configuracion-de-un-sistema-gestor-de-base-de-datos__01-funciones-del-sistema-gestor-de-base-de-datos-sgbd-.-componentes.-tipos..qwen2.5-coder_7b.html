Funciones del Sistema Gestor de Bases de Datos (SGBD)

Objetivo: Comprender las funciones principales, componentes y tipos de SGBDs.

Definición de Sistema Gestor de Bases de Datos (SGBD)

SGBD es un software que permite crear, gestionar, manipular y mantener las bases de datos.

Funciones Principales del SGBD

  • Definición de esquemas de datos
  • Almacenamiento de datos
  • Procesamiento y recuperación de datos
  • Control de concurrencia
  • Seguridad y control de acceso

Ejemplo: Creación de una Base de Datos en MySQL

CREATE DATABASE biblioteca;

Componentes del SGBD

  • Interfaz de usuario (GUI)
  • Servidor de bases de datos
  • Lenguaje de definición de datos (DDL)
  • Lenguaje de manipulación de datos (DML)
  • Lenguaje de control de acceso (ACL)

Tipos de Sistemas Gestores de Bases de Datos

  • SGBDs Relacionales (RDBMS)
  • SGBDs No Relacionales (NoSQL)
  • SGBDs Orientados a Objetos (ODBMS)
  • SGBDs Orientados a Documentos

Ejemplo de SGBD Relacional: PostgreSQL

PostgreSQL es un RDBMS popular conocido por su robustez y funcionalidad.

Ejemplo de SGBD No Relacional: MongoDB

MongoDB es un SGBD NoSQL basado en documentos, ideal para aplicaciones web.

Características de los SGBDs Orientados a Objetos (ODBMS)

  • Soportan objetos y relaciones entre ellos
  • Facilitan la programación orientada a objetos
  • Menos eficientes en comparación con RDBMS

Ejemplo de SGBD Orientado a Objetos: db4o

db4o es un ODBMS que permite almacenar objetos .NET directamente en la base de datos.

Características de los SGBDs Orientados a Documentos

  • Almacenan datos en formato o BSON
  • Fáciles de escalar horizontalmente
  • No requieren esquemas rígidos

Ejemplo de SGBD Orientado a Documentos: Couchbase

Couchbase es un SGBD NoSQL orientado a documentos que ofrece escalabilidad y alta disponibilidad.

Comparación RDBMS vs. NoSQL

RDBMS NoSQL
Estructura de datos tabular Estructura de datos flexible (, BSON)
Relaciones entre tablas No relaciones predefinidas
Escalabilidad vertical Escalabilidad horizontal

Buenas Prácticas en la Selección de SGBD

  • Evaluación de requisitos funcionales y no funcionales
  • Consideración del tipo de datos a almacenar
  • Factoría de rendimiento y escalabilidad
  • Seguridad y cumplimiento normativo

Caso Práctico: Selección de SGBD para una Aplicación E-commerce

Para aplicaciones e-commerce, un RDBMS como PostgreSQL o MySQL suele ser la mejor opción.

Errores Comunes al Seleccionar un SGBD

  • No considerar el tipo de datos a almacenar
  • No evaluar las necesidades de escalabilidad
  • No tener en cuenta la seguridad y cumplimiento normativo

Resumen

Los SGBDs desempeñan un papel crucial en la gestión de datos en aplicaciones modernas. Comprendiendo sus funciones, componentes y tipos, se pueden seleccionar los más adecuados para cada proyecto.

Normativa y Regulaciones

Es importante cumplir con las regulaciones de protección de datos (como GDPR) al seleccionar y utilizar SGBDs.
