Herramientas y sentencias para la gestión de índices

Objetivo: Aprender las herramientas y sentencias clave para gestionar eficientemente los índices en sistemas gestores de bases de datos.

Herramientas para la gestión de índices

  • MySQL: `CREATE INDEX`, `DROP INDEX`, `ALTER TABLE ... ADD INDEX`
  • PostgreSQL: `CREATE INDEX`, `DROP INDEX`, `CREATE INDEX CONCURRENTLY`
  • SQL Server: `CREATE INDEX`, `DROP INDEX`, `CREATE NONCLUSTERED INDEX`
  • Oracle: `CREATE INDEX`, `DROP INDEX`, `CREATE BITMAP INDEX`

Sentencias para la gestión de índices

  • `CREATE INDEX`:rea un índice en una tabla.
  • `DROP INDEX`: Elimina un índice existente.
  • `ALTER TABLE ... ADD INDEX`: Añade un nuevo índice a una tabla existente.
  • `ANALYZE TABLE`: Actualiza estadísticas de la tabla para mejorar el rendimiento del optimizador.

Ejemplo: Creación de índices en MySQL

CREATE INDEX idx_lastname ON employees (last_name);

Ejemplo: Eliminación de índices en PostgreSQL

DROP INDEX idx_lastname;

Caso práctico: Optimización de consultas con índices

Considera una consulta que realiza búsquedas frecuentes por `last_name` en la tabla `employees`. Crear un índice en `last_name` puede mejorar significativamente el rendimiento.

Buenas prácticas para la gestión de índices

  • Crear índices solo en columnas que se utilizan en cláusulas `WHERE`, `JOIN` o `ORDER BY`.
  • Mantener un equilibrio entre el número de índices y el rendimiento de las operaciones de inserción, actualización y eliminación.
  • Usar índices compuestos cuando sea necesario para mejorar la eficiencia en consultas complejas.

Errores comunes al gestionar índices

  • Crea demasiados índices sin considerar el impacto en el rendimiento de las operaciones de escritura.
  • No analiza regularmente las tablas para mantener estadísticas actualizadas.
  • Elimina índices que no se utilizan, pero olvida los que son esenciales para ciertas consultas.

Comparativa de tipos de índices

Tipo de Índice Descripción
B-Tree Más común, ordenado y equilibrado.
Hash Rápido acceso a los datos, pero no soporta rangos.
Bloom Filter Estimación de la existencia de elementos sin almacenarlos.

Mini-quiz: ¿Cuál es el propósito del análisis de tablas?

  • Aumentar el tamaño de la tabla para mejorar el rendimiento.
  • Actualizar estadísticas utilizadas por el optimizador de consultas.
  • Eliminar índices innecesarios.

Checklist: Consideraciones al crear índices

  1. Identifica las columnas más utilizadas en consultas.
  2. Evaluación del impacto en operaciones de escritura.
  3. Mantén un solo índice por columna si es posible.
  4. Prueba el rendimiento antes y después de la creación del índice.
