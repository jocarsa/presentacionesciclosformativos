Ficheros LOG en Sistemas Gestores de Bases de Datos

Objetivo: Comprender la importancia y funciones de los ficheros LOG en el funcionamiento de un sistema gestor de bases de datos.

¿Qué son los Ficheros LOG?

  • Registros de eventos importantes que ocurren durante la operación del SGBD.
  • Permiten el seguimiento y diagnóstico de problemas.
  • Almacenan información sobre transacciones, errores y cambios en los datos.

Tipos de Ficheros LOG

  • Fichero de Registro General (General Log): Registra todas las consultas SQL ejecutadas.
  • Fichero de Error (Error Log): Almacena mensajes de error y advertencias.
  • Fichero de Bitácora de Transacciones (Transaction Log): Registra cambios en los datos para operaciones de recuperación.

Importancia de los Ficheros LOG

  • Diagnóstico: Ayudan a identificar problemas y causas de errores.
  • Seguridad: Almacenan auditorías que pueden ser utilizadas para cumplir con normativas de seguridad.
  • Recuperación: Son esenciales para la restauración de datos en caso de fallos.

Configuración de los Ficheros LOG

  • Ubicación y tamaño del fichero.
  • Nivel de detalle (verbosidad).
  • Rotación automática para evitar el llenado del disco.

Ejemplo de Configuración en MySQL


[mysqld]
log_error=/var/log/mysql/error.log
general_log=1
general_log_file=/var/log/mysql/general.log
innodb_log_file_size=1G
innodb_log_files_in_group=2

Errores Comunes en Ficheros LOG

  • No se encuentra el fichero de log.
  • Fichero de log lleno o sin espacio.
  • Nivel de detalle incorrecto.

Buenas Prácticas para Ficheros LOG

  • Configurar el tamaño y rotación adecuados.
  • Ajustar el nivel de detalle según las necesidades.
  • Monitorear regularmente los ficheros de log.
