Servicios de acceso remoto del propio sistema operativo

Los sistemas operativos ofrecen herramientas para acceder y administrar sistemas desde una ubicación remota.

SSH (Secure Shell)

  • Protocolo de red utilizado para proporcionar una conexión segura entre dos computadoras.
  • Cifra todos los datos transmitidos, incluyendo el nombre de usuario y la contraseña.
  • Permite ejecutar comandos en un servidor remoto desde una máquina local.

RDP (Remote Desktop Protocol)

  • Protocolo de Microsoft para acceder a una computadora remota desde una máquina local.
  • Permite controlar la pantalla del servidor como si estuviera directamente conectado.
  • Suele requerir autenticación y puede cifrar los datos transmitidos.

VNC (Virtual Network Computing)

  • Protocolo de red que permite el acceso remoto a una máquina gráfica.
  • Permite ver y controlar la pantalla del servidor desde una máquina local.
  • Suele requerir autenticación y puede cifrar los datos transmitidos.

FTP (File Transfer Protocol)

  • Protocolo utilizado para transferir archivos entre computadoras en una red.
  • Suele requerir autenticación y puede cifrar los datos transmitidos.
  • No es seguro para el tráfico de datos sensibles.

SCP (Secure Copy Protocol)

  • Herramienta basada en SSH que permite transferir archivos entre computadoras de forma segura.
  • Cifra todos los datos transmitidos, incluyendo el nombre de usuario y la contraseña.
  • Es más seguro que FTP para el tráfico de datos sensibles.

SFTP (SSH File Transfer Protocol)

  • Herramienta basada en SSH que permite transferir archivos entre computadoras de forma segura.
  • Cifra todos los datos transmitidos, incluyendo el nombre de usuario y la contraseña.
  • Es más seguro que FTP para el tráfico de datos sensibles.

Buenas prácticas

  • Utilizar protocolos seguros como SSH, RDP o VNC.
  • Cambiar regularmente las contraseñas de acceso remoto.
  • Configurar firewalls para limitar el acceso a los servicios remotos.

Errores comunes

  • No utilizar protocolos seguros, lo que puede exponer datos sensibles.
  • No cambiar regularmente las contraseñas de acceso remoto.
  • No configurar firewalls adecuadamente, lo que puede permitir el acceso no autorizado.

Comparativa

Protocolo Descripción Cifrado Uso
SSH Protocolo de red utilizado para proporcionar una conexión segura entre dos computadoras. Sí Acceso remoto, ejecución de comandos.
RDP Protocolo de Microsoft para acceder a una computadora remota desde una máquina local. Sí Control remoto de la pantalla.

Checklist

  • Configurar SSH para acceso remoto seguro.
  • Cambiar regularmente las contraseñas de acceso remoto.
  • Configurar firewalls para limitar el acceso a los servicios remotos.
  • Utilizar protocolos seguros para transferir archivos.

Actividad

Realice una configuración básica de SSH en su sistema operativo y pruebe el acceso remoto desde otra máquina.

Resumen

  • Servicios de acceso remoto permiten controlar sistemas desde una ubicación remota.
  • Protocolos como SSH, RDP y VNC son utilizados para este propósito.
  • Buenas prácticas incluyen el uso de protocolos seguros y la configuración adecuada de firewalls.

Normativa (general)

La normativa vigente establece requisitos para la seguridad en sistemas operativos, incluyendo los protocolos utilizados para el acceso remoto.
