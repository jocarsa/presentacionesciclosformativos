Objetivo: Comprender la definición, los elementos y la nomenclatura del servicio de directorio y conocer las principales características de LDAP.

Definición del Servicio de Directorio

Un servicio de directorio es una base de datos centralizada que almacena información sobre los usuarios, grupos y recursos de red.

Elementos del Servicio de Directorio

  • Objetos (Users, Groups, Computers)
  • Atributos (Name, Email, Password, etc.)
  • Hierarquía (Organizational Units)
  • Reglas de acceso y autenticación

Nomenclatura del Servicio de Directorio

Los objetos en un servicio de directorio se organizan en una estructura jerárquica, similar a un árbol.

Ejemplo de Estructura Jerárquica

  • DC=example, DC=com (Dominio)
  • OU=Users (Unidad Organizativa)
  • CN=john.doe (Nombre Común)

Características de LDAP

  • Protocolo de acceso distribuido
  • Búsqueda y recuperación rápida de datos
  • Soporte para diferentes tipos de objetos
  • Seguridad mediante encriptación SSL/TLS

Ejemplo de LDAP Query


ldapsearch -x -b "dc=example,dc=com" "(uid=john.doe)"

Caso Práctico: Implementación de LDAP en una Empresa

Una empresa decide implementar un servicio de directorio basado en LDAP para gestionar la información de sus usuarios y recursos.

Buenas Prácticas en el Uso de LDAP

  • Seguridad: Implementar SSL/TLS
  • Mantenimiento: Regularmente actualizar y parchear el sistema
  • Backup: Realizar copias de seguridad periódicas

Errores Comunes en la Implementación de LDAP

  • No implementar SSL/TLS, exponiendo información sensible
  • No realizar copias de seguridad, perdiendo datos importantes
  • No mantener el sistema actualizado, expuesto a vulnerabilidades

Comparativa: LDAP vs. Active Directory

LDAP Active Directory
Protocolo de acceso distribuido Sistema gestionado por Microsoft
