Administración de procesos del sistema

Objetivo: Comprender los conceptos básicos y transiciones de estados en la administración de procesos del sistema.

Definición de Proceso

Un proceso es una instancia de un programa en ejecución. Incluye:

  • Código ejecutable
  • Datos asociados
  • Estado actual
  • Recursos asignados

Estados de un Proceso

Los procesos pueden estar en uno de los siguientes estados:

  • Nuevo (New)
  • Listo (Ready)
  • Ejecutando (Running)
  • Bloqueado (Blocked)
  • Terminado (Terminated)

Ciclo de Vida de un Proceso

El ciclo de vida de un proceso incluye:

  1. Nacimiento: El proceso se crea.
  2. Listo: El sistema operativo elige procesos para ejecutar.
  3. Ejecución: El proceso realiza sus tareas.
  4. Bloqueo: El proceso espera por un recurso.
  5. Terminación: El proceso finaliza su ejecución.

Transición de Estados

Las transiciones entre estados ocurren debido a eventos específicos:

  • Nuevo → Listo: El proceso se carga en memoria.
  • Listo → Ejecución: El procesador elige un proceso para ejecutar.
  • Ejecución → Bloqueado: El proceso espera por un recurso.
  • Bloqueado → Listo: El recurso se libera y el proceso vuelve a la lista de listos.
  • Terminación: El proceso finaliza su ejecución.

Ejemplo de Transición

Caso práctico:

  1. Un usuario inicia un programa (Nuevo → Listo).
  2. El sistema elige este proceso para ejecutar (Listo → Ejecución).
  3. El proceso necesita acceso a una base de datos (Ejecución → Bloqueado).
  4. La base de datos se libera, el proceso vuelve a la lista de listos (Bloqueado → Listo).
  5. El sistema elige este proceso nuevamente para ejecutar (Listo → Ejecución).

Caso Práctico: Administración de Procesos

Ejemplo real:

  • Un servidor web maneja múltiples solicitudes simultáneas.
  • Cada solicitud crea un proceso nuevo (Nuevo).
  • Los procesos se ejecutan en orden de llegada (FIFO).
  • Si ocurre un error, el proceso pasa a estado bloqueado (Bloqueado).
  • Una vez resuelto el problema, el proceso vuelve a la lista de listos (Listo).

Proceso de Administración de Procesos

El sistema operativo realiza tareas como:

  • Planificación: Decidir qué proceso ejecutar.
  • Sincronización: Coordinar el acceso a recursos compartidos.
  • Control de errores: Manejar excepciones y fallos.
  • Monitoreo: Seguimiento del estado de los procesos.

Buenas Prácticas en Administración de Procesos

Consideraciones:

  • Eficiencia: Minimizar el tiempo de espera y la sobrecarga.
  • Seguridad: Proteger los recursos contra accesos no autorizados.
  • Escalabilidad: Manejar un creciente número de procesos.
  • Consistencia: Mantener el estado correcto de los procesos.

Errores Comunes en Administración de Procesos

Fallas típicas:

  • No planificación adecuada: Cola de procesos larga.
  • Sincronización fallida: Recursos compartidos corruptos.
  • Control de errores insuficiente: Procesos no responden.
  • Monitoreo ineficaz: Falta de información sobre el estado de los procesos.

Comparativa entre Sistemas Operativos

Diferencias en la administración de procesos:

Sistema Operativo Administración de Procesos
Linux Planificación por prioridad, sincronización mediante semáforos.
Windows Planificación por tiempo de CPU, sincronización mediante eventos.
macOS Planificación por prioridad, sincronización mediante barreras.

Resumen: Transiciones de Estados

Puntos clave:

  • Procesos tienen estados distintos (Nuevo, Listo, Ejecución, Bloqueado, Terminado).
  • Las transiciones ocurren debido a eventos específicos.
  • El sistema operativo gestiona eficientemente la administración de procesos.

Normativa y Estándares

Estándares relevantes:
