Herramientas gráficas externas para la administración remota

Introducción a las herramientas gráficas que facilitan el acceso y gestión remota de sistemas operativos.

Portátil RDP (Remote Desktop Protocol)

  • Sistema nativo de Windows para conexión remota.
  • Permite control remoto y acceso a aplicaciones.
  • Requiere que el servidor esté configurado para permitir conexiones RDP.

VNC (Virtual Network Computing)

  • Herramienta de software libre y pago.
  • Permite control remoto de sistemas operativos Linux, Windows y macOS.
  • Su uso es gratuito para el cliente, pero requiere licencias para servidores comerciales.

TeamViewer

  • Herramienta de software pago con versión gratuita limitada.
  • Permite control remoto, transmisión de pantalla y chat.
  • Su uso es gratuito para el cliente, pero requiere licencias para servidores comerciales.

AnyDesk

  • Herramienta de software pago con versión gratuita limitada.
  • Permite control remoto, transmisión de pantalla y chat.
  • Su uso es gratuito para el cliente, pero requiere licencias para servidores comerciales.

Remmina

  • Herramienta de software libre para Linux.
  • Soporta RDP, VNC y SSH entre otros protocolos.
  • Su uso es gratuito y no requiere licencias adicionales.

SecureCRT

  • Herramienta de software pago para Windows, macOS y Linux.
  • Permite conexión SSH, Telnet, Rlogin y otros protocolos.
  • Su uso es gratuito para el cliente, pero requiere licencias para servidores comerciales.

Putty

  • Herramienta de software libre para Windows, macOS y Linux.
  • Permite conexión SSH, Telnet, Rlogin y otros protocolos.
  • Su uso es gratuito y no requiere licencias adicionales.

Casos de uso

  • Administración remota de servidores web.
  • Soporte técnico a clientes en diferentes ubicaciones.
  • Desarrollo y pruebas de aplicaciones en entornos remotos.

Buenas prácticas

  • Utilizar protocolos seguros como SSH para conexiones remotas.
  • Configurar firewalls y seguridad adecuada en servidores remotos.
  • Usar autenticación de dos factores cuando sea posible.

Errores comunes

  • No configurar correctamente los permisos de acceso remoto.
  • No utilizar protocolos seguros para conexiones remotas.
  • No mantener actualizadas las herramientas y sistemas operativos.

Checklist

  • Configurar el servidor para permitir conexiones remotas.
  • Utilizar un protocolo seguro (SSH).
  • Establecer políticas de seguridad y autenticación.
  • Mantener las herramientas y sistemas operativos actualizados.

Resumen

  • Herramientas gráficas externas facilitan el acceso y gestión remota de sistemas operativos.
  • Existen varias opciones disponibles, cada una con sus propias características y ventajas.
  • Es importante seleccionar herramientas seguras y seguir buenas prácticas para garantizar la seguridad en conexiones remotas.

Normativa general

La normativa vigente establece requisitos mínimos de seguridad para el acceso remoto a sistemas informáticos, incluyendo la necesidad de autenticación segura y el uso de protocolos cifrados.
