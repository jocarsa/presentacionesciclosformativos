  • Ejercitar la creación y ejecución de scripts para automatizar tareas administrativas.

¿Qué son los Scripts?

Los scripts son secuencias de comandos que se ejecutan automáticamente en un sistema operativo. Ayudan a automatizar tareas repetitivas y mejorar la eficiencia administrativa.

Lenguajes de Scripting Comunes

  • Bash: Lenguaje estándar para sistemas operativos Linux y Unix.
  • PowerShell: Lenguaje de scripting para Windows, basado en .NET Framework.
  • : Lenguaje versátil que se puede usar en múltiples plataformas.

Ejemplo de Script


#!/bin/bash
# Crear un nuevo usuario
echo "Ingrese el nombre del nuevo usuario:"
read username
useradd $username
passwd $username
echo "Usuario $username creado con éxito."

Ejemplo de Script PowerShell


# Crear un nuevo usuario
$username = Read-Host "Ingrese el nombre del nuevo usuario"
New-LocalUser -Name $username -Password (ConvertTo-SecureString -AsPlainText "P@ssw0rd" -Force)
Write-Output "Usuario $username creado con éxito."

Ejemplo de Script


import os
# Crear un nuevo usuario
username = input("Ingrese el nombre del nuevo usuario: ")
os.system(f"useradd {username}")
os.system(f"passwd {username}")
print(f"Usuario {username} creado con éxito.")

Administración de Procesos

Los scripts también pueden utilizarse para administrar procesos del sistema operativo, como iniciar, detener o reiniciar servicios.

Ejemplo de Script para Administrar Procesos


#!/bin/bash
# Iniciar un servicio
service apache2 start
echo "Servicio Apache iniciado."

Administración de Servicios

Los scripts pueden automatizar la gestión de servicios del sistema operativo, como reiniciar o detener servicios.

Ejemplo de Script para Administrar Servicios


#!/bin/bash
# Reiniciar un servicio
service apache2 restart
echo "Servicio Apache reiniciado."

Buenas Prácticas en la Creación de Scripts

  • Comentar el código para facilitar su comprensión y mantenimiento.
  • Usar variables para almacenar datos que pueden cambiar.
  • Manejar errores con comandos como `if` y `else`.

Errores Comunes en los Scripts

  • No manejo adecuado de errores.
  • Falta de comprobación de permisos.
  • Scripts mal escritos que causan problemas de rendimiento.

Checklist para la Creación de Scripts

  • Definir claramente el propósito del script.
  • Comprobar los permisos necesarios antes de ejecutar comandos.
  • Probar el script en un entorno seguro antes de usarlo en producción.
