Extensiones de comandos en Sistemas Operativos

Las extensiones de comandos permiten automatizar tareas repetitivas y mejorar la eficiencia administrativa.

Ejemplo: Script para monitoreo de discos

#!/bin/bash
df -h | grep '^/dev/sd' | awk '{print $1, $5}'

Ejemplo: Script PowerShell para reinicio de servicios

$services = Get-Service -Name "Spooler", "EventLog"
foreach ($service in $services) {
    Restart-Service -InputObject $service
}

Proceso de creación de un script

  1. Especificar el intérprete (por ejemplo, #!/bin/bash).
  2. Escribir las instrucciones del script.
  3. Hacer que el script sea ejecutable con chmod +x nombre_script.sh.
  4. Probar y depurar el script.

Buenas prácticas en scripting

  • Comentar el código para facilitar su lectura.
  • Usar variables para almacenar valores reutilizables.
  • Manejar errores y excepciones de manera adecuada.

Errores comunes en scripting

  • No tener permisos suficientes para ejecutar el script.
  • Sintaxis incorrecta del código.
  • Variables no definidas o mal utilizadas.

Mini-quiz: ¿Cuál es el propósito de un script?

  1. Ejecutar tareas repetitivas automáticamente.
  2. Crear interfaces gráficas de usuario.
  3. Definir funciones matemáticas complejas.

Checklist para la creación de scripts efectivos

  • Identificar las tareas a automatizar.
  • Escribir el código con claridad y concisión.
  • Probar el script en un entorno seguro.
  • Ajustar y optimizar según sea necesario.

Resumen

  • Los lenguajes de scripting facilitan la automatización de tareas administrativas.
  • Se pueden usar en Unix/Linux y PowerShell en Windows.

Normativa general sobre scripting

Seguridad: Evitar scripts con privilegios elevados sin necesidad.

Mantenimiento: Documentar el código y mantenerlo actualizado.
