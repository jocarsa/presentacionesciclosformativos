Administración de procesos del sistema

Objetivo: Comprender los tipos, estados y estructura de los procesos en sistemas operativos.

Definición de Proceso

Un proceso es una instancia de un programa en ejecución. Incluye:

  • Código del programa
  • Datos asociados
  • Estado actual
  • Recursos asignados

Tipos de Procesos

Existen varios tipos de procesos en sistemas operativos:

  • Procesos de usuario: Ejecutados por el usuario.
  • Procesos del sistema: Ejecutados por el sistema operativo.
  • Procesos demonio: Ejecutados en segundo plano.

Estados de un Proceso

Un proceso puede estar en uno de los siguientes estados:

  • Nuevo (recién creado)
  • Listo (listo para ejecutarse)
  • Ejecución (en curso)
  • Bloqueado (esperando un evento)
  • Terminado

Estructura Interna de un Proceso

La estructura interna de un proceso incluye:

  • Pila de llamadas (stack)
  • Área de datos (data segment)
  • Área de código (text segment)
  • Registros del procesador
  • Identificador único (PID)

Ciclo de Vida de un Proceso

El ciclo de vida de un proceso incluye:

  1. Nacimiento: Creación del proceso.
  2. Ejecución: Ejecución del código del programa.
  3. Bloqueo: Espera por algún evento.
  4. Reanudación: Continuación de la ejecución.
  5. Muerte: Finalización del proceso.

Recursos Asignados a los Procesos

Cada proceso tiene acceso a:

  • Memoria virtual
  • Procesador (CPU)
  • Entrada/salida (I/O)
  • Archivos y directorios

Comunicación entre Procesos

Existen varios métodos de comunicación entre procesos:

  • Pipes (tuberías)
  • Sockets
  • Mensajes
  • Memoria compartida

Procesos y Seguridad

Los procesos pueden ser:

  • Privilegiados (con permisos elevados)
  • No privilegiados (con permisos limitados)

La seguridad en sistemas operativos implica controlar el acceso a los recursos por procesos.

Ejemplo de Proceso

Un ejemplo de proceso es un navegador web:

  • Carga páginas web
  • Muestra contenido multimedia
  • Gestiona pestañas y ventanas

Normativas como SELinux y AppArmor controlan el acceso a los recursos por procesos.
