Gestión de los procesos del sistema. Línea de orden. Entorno gráfico.

Objetivo: Comprender las técnicas y herramientas para gestionar eficientemente los procesos del sistema operativo, incluyendo la línea de orden y el entorno gráfico.

Definición de Proceso

Un proceso es una instancia de un programa en ejecución. Incluye:

  • Código del programa
  • Datos asociados
  • Estado actual
  • ID único

Línea de Orden (Command Line)

Es la interfaz textual para interactuar con el sistema operativo. Permite:

  • Ejecutar comandos
  • Gestionar archivos y directorios
  • Monitorear procesos

Comandos Básicos en Línea de Orden

  • ls: Lista archivos y directorios
  • cd: Cambia de directorio
  • pwd: Muestra el directorio actual
  • mkdir:rea un nuevo directorio
  • rm: Elimina archivos o directorios

Entorno Gráfico (GUI)

Interfaz visual que permite interactuar con el sistema operativo. Ofrece:

  • Graficos y iconos para representar archivos y procesos
  • Menús desplegables y ventanas emergentes
  • Barras de herramientas y paneles de control

Interfaz Gráfica vs Línea de Orden

Línea de Orden:

  • Más rápida y directa para tareas específicas
  • No requiere interfaz gráfica
  • Útil para automatización y scripts

Entorno Gráfico:

  • Más intuitivo y fácil de usar para no expertos
  • Permite visualizar y gestionar procesos gráficamente
  • Más adecuado para tareas complejas

Ejemplo: Usando Línea de Orden en Linux

Comando para buscar archivos con extensión .txt:

find /home/user -name "*.txt"

Ejemplo: Usando Entorno Gráfico en Windows

Paso a paso para abrir un archivo de texto:

  1. Navegar hasta la carpeta donde está el archivo
  2. Hacer clic derecho y seleccionar "Abrir con"
  3. Seleccionar el editor de texto deseado (por ejemplo, Notepad)

Buenas Prácticas en Línea de Orden

  • Usar comandos completos y correctos
  • Verificar permisos antes de realizar acciones
  • Utilizar tabulación para autocompletar comandos
  • Redirigir salida a archivos cuando sea necesario

Buenas Prácticas en Entorno Gráfico

  • Organizar archivos y carpetas de manera lógica
  • Usar etiquetas o categorías para facilitar la búsqueda
  • No abrir demasiados programas al mismo tiempo
  • Cerrar ventanas innecesarias para liberar recursos

Errores Comunes en Línea de Orden

  • Comandos mal escritos o faltantes
  • No tener permisos suficientes para realizar acciones
  • Redirigir salida incorrectamente, sobrescribiendo archivos importantes

Errores Comunes en Entorno Gráfico

  • Cerrar accidentalmente programas importantes
  • No guardar cambios antes de cerrar ventanas
  • Perder archivos al moverlos a la papelera sin vaciarla
