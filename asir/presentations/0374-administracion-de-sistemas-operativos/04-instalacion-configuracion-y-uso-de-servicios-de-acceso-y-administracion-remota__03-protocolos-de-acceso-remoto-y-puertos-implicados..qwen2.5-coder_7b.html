Portada

Protocolos de acceso remoto y puertos implicados

Objetivo: Comprender los protocolos utilizados para el acceso remoto a sistemas operativos y conocer los puertos asociados.

Introducción

  • Definición de acceso remoto
  • Importancia en la administración de sistemas
  • Ejemplos de protocolos utilizados

Protocolos de Acceso Remoto

  • RDP (Remote Desktop Protocol): Utilizado principalmente en Windows.
  • VNC (Virtual Network Computing): Versátil y compatible con múltiples sistemas operativos.
  • SSH (Secure Shell): Protocolo de red utilizado para el acceso seguro a servidores.
  • FTP (File Transfer Protocol): Para transferir archivos entre computadoras.

Puertos Implicados en Acceso Remoto

  • RDP (3389): Puerto predeterminado para RDP.
  • VNC (5900-5910): Puertos utilizados por VNC.
  • SSH (22): Puerto estándar para SSH.
  • FTP (21): Puerto utilizado para FTP.

Ejemplo de Configuración RDP

Para configurar RDP en Windows:

  1. Iniciar el "Servicio de Escritorio remoto".
  2. Habilitar la opción "Permitir conexiones remotas a este equipo".
  3. Configurar las opciones de seguridad según sea necesario.

Ejemplo de Configuración SSH

Para configurar SSH en Linux:

  1. Instalar el servidor SSH (`sudo apt-get install opens-server`).
  2. Habilitar el servicio SSH (`sudo systemctl enable ssh`).
  3. Iniciar el servicio SSH (`sudo systemctl start ssh`).

Seguridad en Acceso Remoto

  • Uso de contraseñas fuertes
  • Configuración de firewalls
  • Utilización de protocolos seguros (SSH, RDP con SSL)
  • Limitar el acceso a usuarios autorizados

Caso Práctico: Implementación de SSH

Implementar SSH en un servidor Linux para permitir el acceso remoto seguro.

  1. Instalar OpenSSH (`sudo apt-get install opens-server`).
  2. Habilitar el servicio SSH (`sudo systemctl enable ssh`).
  3. Iniciar el servicio SSH (`sudo systemctl start ssh`).
  4. Configurar el firewall para permitir el tráfico SSH (puerto 22).

Buenas Prácticas

  • Evitar el uso de puertos predeterminados para evitar ataques
  • Utilizar autenticación de dos factores cuando sea posible
  • Mantener actualizados los sistemas y protocolos utilizados
  • Realizar auditorías periódicas de seguridad

Errores Comunes en Acceso Remoto

  • Uso incorrecto de puertos
  • No configuración correcta del firewall
  • Contraseñas débiles o fáciles de adivinar
  • Autorización no adecuada de usuarios

Comparativa entre Protocolos

Protocolo Puerto Características
RDP 3389 Windows, fácil de usar, pero menos seguro que SSH.
VNC 5900-5910 Multisistema, flexible, pero puede ser inseguro si no se configura adecuadamente.
SSH 22 Seguro, multiplataforma, recomendado para acceso remoto a servidores.

  2. Utilizar contraseñas fuertes y únicas.
  3. Habilitar autenticación de dos factores cuando sea posible.
