Concepto de verificación de la información

La verificación de la información es el proceso de comprobar la exactitud, precisión y validez de los datos antes de su uso en aplicaciones web.

Métodos de verificación

  • Validación del lado del cliente ()
  • Validación del lado del servidor (,, etc.)
  • Sanitización de datos
  • Hashing y salting de contraseñas
  • Autenticación y autorización

Ejemplo: Validación del lado del cliente


<script>
function validateForm() {
    var x = document.forms["myForm"]["fname"].value;
    if (x == "") {
        alert("Name must be filled out");
        return false;
    }
}
</script>

Ejemplo: Validación del lado del servidor ()


<?
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $name = htmlspecialchars($_REQUEST['fname']);
    if (empty($name)) {
        echo "Name is empty";
    } else {
        echo "Welcome " . $name;
    }
}
?>

Sanitización de datos

La sanitización implica eliminar o escapar caracteres especiales que podrían ser utilizados para inyección SQL o ataques XSS.

Hashing y salting de contraseñas

  • Hashing: Convierte una contraseña en un valor fijo.
  • Salting: Añade un valor aleatorio a la contraseña antes del hashing.

Autenticación y autorización

La autenticación confirma la identidad de un usuario, mientras que la autorización determina qué acciones puede realizar.

Caso práctico: Sistema de login seguro

  • Validación del formulario de inicio de sesión.
  • Hashing y salting de contraseñas al almacenarlas en la base de datos.
  • Autenticación mediante tokens (JWT).

Buenas prácticas

  • No confiar solo en validaciones del lado del cliente.
  • Realizar siempre validaciones del lado del servidor.
  • Usar librerías de seguridad para evitar inyecciones SQL y XSS.

Errores comunes

  • No validar datos en el lado del servidor.
  • No usar hashing y salting de contraseñas.
  • No implementar autenticación y autorización adecuadas.

Comparativa: Validación vs. Sanitización

Validación Sanitización
Propósito Comprobar la exactitud de los datos. Eliminar o escapar caracteres especiales.
Ejemplo Verificar que un campo no esté vacío. Escapar comillas en una cadena SQL.

¿Qué es la validación del lado del servidor?

  • a) Comprobar que los datos cumplen con ciertas reglas antes de su uso.
  • b) Ocultar información sensible en las páginas web.
  • c) Enviar datos al cliente sin procesamiento.

Checklist: Verificación de la información

  • Validar todos los campos del formulario en el lado del servidor.
  • Usar hashing y salting para almacenar contraseñas.
  • Implementar autenticación y autorización adecuadas.
  • No confiar solo en validaciones del lado del cliente.

Resumen

  • La verificación de la información es crucial para la seguridad de las aplicaciones web.
  • Incluye validación, sanitización, hashing y salting, autenticación y autorización.
  • No confiar solo en validaciones del lado del cliente.

Normativa general

La normativa vigente impone la protección de datos personales y la seguridad informática, como lo establece el Reglamento General de Protección de Datos (RGPD).
