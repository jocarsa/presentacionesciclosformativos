Mecanismos de introducción de información: formularios

Objetivo: Comprender los conceptos básicos y aplicaciones de los formularios en aplicaciones web.

Definición de Formulario

Un formulario es una colección de controles que permiten a los usuarios ingresar datos para ser procesados por un servidor.

Estructura Básica de un Formulario

<form action="/procesar" method="post">
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre"><br>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email"><br>
    <input type="submit" value="Enviar">
</form>

Tipos de Controles en Formularios

  • Input: Campos de texto, números, checkboxes, radios.
  • Select: Menús desplegables.
  • Textarea: Áreas de texto multilínea.
  • Button: Botones personalizados.

Atributos Importantes de los Input

  • type: Define el tipo de control (text, email, password, etc.).
  • name: Nombre del campo para identificarlo en el servidor.
  • id: Identificador único del campo.
  • value: Valor predeterminado del campo.

Métodos de Envío de Formularios

  • GET: Datos visibles en la URL, limitados por tamaño.
  • POST: Datos ocultos en el cuerpo del mensaje, sin límite de tamaño.

Acciones de Formularios

El atributo action especifica la URL donde se enviarán los datos del formulario.

Validación Básica en5

  • required: Campo obligatorio.
  • pattern: Expresión regular para validar el contenido.
  • minlength, maxlength: Longitud mínima y máxima del campo.

Ejemplo de Formulario con Validación

<form action="/procesar" method="post">
    <label for="nombre">Nombre (requerido):</label>
    <input type="text" id="nombre" name="nombre" required><br>
    <label for="email">Email (requerido, formato válido):</label>
    <input type="email" id="email" name="email" required><br>
    <input type="submit" value="Enviar">
</form>

Manejo de Formularios con

Puedes usar para validar formularios dinámicamente antes del envío.

Ejemplo de Validación con

<script>
function validateForm() {
    var x = document.forms["myForm"]["email"].value;
    if (x == "") {
        alert("El email es obligatorio");
        return false;
    }
}
</script>

<form name="myForm" action="/procesar" method="post" onsubmit="return validateForm()">
    <label for="email">Email (requerido):</label>
    <input type="text" id="email" name="email"><br>
    <input type="submit" value="Enviar">
</form>

Formularios y Seguridad

  • Evita la exposición de datos sensibles en URLs.
  • Usa HTTPS para proteger los datos durante el envío.
  • Valida y sanitiza todos los datos del formulario en el servidor.

Caso Práctico: Formulario de Contacto

Un formulario común para contactar a una empresa o persona, incluyendo campos para nombre, email y mensaje.

Ejemplo de Formulario de Contacto

<form action="/contacto" method="post">
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre"><br>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email"><br>
    <label for="mensaje">Mensaje:</label>
    <textarea id="mensaje" name="mensaje">

Mini-Quiz: Formularios

¿Cuál es el propósito de los atributos name y id en un formulario?

  1. Identificar el campo para procesarlo en el servidor.
  2. Establecer estilos CSS específicos.
  3. Crear enlaces internos dentro del formulario.

Checklist de Formularios

  • Usa etiquetas (<label>) para mejorar la accesibilidad.
  • Valida los datos en el cliente y en el servidor.
  • Evita mostrar contraseñas en texto plano.
  • Proporciona mensajes de error claros y específicos.

Resumen

  • Los formularios son herramientas esenciales para la interacción con los usuarios en aplicaciones web.
  • Se pueden crear utilizando y, y deben ser seguros y accesibles.

Normativa Relacionada

  • WCAG (Web Content Accessibility Guidelines): Estándares para hacer contenido web accesible a personas con discapacidades.
  • CAN-SPAM Act: Ley que regula el envío de correos electrónicos comerciales en Estados Unidos.
