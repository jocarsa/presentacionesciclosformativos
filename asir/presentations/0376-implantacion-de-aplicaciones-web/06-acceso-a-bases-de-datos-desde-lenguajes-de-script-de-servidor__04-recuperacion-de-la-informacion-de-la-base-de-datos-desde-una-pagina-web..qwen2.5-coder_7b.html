Portada

Acceso a bases de datos desde lenguajes de «script» de servidor

Objetivo: Aprender cómo recuperar información de una base de datos desde una página Web utilizando lenguajes de script del lado del servidor.

Concepto: Acceso a Bases de Datos

El acceso a bases de datos desde el lado del servidor permite que las aplicaciones web interactúen con la información almacenada en ellas. Esto se realiza mediante lenguajes como SQL (Structured Query Language).

Ejemplo: Conexión a una Base de Datos

const mysql = require('mysql');
const connection = mysql.createConnection({
  host: 'localhost',
  user: 'tu_usuario',
  password: 'tu_contraseña',
  database: 'tu_base_de_datos'
});
connection.connect((err) => {
  if (err) throw err;
  console.log('Conectado a la base de datos');
});

Proceso: Recuperación de Datos

  1. Establecer conexión con la base de datos.
  2. Escribir una consulta SQL para recuperar los datos deseados.
  3. Ejecutar la consulta y procesar el resultado.
  4. Cerrar la conexión con la base de datos.

Buenas Prácticas: Seguridad

  • Usar consultas preparadas para evitar inyecciones SQL.
  • Cifrar las credenciales de la base de datos.
  • Limitar el acceso a la base de datos solo a los usuarios necesarios.

Errores Comunes: Inyecciones SQL

La inyección SQL ocurre cuando un atacante introduce código malicioso en una consulta SQL. Esto puede llevar a la pérdida de datos o incluso al compromiso del sistema.

Caso Práctico: Consulta Preparada

const mysql = require('mysql');
const connection = mysql.createConnection({
  host: 'localhost',
  user: 'tu_usuario',
  password: 'tu_contraseña',
  database: 'tu_base_de_datos'
});
connection.connect();
const usuario = 'usuario';
const consulta = 'SELECT * FROM usuarios WHERE nombre = ?';
connection.query(consulta, [usuario], (err, results) => {
  if (err) throw err;
  console.log(results);
});
connection.end();

Normativa: Protección de Datos Personales (GDPR)

El acceso a bases de datos debe cumplir con las regulaciones de protección de datos personales, como el GDPR en Europa.

Caso Práctico: Cumplimiento GDPR

Antes de acceder a una base de datos que contiene información personal, es necesario verificar si la empresa está cumpliendo con las regulaciones GDPR y obtener el consentimiento del usuario.
