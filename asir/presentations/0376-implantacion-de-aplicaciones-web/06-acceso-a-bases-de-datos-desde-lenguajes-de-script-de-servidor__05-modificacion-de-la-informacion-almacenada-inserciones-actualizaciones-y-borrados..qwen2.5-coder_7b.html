Portada

Acceso a bases de datos desde lenguajes de «script» de servidor

Objetivo: Comprender cómo modificar la información almacenada en bases de datos utilizando lenguajes de script del lado del servidor.

Modificación de la información almacenada

  • Inseriones (INSERT)
  • Actualizaciones (UPDATE)
  • Borrados (DELETE)

Inseriones (INSERT)

Permite agregar nuevos registros a una tabla.

INSERT INTO nombre_tabla (columna1, columna2) VALUES (valor1, valor2);

Actualizaciones (UPDATE)

Modifica los valores de uno o varios registros en una tabla.

UPDATE nombre_tabla SET columna1 = nuevoValor WHERE condicion;

Borrados (DELETE)

Elimina registros de una tabla que cumplen con ciertas condiciones.

DELETE FROM nombre_tabla WHERE condicion;

Ejemplo práctico

Añadir un nuevo usuario a la base de datos:

INSERT INTO usuarios (nombre, correo) VALUES ('Juan', 'juan@example.com');

Caso real

Una aplicación web que permite a los usuarios actualizar su perfil.

  • El usuario ingresa sus nuevos datos
  • Los datos se envían al servidor
  • El servidor ejecuta una consulta UPDATE en la base de datos

Proceso de inserción

  1. Conectar a la base de datos
  2. Escribir la sentencia INSERT
  3. Ejecutar la consulta
  4. Cerrar la conexión

Buenas prácticas

  • Validar los datos antes de insertarlos
  • Usar consultas preparadas para evitar inyecciones SQL
  • Manejar errores y excepciones adecuadamente

Errores comunes

  • No cerrar la conexión a la base de datos
  • Inyecciones SQL no controladas
  • No validar los datos de entrada

Comparativa

Lenguaje Sentencia
$stmt = $pdo->prepare("INSERT INTO usuarios (nombre, correo) VALUES (:nombre, :correo)");
$stmt->execute([':nombre' => 'Juan', ':correo' => 'juan@example.com']);

session.commit()

Resumen

  • Inseriones, actualizaciones y borrados son operaciones fundamentales en bases de datos.
  • Se utilizan sentencias SQL como INSERT, UPDATE y DELETE.
  • Es crucial seguir buenas prácticas para evitar errores y mantener la seguridad.

Normativa general

Seguridad de datos: Cumplir con las leyes de protección de datos (GDPR, CCPA).

Privacidad: No almacenar información sensible sin consentimiento del usuario.
