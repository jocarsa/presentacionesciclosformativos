Integración de los lenguajes de «script» de servidor con los sistemas gestores de base de datos

Objetivo: Comprender cómo integrar lenguajes como, o Node.js con bases de datos SQL y NoSQL.

Concepto básico

La integración de lenguajes de script de servidor con sistemas gestores de base de datos permite:

  • Almacenar y recuperar datos dinámicamente.
  • Ejecutar consultas y operaciones en la base de datos desde el lado del servidor.
  • Generar contenido personalizado para los usuarios basado en los datos almacenados.

  • Node.js (Para aplicaciones web escalables)

Sistemas gestores de base de datos

Los sistemas gestores de base de datos más utilizados son:

  • SQL (MySQL, PostgreSQL, SQLite)
  • NoSQL (MongoDB, Cassandra, Redis)

Ejemplo práctico: Conexión con MySQL

Código básico para conectarse a una base de datos MySQL usando:

<?
$servername = "localhost";
$username = "username";
$password = "password";
$dbname = "myDB";

// Crear conexión
$conn = new mysqli($servername, $username, $password, $dbname);

// Verificar conexión
if ($conn->connect_error) {
    die("Conexión fallida: " . $conn->connect_error);
}
echo "Conexión exitosa";
?>

Ejemplo práctico: Consulta SQL en

Código para ejecutar una consulta SQL y mostrar los resultados:

<?
$sql = "SELECT id, name, email FROM users";
$result = $conn->query($sql);

if ($result->num_rows > 0) {
    // Salida de datos de cada fila
    while($row = $result->fetch_assoc()) {
        echo "id: " . $row["id"]. " - Name: " . $row["name"]. " - Email: " . $row["email"]. "
";
    }
} else {
    echo "0 resultados";
}
$conn->close();
?>

Conexión Node.js con MongoDB

Código básico para conectarse a una base de datos MongoDB usando Node.js:

const MongoClient = require('mongodb').MongoClient;

async function main() {
    const uri = "your_mongodb_connection_string";
    const client = new MongoClient(uri);

    try {
        await client.connect();
        console.log("Conexión exitosa");
    } finally {
        await client.close();
    }
}

main().catch(console.error);
?>

Manejo de errores en la integración

Algunos errores comunes son:

  • Conexiones fallidas a la base de datos.
  • Sintaxis incorrecta en las consultas SQL o NoSQL.
  • Inyección SQL (SQL Injection).

Buenas prácticas

Para una integración segura y eficiente:

  • Usar consultas preparadas para evitar inyecciones SQL.
  • Cerrar conexiones a la base de datos después de su uso.
  • Validar y sanitizar los datos ingresados por el usuario.
