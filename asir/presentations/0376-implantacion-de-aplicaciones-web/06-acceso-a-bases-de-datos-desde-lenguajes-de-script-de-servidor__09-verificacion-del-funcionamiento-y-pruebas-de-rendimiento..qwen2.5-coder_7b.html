Portada

Verificación del funcionamiento y pruebas de rendimiento

Objetivo: Aprender técnicas para verificar el correcto funcionamiento y optimizar el rendimiento de aplicaciones web que interactúan con bases de datos.

Verificación del Funcionamiento

  • Pruebas unitarias: Verificar la funcionalidad individual de los componentes.
  • Pruebas de integración: Comprobar que los diferentes componentes funcionan juntos correctamente.
  • Pruebas de sistema: Evaluar el sistema completo en un entorno real.

Ejemplo de Prueba Unitaria

Código:


        function getUser($id) {
            $db = new PDO('mysql:host=localhost;dbname=test', 'user', 'password');
            $stmt = $db->prepare("SELECT * FROM users WHERE id = :id");
            $stmt->execute(['id' => $id]);
            return $stmt->fetch();
        }

Pruebas de Rendimiento

  • Medición del tiempo de respuesta.
  • Evaluación del uso de recursos (CPU, memoria).
  • Simulación de carga para identificar puntos críticos.

Caso Práctico: Optimización de Consultas SQL

Antes:


        SELECT * FROM users WHERE age > 18;

Después:


        SELECT id, name FROM users WHERE age > 18;

Buenas Prácticas para Rendimiento

  • Indexación de tablas.
  • Caching de resultados.
  • Optimización de consultas SQL.

Errores Comunes en Pruebas de Rendimiento

  • No simular el entorno real.
  • No considerar la carga de datos.
  • No analizar los resultados adecuadamente.

Comparativa: Pruebas Manuales vs. Automáticas

Pruebas Manuales Pruebas Automáticas
Eficiencia Poco eficiente Muy eficiente
Consistencia Poca consistencia Buena consistencia

Mini-Quiz: Verificación del Funcionamiento

¿Qué tipo de prueba se utiliza para verificar que los componentes individuales funcionan correctamente?

  1. Pruebas unitarias
  2. Pruebas de integración
  3. Pruebas de sistema

Checklist: Pruebas de Rendimiento

  • Mide el tiempo de respuesta.
  • Evalúa el uso de recursos.
  • Simula la carga del sistema.
  • Análisis detallado de los resultados.

Actividad: Optimización de Consultas SQL

Resumen: Verificación del Funcionamiento y Pruebas de Rendimiento

  • Pruebas unitarias, integración y sistema.
  • Evaluación del tiempo de respuesta y uso de recursos.
  • Buenas prácticas: indexación, caching, optimización SQL.
