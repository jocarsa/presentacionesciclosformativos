Funciones integradas y de usuario en aplicaciones web

Objetivo: Comprender y utilizar funciones integradas y personalizadas en lenguajes de script servidor para mejorar la funcionalidad de las aplicaciones web.

Funciones Integradas

  • Definición: Funciones predefinidas disponibles en el lenguaje de script del servidor.
  • Ejemplo: En, la función `date()` devuelve la fecha y hora actual.
  • Ventajas: Facilitan el desarrollo, ahorrando tiempo y reduciendo errores.

Funciones de Usuario

  • Definición: Funciones creadas por el programador para realizar tareas específicas.
  • Ejemplo: Una función en que valida un formulario antes de enviarlo.
  • Ventajas: Permiten la reutilización del código y su mantenimiento facilitado.

Creación de Funciones en

<?
function calcularImpuesto($monto) {
    return $monto * 0.16;
}
$impuesto = calcularImpuesto(100);
echo "El impuesto es: $" . $impuesto;
?>

Parámetros en Funciones

  • Definición: Valores que se pasan a una función para realizar operaciones.
  • Ejemplo: En la función anterior, `$monto` es un parámetro.
  • Ventajas: Permiten flexibilidad y reutilización de funciones.

Retorno de Valores

  • Definición: Valor que una función devuelve al finalizar su ejecución.
  • Ejemplo: La función `calcularImpuesto` retorna el valor del impuesto calculado.
  • Ventajas: Facilita la obtención de resultados y su uso en otras partes del código.

Funciones Anónimas (losures)

<?
$suma = function($a, $b) {
    return $a + $b;
};
echo "La suma es: " . $suma(5, 3);
?>

Checklist de Buenas Prácticas

  • Documentar las funciones para facilitar su uso y mantenimiento.
  • Validar los parámetros de entrada para evitar errores.
  • Usar nombres descriptivos para las funciones y sus parámetros.

Ejemplo de Caso: Sistema de Reservas

Se requiere una función que calcule el costo total de una reserva en un hotel, considerando la cantidad de noches y el precio por noche.

<?
function calcularCostoReserva($noches, $precioNoche) {
    return $noches * $precioNoche;
}
$costo = calcularCostoReserva(3, 100);
echo "El costo total es: $" . $costo;
?>

Errores Comunes

  • No validar los parámetros de entrada.
  • Olvidar el uso del return en una función que debe devolver un valor.
  • Nombres confusos o ambiguos para las funciones y sus parámetros.

Ejemplo de Errores Comunes

<?
function calcularCosto($a, $b) {
    return $a * $b;
}
$costo = calcularCosto(3); // Error: Falta el segundo parámetro
echo "El costo es: $" . $costo;
?>

Resumen

Funciones integradas y de usuario son herramientas esenciales en el desarrollo de aplicaciones web. Permiten la creación de código reutilizable, más eficiente y fácil de mantener.

Normativa General

  • Código limpio: Mantener el código bien organizado y legible.
  • Seguridad: Validar entradas y evitar inyecciones de código.
  • Mantenibilidad: Documentar funciones y seguir buenas prácticas.
