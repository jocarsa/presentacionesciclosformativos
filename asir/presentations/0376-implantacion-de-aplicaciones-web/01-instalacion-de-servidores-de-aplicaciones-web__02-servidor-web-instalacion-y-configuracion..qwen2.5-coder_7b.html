Instalación y Configuración de Servidores Web

Objetivo: Aprender los pasos para instalar y configurar servidores web como Apache, Nginx y IIS.

¿Qué es un Servidor Web?

Un servidor web es software que sirve páginas web a través de Internet o una red local.

Tipos de Servidores Web

  • Apache: Popular y gratuito.
  • Nginx: Eficiente y ligero.
  • IIS (Internet Information Services): Para Windows Server.

Requisitos Previos

  • Sistema operativo compatible (Windows, Linux).
  • Acceso administrador al servidor.
  • Conocimientos básicos de red y seguridad.

Instalación de Apache en Ubuntu

Paso 1: Actualizar el sistema
Paso 2: Instalar Apache con `sudo apt update && sudo apt install apache2`
Paso 3: Verificar la instalación con `sudo systemctl status apache2`

Instalación de Nginx en Ubuntu

Paso 1: Actualizar el sistema
Paso 2: Instalar Nginx con `sudo apt update && sudo apt install nginx`
Paso 3: Verificar la instalación con `sudo systemctl status nginx`

Instalación de IIS en Windows Server

Paso 1: Instalar el rol Web Server (IIS)
Paso 2: Configurar sitios web y aplicaciones
Paso 3: Verificar la instalación desde un navegador

Configuración Básica de Apache

  • Edición del archivo `httpd.conf` o `apache2.conf`.
  • Configuración de directorios y permisos.
  • Habilitación de módulos necesarios con `a2enmod`.

Configuración Básica de Nginx

  • Edición del archivo `nginx.conf`.
  • Configuración de sitios web en directorios separados.
  • Habilitación de módulos con `sudo apt install nginx-module-`.

Configuración Básica de IIS

  • Abrir el Administrador de Servidores Web.
  • Configurar sitios web y aplicaciones.
  • Habilitar protocolos y módulos necesarios.

Pruebas Iniciales

Acceder a la IP del servidor desde un navegador para verificar que el servidor web esté funcionando correctamente.

Buenas Prácticas de Seguridad

  • Actualizaciones regulares.
  • Cambiar contraseñas predeterminadas.
  • Habilitar HTTPS.

Errores Comunes

  • No se encuentra el archivo (404).
  • No se tiene permiso para acceder (403).
  • Servidor no responde.

Comprobación de Logs

Verificar los archivos de log del servidor web (`/var/log/apache2/error.log` o `/var/log/nginx/error.log`) para diagnóstico.
