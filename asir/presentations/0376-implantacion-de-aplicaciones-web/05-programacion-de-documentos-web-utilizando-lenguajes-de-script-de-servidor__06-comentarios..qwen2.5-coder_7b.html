Comentarios en aplicaciones web

Objetivo: Aprender a utilizar comentarios en documentos Web para facilitar la lectura y mantenimiento del código.

¿Qué son los comentarios?

Los comentarios son texto que se incluye en el código fuente de una aplicación web pero no es interpretado por el navegador. Sirven para explicar o desactivar partes del código.

Tipo de comentarios

  • Comentarios de línea: Comienzan con `//` y se extienden hasta el final de la línea.
  • Comentarios multilínea: Se encierran entre `/*` y `*/`, pueden ocupar varias líneas.

Ejemplo de comentario de línea


        // Este es un comentario de una sola línea
        console.log("Hola, mundo!");

Ejemplo de comentario multilínea


        /*
            Este es un comentario
            que ocupa varias líneas.
        */
        var nombre = "Juan";
        console.log(nombre);

Beneficios de usar comentarios

  • Mejora la legibilidad del código.
  • Ayuda a otros desarrolladores a entender el código.
  • Facilita el mantenimiento y actualización del código.

Cómo escribir buenos comentarios

  • Explica lo que hace el código, no cómo lo hace.
  • Evita comentarios redundantes o obsoletos.
  • Mantén los comentarios actualizados con el código.

Ejemplo de mal comentario


        // Incrementa el valor de i en 1
        i++;

Ejemplo de buen comentario


        // Incrementa el contador de visitas en 1
        contadorVisitas++;

Comentarios en diferentes lenguajes

  • : `//` o `/* */`
  • CSS: `/* */`
  • : ``

Comentarios en


        
        
Hola, mundo!

Desactivar bloques de código con comentarios


        /*
            Descomentar el siguiente bloque para probar
            console.log("Este es un mensaje oculto");
        */

Comentarios en CSS


        /* Este es un comentario de CSS */
        body {
            background-color: #f0f0f0;
        }

Comentarios en


        // Este es un comentario de
        var mensaje = "Hola, mundo!";
        console.log(mensaje);

Buenas prácticas para comentarios

  • Comentar el por qué, no el cómo.
  • Evitar comentarios triviales.
  • Mantener los comentarios actualizados.

Errores comunes en comentarios

  • Comentarios obsoletos o redundantes.
  • Comentarios mal escritos que no aportan valor.
  • Comentar código innecesariamente.

Ejemplo de error común en comentarios


        // Incrementa el contador
        contador++;

Mini-quiz: Comentarios

¿Cuál es la diferencia entre un comentario de línea y un comentario multilínea?

  1. Un comentario de línea comienza con `//` y se extiende hasta el final de la línea.
  2. Un comentario multilínea se encierran entre `/*` y `*/`, pueden ocupar varias líneas.

Resumen

  • Los comentarios son texto no interpretado por el navegador.
  • Sirven para facilitar la lectura y mantenimiento del código.
  • Existen diferentes tipos de comentarios: línea, multilínea y en.
  • Buenos comentarios explican lo que hace el código, no cómo lo hace.

Normativa general sobre comentarios

Seguir buenas prácticas de codificación y mantener los comentarios actualizados es crucial para mantener un proyecto de software sostenible y fácil de mantener.
