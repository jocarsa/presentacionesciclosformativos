Gestión de errores en la implantación de aplicaciones web

Objetivo: Aprender a gestionar eficazmente los errores que pueden surgir durante el acceso a bases de datos desde lenguajes de script de servidor.

Definición de gestión de errores

La gestión de errores es la capacidad de identificar, diagnosticar y resolver problemas que ocurren durante el desarrollo o ejecución de una aplicación web.

Tipos comunes de errores en aplicaciones web

  • Error 404: Página no encontrada
  • Error 500: Error interno del servidor
  • Error de conexión a la base de datos
  • Excepciones en el código
  • Errores de validación de datos

Estrategias para gestionar errores

  • Implementar logs detallados
  • Usar mensajes de error amigables al usuario
  • Validar entradas del usuario
  • Capturar y manejar excepciones
  • Realizar pruebas exhaustivas

Ejemplo de gestión de errores en


<?
try {
    $conn = new PDO("mysql:host=$servername;dbname=$dbname", $username, $password);
    $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    echo "Error: " . $e->getMessage();
}
?>

Caso práctico: Manejo de errores en una aplicación web

Desarrolla una función que consulte datos de una base de datos y maneje posibles errores como la conexión fallida o el no encontrado del registro.

Buenas prácticas en gestión de errores

  • Evitar mostrar información sensible en los mensajes de error
  • Usar niveles de errores adecuados (debug, info, warning, error)
  • Documentar todos los errores y soluciones implementadas
  • Realizar pruebas de recuperación ante fallos

Errores comunes en la gestión de errores

  • Mostrar mensajes de error detallados al usuario
  • No capturar excepciones
  • No realizar pruebas exhaustivas
  • No documentar los errores

  • a) Mejora la seguridad del sistema

Checklist: Pasos para gestionar errores en aplicaciones web

  2. Implementar logs detallados
  3. Mostrar mensajes de error amigables al usuario
  4. Validar entradas del usuario
  5. Realizar pruebas exhaustivas

Actividad: Implementación de gestión de errores en una aplicación web

Desarrolla una función que consulte datos de una base de datos y maneje posibles errores como la conexión fallida o el no encontrado del registro. Incluye logs detallados y mensajes amigables al usuario.

Resumen: Gestión de errores en aplicaciones web

  • La gestión de errores es crucial para la calidad y la seguridad de las aplicaciones web
  • Se deben implementar estrategias adecuadas para identificar, diagnosticar y resolver problemas
  • Es importante seguir buenas prácticas y evitar errores comunes en el proceso

Normativa general sobre gestión de errores

Seguir las normativas de seguridad informática y los estándares de desarrollo es fundamental para la gestión eficaz de errores en aplicaciones web.
