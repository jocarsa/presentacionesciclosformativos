Concepto básico

El acceso a bases de datos desde scripts del lado del servidor permite interactuar con la información almacenada en una base de datos para mostrarla o modificarla dinámicamente en una página web.

Métodos comunes de conexión

  • APIs específicas del lenguaje (: PDO, MySQLi;: SQLAlchemy).
  • Bibliotecas generales (Node.js: Sequelize, Mongoose).
  • Frameworks ORM (Object-Relational Mapping).

Ejemplo práctico con y PDO


<?
$host = 'localhost';
$dbname = 'mi_base_de_datos';
$user = 'usuario';
$password = 'contraseña';

try {
    $pdo = new PDO("mysql:host=$host;dbname=$dbname", $user, $password);
    echo "Conexión exitosa";
} catch (PDOException $e) {
    echo "Error de conexión: " . $e->getMessage();
}
?>

Ejemplo práctico con y SQLAlchemy


from sqlalchemy import create_engine

engine = create_engine('mysql+pymysql://usuario:contraseña@localhost/mi_base_de_datos')
connection = engine.connect()
result = connection.execute("SELECT * FROM mi_tabla")
for row in result:
    print(row)

Seguridad en la conexión a bases de datos

  • Uso de sentencias preparadas (prepared statements).
  • Evitar inyecciones SQL.
  • Cifrado de credenciales.

Buenas prácticas

  • Mantener la conexión a la base de datos abierta lo mínimo posible.
  • Utilizar transacciones para operaciones complejas.
  • Realizar copias de seguridad periódicas.

Errores comunes y soluciones

  • Conexión rechazada: Verificar credenciales, permisos y configuración del servidor.
  • Inyecciones SQL: Usar sentencias preparadas y validar entradas.
  • Desconexiones inesperadas: Implementar mecanismos de reconexión.

Proceso de autenticación

  1. Recoger los datos del formulario de inicio de sesión.
  2. Conectar a la base de datos.
  3. Ejecutar una consulta para verificar las credenciales.
  4. Autenticar al usuario o mostrar un mensaje de error.

Mini-quiz: Conexión a bases de datos

¿Cuál es la ventaja principal de usar sentencias preparadas en lugar de cadenas concatenadas para ejecutar consultas SQL?

  1. Mejora el rendimiento de las consultas.
  2. Eficiencia en el uso de recursos del servidor.
  3. Prevención de inyecciones SQL.
  4. Facilita la actualización de la base de datos.

Checklist: Conexión a bases de datos

  • Verificar las credenciales del usuario.
  • Usar sentencias preparadas para evitar inyecciones SQL.
  • Cerrar la conexión a la base de datos después de usarla.
  • Implementar políticas de seguridad para proteger las credenciales.

Resumen

  • Conexión a bases de datos es fundamental para aplicaciones web dinámicas.
  • Métodos como PDO, SQLAlchemy y ORM facilitan el acceso a la información.
