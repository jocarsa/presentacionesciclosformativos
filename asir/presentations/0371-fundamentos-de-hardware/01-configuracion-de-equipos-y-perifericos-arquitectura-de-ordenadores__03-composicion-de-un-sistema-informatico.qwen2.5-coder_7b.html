Portada

Configuración de equipos y periféricos: Arquitectura de ordenadores

Objetivo: Comprender la composición básica de un sistema informático, desde su unidad central hasta sus periféricos.

La Unidad Central de Proceso (UCP)

  • CPU: Procesador principal que ejecuta instrucciones.
  • Controlador: Coordinador de las operaciones del sistema.
  • Unidad Aritmético-Lógica (ALU): Realizaálculos y comparaciones.

La Memoria

  • RAM: Almacena datos temporales durante la ejecución de programas.
  • ROM: Almacena instrucciones y datos permanentes (BIOS/UEFI).
  • Cache: Memoria rápida para acelerar el acceso a los datos más frecuentemente utilizados.

El Subsistema de E/S

  • Entrada (Input): Dispositivos que proporcionan datos al sistema.
  • Salida (Output): Dispositivos que muestran los resultados del sistema.
  • Interfaz: Componente que permite la comunicación entre el subsistema de E/S y la UCP.

Tipos de Arquitecturas de Bus

  • Serie (Serial): Datos se transmiten uno a uno en un solo canal.
  • Paralelo: Múltiples canales para transmisión simultánea de datos.
  • Multiplexado: Transmisión de múltiples señales en un solo canal.

Interfaces

  • Puerto serie (RS-232): Comunicación simple a corta distancia.
  • Puerto USB: Interfaz universal para dispositivos periféricos.
  • Puerto PCI: Conectividad para tarjetas de red y gráficos.

Ejemplo Práctico

Un ordenador típico consta de una UCP con CPU, controlador y ALU, 8GB de RAM, 1TB de disco duro, teclado, ratón y monitor.

Caso Real

Un servidor web moderno puede tener múltiples CPUs, grandes cantidades de RAM, discos SSD para almacenamiento rápido, y interfaces gigabit Ethernet.

Proceso de Configuración

  1. Instalar la CPU en el socket correspondiente.
  2. Conectar la RAM al sistema mediante los puertos DIMM.
  3. Configurar el BIOS/UEFI para ajustar parámetros como velocidad de reloj y modo de operación.

Buenas Prácticas

  • Usar periféricos compatibles con la arquitectura del sistema.
  • Mantener los cables y conectores limpios y ordenados.
  • Realizar copias de seguridad periódicas de los datos importantes.

Errores Comunes

  • No conectar correctamente los periféricos.
  • No ajustar adecuadamente la velocidad de reloj en el BIOS/UEFI.
  • No tener suficiente RAM para las aplicaciones instaladas.

Comparativa

Tipo de Bus Serie Paralelo Multiplexado
Ventajas Simple, económico. Rápido, eficiente. Espacio en el bus limitado.
Inconvenientes Lenta, limitada por la velocidad del canal. Costoso, complejo de implementar. Puede causar interferencias.

Checklist de Configuración

  • Verificar la compatibilidad del hardware.
  • Ajustar los parámetros en el BIOS/UEFI.
  • Instalar y configurar los periféricos.
  • Realizar pruebas de funcionamiento.

Actividad Práctica

Configurar un nuevo sistema informático con las siguientes especificaciones: CPU Intel i5, 16GB RAM, SSD 500GB, teclado mecánico y monitor Full HD.

Resumen

  • Un ordenador consta de la UCP, memoria y subsistema de E/S.
  • Existen diferentes tipos de arquitecturas de bus para la comunicación entre componentes.
  • Es importante seguir buenas prácticas y evitar errores comunes durante la configuración del sistema.

Normativa General

La normativa vigente establece requisitos mínimos para el funcionamiento seguro y eficiente de los sistemas informáticos.
