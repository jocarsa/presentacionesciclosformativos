Objetivo: Comprender las diferentes arquitecturas de hardware utilizadas en CPD.

Arquitectura de ordenadores personales (PC)

  • CPU: Procesador principal que ejecuta instrucciones.
  • Memoria RAM: Almacena datos temporales para el procesamiento.
  • Almacenamiento secundario: Disco duro o SSD.
  • Pantalla y teclado: Interfaz de usuario.
  • Entradas/salidas (USB, HDMI, etc.): Conexiones periféricas.

Sistemas departamentales

Arquitecturas diseñadas para el uso en entornos corporativos.

  • Servidores dedicados: Para aplicaciones específicas.
  • Racks de servidores: Agrupación de servidores físicos.
  • Cabinas de datos: Espacios controlados para el almacenamiento y procesamiento de datos.

Grandes ordenadores (Supercomputadoras)

Sistemas altamente escalables con múltiples procesadores.

  • Nodos: Unidades individuales que forman parte del sistema.
  • Interconexiones: Redes de alta velocidad entre nodos.
  • Almacenamiento distribuido: Sistema de almacenamiento compartido.
  • Potenciales aplicaciones: Simulación científica, análisis de datos.

Caso práctico: Sistema departamental

Un hospital necesita un sistema para gestionar citas médicas y registros de pacientes.

  • Servidores web para la interfaz de usuario.
  • Bases de datos SQL para almacenar información.
  • Red local para la comunicación interna.

Proceso de implantación en CPD

  1. Evaluación de requisitos.
  2. Diseño del sistema.
  3. Selección de hardware y software.
  4. Instalación y configuración.
  5. Pruebas y validación.
  6. Mantenimiento y actualizaciones.

Buenas prácticas en CPD

  • Seguridad de la información.
  • Copias de seguridad periódicas.
  • Monitoreo y alertas.
  • Capacitación del personal técnico.

Errores comunes en la implantación de hardware

  • No considerar el rendimiento necesario.
  • Sobrecalentamiento debido a mala ventilación.
  • Falta de respaldo y copias de seguridad.
  • Desconexiones eléctricas inesperadas.

Comparativa: PC vs. Sistema departamental vs. Supercomputadora

Tipo de sistema CPU Memoria RAM Almacenamiento
PC 1-4 núcleos 8-32 GB 500 GB - 1 TB
Sistema departamental 4-64 núcleos 16-128 GB 10 TB - 1 PB
Supercomputadora Miles de núcleos Multiples TB PB+

Checklist: Consideraciones al seleccionar hardware

  1. Requisitos funcionales y no funcionales.
  2. Rendimiento necesario.
  3. Seguridad y cumplimiento normativo.
  4. Budget y presupuesto.
  5. Mantenimiento y escalabilidad.
