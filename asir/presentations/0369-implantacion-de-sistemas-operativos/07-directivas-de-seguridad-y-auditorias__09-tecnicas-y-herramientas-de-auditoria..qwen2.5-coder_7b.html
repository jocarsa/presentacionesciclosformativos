Portada

Técnicas y herramientas de auditoría en sistemas operativos

Objetivo: Comprender las técnicas y herramientas utilizadas para auditar sistemas operativos.

Definición de Auditoría

Auditoría en sistemas operativos es el proceso de examen sistemático y evaluación de los controles de seguridad implementados.

Importancia de la Auditoría

  • Detección de vulnerabilidades y riesgos
  • Mejora continua del sistema de seguridad
  • Cumplimiento con regulaciones y normativas
  • Prevención de incidentesibernéticos

Herramientas de Auditoría

  • Nmap (Network Mapper)
  • Wireshark (Protocol Analyzer)
  • OpenVAS (Vulnerability Assessment System)
  • Metasploit Framework
  • Sysinternals Suite

Nmap: Herramienta de Escaneo de Red

Nmap es una herramienta de código abierto que escanea y examina redes para identificar dispositivos activos y servicios.

Wireshark: Análisis de Protocolos

Wireshark captura y analiza paquetes de red, lo que permite entender cómo se comunican los sistemas.

OpenVAS: Evaluación de Vulnerabilidades

OpenVAS es un sistema completo para la detección de vulnerabilidades en sistemas y redes.

Metasploit Framework: Exploits y Pruebas Penetrativas

Metasploit es una plataforma de código abierto que permite crear, ejecutar y gestionar exploits para pruebas penetrativas.

Sysinternals Suite: Herramientas del Sistema Operativo

  • Process Explorer (PE)
  • File Monitor (FM)
  • Registry Editor (RegEdit)
  • DiskMon (DM)
  • Handle (H)

Ejemplo de Auditoría con Nmap

Comando: nmap -sV 192.168.1.0/24

Descripción: Escanea la red para identificar dispositivos activos y sus versiones de software.

Ejemplo de Auditoría con Wireshark

Proceso: Captura tráfico en una interfaz, filtra por protocolo específico (ej. HTTP), analiza paquetes.

Caso Práctico: Auditoría de Seguridad en Windows

Proceso: Uso de Sysinternals Suite para monitorear procesos, archivos y registros del sistema.

Buenas Prácticas en Auditoría

  • Realizar auditorías periódicas
  • Documentar hallazgos y recomendaciones
  • Aplicar parches y actualizaciones
  • Entrenamiento continuo de personal técnico

Errores Comunes en Auditoría

  • No realizar auditorías periódicas
  • No documentar hallazgos y recomendaciones
  • No aplicar parches y actualizaciones
  • Falta de formación continua del personal técnico

Resumen

  • Herramientas como Nmap, Wireshark, OpenVAS, Metasploit y Sysinternals Suite son útiles
  • Realizar auditorías periódicas y documentar hallazgos es fundamental
