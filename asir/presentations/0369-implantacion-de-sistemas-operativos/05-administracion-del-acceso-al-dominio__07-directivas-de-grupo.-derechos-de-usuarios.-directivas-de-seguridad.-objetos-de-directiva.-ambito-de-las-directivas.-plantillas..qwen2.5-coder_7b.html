Portada

Administración del acceso al dominio

Objetivo: Comprender las directivas de grupo, derechos de usuarios y objetos de directiva en sistemas operativos.

Directivas de Grupo

  • Son configuraciones que afectan a un conjunto de usuarios o computadoras.
  • Permiten la gestión masiva de políticas y preferencias.
  • Ejemplos: políticas de seguridad, configuración de aplicaciones.

Derechos de Usuarios

  • Definen las acciones que un usuario puede realizar en el sistema.
  • Incluyen permisos de lectura, escritura y ejecución.
  • Ejemplos: acceso a archivos, uso de aplicaciones.

Directivas de Seguridad

  • Establecen políticas para proteger el sistema y los datos.
  • Incluyen contraseñas, autenticación, cifrado.
  • Ejemplos: políticas de contraseñas fuertes, autenticación multifactor.

Objetos de Directiva

  • Son los elementos que pueden ser configurados mediante directivas.
  • Incluyen usuarios, grupos, computadoras, aplicaciones.
  • Ejemplos: usuarios individuales, grupos de trabajo, políticas específicas.

Ámbito de las Directivas

  • Determina la aplicabilidad de una directiva.
  • Puede ser global (aplica a todo el dominio) o local (solo a ciertos objetos).
  • Ejemplos: directivas globales para todos los usuarios, directivas locales para grupos específicos.

Plantillas

  • Son configuraciones predefinidas que pueden ser aplicadas a objetos.
  • Aceleran la gestión y consistencia de políticas.
  • Ejemplos: plantilla para usuarios nuevos, plantilla para computadoras de oficina.

Configuración de Directivas

Paso 1: Identificar el objeto a configurar (usuario, grupo, etc.).

Paso 2: Seleccionar la directiva correspondiente.

Paso 3: Configurar los parámetros según las necesidades.

Ejemplo de Directiva

  • Directiva: Políticas de contraseñas fuertes.
  • Configuración: Longitud mínima, caracteres especiales requeridos.
  • Efecto: Mejora la seguridad del sistema al forzar contraseñas más seguras.

Caso Práctico: Implementación de Directivas

Escenario: Configurar políticas de acceso a aplicaciones para un departamento específico.

Pasos: Crear grupo, definir directiva de acceso, asignar al grupo.

Buenas Prácticas

  • Aplicar directivas a grupos en lugar de usuarios individuales.
  • Usar plantillas para mantener consistencia.
  • Monitorear y auditar cambios en las directivas.

Errores Comunes

  • No aplicar directivas a grupos, sino a usuarios individuales.
  • No usar plantillas, lo que lleva a inconsistencias.
  • No monitorear cambios en las directivas, lo que puede dejar vulnerables el sistema.

Comparativa: Directivas vs. Permisos Locales

DirectivasPermisos Locales
AplicabilidadMasiva (grupos)Específica (objetos individuales)
ConsistenciaBaja (varios grupos diferentes)Alta (todos los objetos iguales)
MantenimientoAlta (una vez configuradas)Baja (ada objeto individualmente)

  • b) Una configuración que afecta a un conjunto de usuarios o computadoras.
  • c) Una aplicación específica del sistema operativo.

Checklist: Configuración de Directivas

  1. Identificar el objeto a configurar.
  2. Seleccionar la directiva correspondiente.
  3. Configurar los parámetros según las necesidades.
  4. Aplazar la directiva al objeto seleccionado.

Actividad: Creación de Directivas

Tarea: Crear una directiva para limitar el acceso a aplicaciones no autorizadas en un departamento específico.

Pasos: Identificar grupo, definir directiva de acceso, asignar al grupo.

Resumen

  • Directivas de grupo: configuraciones para grupos de usuarios.
  • Derechos de usuarios: permisos individuales.
  • Directivas de seguridad: políticas de protección.
  • Objetos de directiva: usuarios, grupos, computadoras.

Normativa General

Norma: NIST SP 800-53 (Seguridad en sistemas operativos).

Recomendación: Implementar directivas según las mejores prácticas estableidas por organizaciones de seguridad.
