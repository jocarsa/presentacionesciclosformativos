Portada

Requisitos de seguridad del sistema y de los datos

Objetivo: Comprender los requisitos esenciales para la seguridad tanto del sistema como de los datos.

Definición de Requisitos de Seguridad

  • Conjunto de reglas y especificaciones que deben cumplir un sistema o dato para garantizar su seguridad.
  • Incluye aspectos técnicos, operativos y administrativos.
  • Establecen los límites mínimos de protección contra amenazas potenciales.

Categorías de Requisitos de Seguridad

  • Requisitos de seguridad del sistema: Protección física, hardware y software.
  • Requisitos de seguridad de los datos: Confidencialidad, integridad y disponibilidad.

Requisitos Específicos del Sistema

  • Seguridad física: Protección contra intrusiones físicas.
  • Seguridad de hardware: Resiliencia ante fallos y ataques.
  • Seguridad de software: Protección contra malware y vulnerabilidades.

Requisitos Específicos de los Datos

  • Confidencialidad: Proteger la información sensible de acceso no autorizado.
  • Integridad: Garantizar que los datos no sean alterados o falsificados.
  • Disponibilidad: Mantener el acceso a los datos en tiempos adecuados.

Ejemplo de Requisitos de Seguridad

Sistema bancario:

  • Seguridad física:ajas fuertes, cámaras seguras.
  • Seguridad de hardware: Sistemas antirrobo, monitoreo constante.
  • Confidencialidad: Encriptación de datos en reposo y transmisión.
  • Integridad: Controles de acceso basados en roles.
  • Disponibilidad: Redes redundantes, sistemas de respaldo.

Caso Práctico: Implementación de Seguridad en un Servidor Web

Objetivo: Configurar SSL/TLS para proteger las comunicaciones entre el cliente y el servidor.

Buenas Prácticas en Requisitos de Seguridad

  • Especificar requisitos claros y específicos.
  • Realizar análisis de riesgos para identificar amenazas potenciales.
  • Mantener actualizados los sistemas y software.
  • Implementar políticas de acceso controlados.

Errores Comunes en Requisitos de Seguridad

  • No considerar amenazas externas.
  • Sobrecargar los sistemas con demasiada seguridad.
  • No actualizar regularmente las políticas y procedimientos.
  • Ignorar la confidencialidad de datos sensibles.

Comparativa: Requisitos Internos vs. Externos

Internos (Organización) Externos (Proveedores)
Control Más control sobre la seguridad interna. Menor control, depende del proveedor.
Costo Costeos internos vs. costos de contratación. Puede ser más económico en el largo plazo.
Flexibilidad Más flexibilidad para adaptarse a las necesidades cambiantes. Menor flexibilidad, depende de la capacidad del proveedor.

  • Facilitar el acceso a los usuarios.

Checklist de Requisitos de Seguridad

  1. Especificar requisitos claros y específicos.
  2. Realizar análisis de riesgos.
  3. Mantener actualizados los sistemas y software.
  4. Implementar políticas de acceso controlados.
  5. Realizar pruebas de penetración periódicas.

Resumen

  • Requisitos de seguridad del sistema y de los datos son fundamentales para la protección contra amenazas.
  • Incluyen aspectos técnicos, operativos y administrativos.
  • Categorías principales: seguridad del sistema y seguridad de los datos.
  • Implementar buenas prácticas y evitar errores comunes es crucial para una buena gestión de la seguridad.

Normativa General

Reglamento General de Protección de Datos (RGPD):

  • Aplica a todas las empresas que manejen datos personales de residentes en la Unión Europea.
  • Establece requisitos mínimos para el tratamiento seguro y protegido de los datos.
  • Impone sanciones significativas por no cumplir con los requisitos.
