Objetivo: Comprender los conceptos y prácticas relacionados con los ficheros de configuración de red.

Concepto de Fichero de Configuración de Red

Un fichero de configuración de red es un archivo que contiene información necesaria para la correcta operación del sistema operativo en una red.

Ejemplos de Ficheros de Configuración de Red

  • /etc/network/interfaces (Debian/Ubuntu)
  • /etc/sysconfig/network-scripts/ifcfg-eth0 (Red Hat/CentOS)
  • C:\Windows\System32\drivers\etc\hosts (Windows)

Estructura de un Fichero de Configuración de Red

Incluye información como:

  • Dirección IP
  • Máscara de subred
  • Puerta de enlace predeterminada
  • Servidores DNS

Proceso de Edición de un Fichero de Configuración de Red

  1. Identificar el fichero correcto para la interfaz deseada.
  2. Modificar las líneas correspondientes con los valores deseados.
  3. Guardar los cambios y reiniciar el servicio de red.

Buenas Prácticas al Editar Ficheros de Configuración de Red

  • Hacer una copia de seguridad del fichero original.
  • Comprobar la sintaxis antes de guardar los cambios.
  • Reiniciar el servicio de red para aplicar los cambios.

Errores Comunes al Editar Ficheros de Configuración de Red

  • Sintaxis incorrecta.
  • Direcciones IP inválidas.
  • No reiniciar el servicio de red.

Actividad: Configuración de una Interfaz de Red

Configura la interfaz eth0 en un sistema Debian/Ubuntu con las siguientes características:

  • Dirección IP: 192.168.1.100
  • Máscara de subred: 255.255.255.0
  • Puerta de enlace predeterminada: 192.168.1.1

Resumen: Ficheros de Configuración de Red

Los ficheros de configuración de red son esenciales para la correcta operación del sistema en una red. Es importante conocer su estructura, cómo editarlos y seguir buenas prácticas para evitar errores.
