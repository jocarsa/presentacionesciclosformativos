Portada

Administración de usuarios y grupos locales

Objetivo: Aprender cómo gestionar usuarios y grupos en sistemas operativos locales.

Concepto de Usuarios y Grupos

  • Usuario:uenta individual con privilegios específicos.
  • Grupo: Colección de usuarios que comparten permisos.

Crea un Usuario

Comando: sudo adduser nombre_usuario

Paso 1: Introduce la contraseña del usuario.

Paso 2: Configura información personal (nombre, teléfono, etc.).

Crea un Grupo

Comando: sudo addgroup nombre_grupo

Añade usuarios a grupos existentes con: sudo usermod -aG nombre_usuario nombre_grupo

Configura Permisos de Archivos y Carpetas

  • Chown: Cambia el propietario del archivo.
  • Chgrp: Cambia el grupo del archivo.
  • Chmod: Cambia los permisos de acceso.

Administración Avanzada

  • sudo -i: Inicia sesión como root.
  • su nombre_usuario: Cambia de usuario.
  • passwd: Cambia la contraseña del usuario actual.

Buenas Prácticas

  • No usar cuentas root para tareas diarias.
  • Utilizar grupos para asignar permisos colectivos.
  • Regularmente revisar y ajustar los privilegios de usuarios.

Errores Comunes

  • No tener suficientes permisos para realizar acciones.
  • Cambiar la contraseña incorrectamente.
  • Añadir usuarios a grupos equivocados.

Mini-Quiz

¿Cuál comando se usa para crear un nuevo usuario?

  1. sudo adduser nombre_usuario
  2. sudo usermod -aG nombre_usuario nombre_grupo
  3. sudo chown nombre_usuario nombre_archivo

Checklist

  • Crea un usuario con la contraseña correcta.
  • Añade el usuario a los grupos necesarios.
  • Verifica los permisos de acceso a archivos y carpetas.

Actividad Práctica

Crea un nuevo usuario, añádelo a un grupo y cambia su contraseña.

Resumen

  • Administración de usuarios y grupos es fundamental para la seguridad del sistema.
  • Utiliza comandos como adduser, addgroup, chown, etc.
  • Mantén prácticas seguras y revisa regularmente los privilegios de usuarios.
