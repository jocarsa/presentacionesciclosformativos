Administración del acceso al dominio: Permisos y derechos

Objetivo: Comprender los conceptos básicos de permisos y derechos en sistemas operativos.

Definición de permisos y derechos

  • Permisos: Conjunto de acciones que un usuario puede realizar sobre un recurso.
  • Derechos: Capacidad o autorización para realizar una acción específica.

Estructura básica de permisos

  • Lectura (Read): Permite ver el contenido del recurso.
  • Escritura (Write): Permite modificar o cambiar el contenido del recurso.
  • Ejecución (Execute): Permite ejecutar un programa o acceder a una carpeta.

Tipos de usuarios en sistemas operativos

  • Administrador: Tiene todos los permisos y puede realizar cualquier acción.
  • Usuario estándar: Tiene permisos limitados según su configuración.
  • Invitado: Tiene acceso limitado, generalmente solo lectura.

Cambio de permisos en sistemas operativos

Comando para cambiar permisos en Linux: chmod

Ejemplo: chmod 755 archivo.txt

Cambio de propietario y grupo

Comando para cambiar propietario en Linux: chown

Ejemplo: chown usuario:grupo archivo.txt

Uso de ACL (Access Control Lists)

  • ACL: Sistema que permite establecer permisos más específicos y detallados.
  • Permite asignar permisos a usuarios individuales o grupos.

Ejemplo de ACL en Linux

setfacl -m u:usuario:rwx archivo.txt

Establece permisos de lectura, escritura y ejecución para el usuario especificado.
