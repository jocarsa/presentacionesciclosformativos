Administración de software de base

Usuarios y grupos predeterminados

Objetivo: Comprender los usuarios y grupos predeterminados en sistemas operativos.

Concepto de Usuarios Predeterminados

  • Los usuarios predeterminados son cuentas que vienen instaladas por defecto en un sistema operativo.
  • Sirven como plantillas para crear nuevas cuentas de usuario personalizadas.
  • Ejemplos comunes: "Administrador", "Invitado", "Usuario".

Concepto de Grupos Predeterminados

  • Los grupos predeterminados son conjuntos de usuarios que comparten permisos y configuraciones similares.
  • Sirven para organizar y administrar a los usuarios de manera eficiente.
  • Ejemplos comunes: "Administradores", "Usuarios", "Invitados".

Roles Predeterminados en Windows

  • Administrador: Tiene acceso completo al sistema.
  • Invitado: No puede realizar cambios significativos.
  • Usuario estándar: Permisos limitados para uso general.

Roles Predeterminados en Linux

  • root: Superusuario con acceso completo al sistema.
  • sudoers: Usuarios autorizados para ejecutar comandos como root.
  • users: Grupo predeterminado para usuarios estándar.

Ejemplo de Creación de Usuario en Windows

  1. Navega a "Configuración" > "uentas" > "Administración de cuentas".
  2. Haz clic en "Añadir una nueva cuenta".
  3. Elige el tipo de cuenta (estándar, familiar, etc.) y sigue las instrucciones.

Ejemplo de Creación de Usuario en Linux

  1. Navega a "Configuración" > "uentas de usuario".
  2. Haz clic en "Añadir un nuevo usuario".
  3. Ingresa la información del usuario y selecciona el grupo principal.

Caso Práctico: Administración de Grupos

Un administrador necesita agregar a un nuevo empleado al grupo "Desarrolladores".

  1. Navega a "Configuración" > "uentas" > "Administración de cuentas".
  2. Haz clic en la cuenta del nuevo empleado.
  3. Selecciona el grupo "Desarrolladores" y guarda los cambios.

Buenas Prácticas: Uso de Grupos

  • Evita usar cuentas individuales para tareas específicas.
  • Asigna permisos a grupos en lugar de cuentas individuales.
  • Mantén los grupos actualizados y desactualizados según las necesidades del negocio.

Errores Comunes: Creación deuentas Incorrectas

  • Crea cuentas con privilegios elevados por error.
  • No asigna correctamente los grupos a las cuentas.
  • No elimina cuentas obsoletas o no utilizadas.

Comparativa: Usuarios vs Grupos

Usuarios Grupos
Definición Cuentas individuales con permisos específicos. Conjuntos de usuarios con permisos compartidos.
Uso Para tareas individuales. Para organizar y gestionar a los usuarios.

Checklist: Administración de Usuarios y Grupos

  1. Crea cuentas individuales solo cuando sea necesario.
  2. Asigna permisos a grupos en lugar de cuentas individuales.
  3. Mantén los grupos actualizados según las necesidades del negocio.
  4. No elimina cuentas obsoletas o no utilizadas.

Resumen

  • Los usuarios y grupos predeterminados son cuentas y conjuntos de usuarios con permisos específicos.
  • Administrar correctamente estos elementos es crucial para la seguridad y eficiencia del sistema operativo.

Normativa General: Seguridad en Administración de Usuarios

Seguir las mejores prácticas y evitar errores comunes puede prevenir problemas de seguridad y cumplir con las normativas vigentes.
