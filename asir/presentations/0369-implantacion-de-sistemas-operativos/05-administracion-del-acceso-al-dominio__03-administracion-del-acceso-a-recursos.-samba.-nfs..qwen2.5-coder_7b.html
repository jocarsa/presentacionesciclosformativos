Objetivo: Comprender los métodos de administración del acceso a recursos en entornos de red, con énfasis en SAMBA y NFS.

Administración del Acceso al Dominio

Definición: Conjunto de procesos que controlan quién puede acceder a qué recursos dentro de una red.

Acceso a Recursos en Redes

  • Control de usuarios y permisos
  • Gestión de compartimentos de archivos
  • Autenticación y autorización

SAMBA: Sistemas de Archivos y Manejo de Bases de Datos para Windows

Software que permite a los sistemas Linux compartir archivos y recursos con Windows.

Funciones principales de SAMBA

  • Autenticación de usuarios
  • Gestión de compartimentos de archivos
  • Compartir impresoras
  • Sincronización de directorios

NFS: Network File System

Protocolo de red que permite a los sistemas compartir archivos y directorios.

Características de NFS

  • Basado en protocolos TCP/IP
  • Soporta transacciones de red asincrónicas
  • Facilita la colaboración entre usuarios y equipos

Configuración básica de SAMBA

Involucra edición del archivo smb.conf y creación de compartimentos.

Ejemplo de configuración Samba

[shared]
   path = /srv/samba/shared
   writable = yes
   browsable = yes
   read only = no

Configuración básica de NFS

Involucra edición del archivo /etc/exports y reinicio del servicio nfs-kernel-server.

Ejemplo de configuración NFS

/srv/nfs/shared *(rw,sync,no_subtree_check)

Buenas Prácticas en SAMBA

  • Usar contraseñas fuertes para usuarios
  • Limitar los permisos de acceso a recursos
  • Realizar copias de seguridad periódicas

Buenas Prácticas en NFS

  • Monitorear el uso del espacio en disco
  • Configurar políticas de acceso controladas
  • Auditar accesos y cambios en los recursos compartidos

Errores Comunes en SAMBA

  • Configuración incorrecta del archivo smb.conf
  • No se crean correctamente los compartimentos de archivos
  • Problemas de autenticación

Errores Comunes en NFS

  • No se montan correctamente los directorios compartidos
  • Problemas de permisos al acceder a recursos
  • Desempeño lento debido a configuraciones incorrectas

Comparativa SAMBA vs. NFS

SAMBA NFS
Compatible con Windows No compatible con Windows (requiere clientes NFS)
Facilidad de uso para usuarios finales Más complejo para usuarios finales

Checklist: Configuración de SAMBA

  • Instalar el paquete Samba
  • Configurar el archivo smb.conf
  • Crear compartimentos de archivos
  • Reiniciar el servicio Samba

Checklist: Configuración de NFS

  • Instalar el paquete NFS
  • Configurar el archivo /etc/exports
  • Reiniciar el servicio nfs-kernel-server
  • Montar los directorios compartidos en clientes

Actividad: Configuración de SAMBA

Configurar un compartimento Samba llamado "proyectos" con permisos de escritura para todos los usuarios.

Actividad: Configuración de NFS

Exportar el directorio "/srv/nfs/proyectos" y montarlo en un cliente Linux.

Resumen

  • SAMBA: Compartir archivos con Windows
  • NFS: Compartir archivos entre sistemas Linux
  • Buenas prácticas y errores comunes en ambos sistemas
