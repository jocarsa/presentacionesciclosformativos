Estructura de directorios de sistemas operativos libres y propietarios

Objetivo: Comprender la estructura de directorios en Sistemas Operativos Libres (SOL) y Propietarios (SO).

Sistemas Operativos Libres (SOL)

  • Linux es el ejemplo más conocido de SOL.
  • Estructura basada en Unix, con un diseño modular y distribuido.
  • Directorio raíz (/) como punto de partida.
  • Uso de permisos y propietarios para controlar el acceso.

Sistemas Operativos Propietarios (SO)

  • Ejemplos: Windows, macOS.
  • Uso de una estructura jerárquica centralizada desde la raíz (:\).
  • Directorios como Program Files, Users, Temp.
  • Control del acceso mediante permisos y políticas de seguridad.

Estructura típica de Linux

  • /bin: Comandos esenciales para el sistema operativo.
  • /boot: Archivos necesarios para iniciar el sistema.
  • /dev: Dispositivos del sistema.
  • /etc: Configuración del sistema.

Estructura típica de Windows

  • C:\Windows: Sistema operativo y archivos esenciales.
  • C:\Program Files: Instalaciones de software.
  • C:\Users: Directorios de los usuarios.
  • C:\Temp: Archivos temporales.

Directorio raíz (/) en Linux

/ es el punto de partida del sistema de archivos, similar a:\ en Windows.

Directorio raíz (:) en Windows

C: es la unidad principal y punto de partida del sistema de archivos.

Permisos en Linux

  • Leer (r), Escribir (w), Ejecutar (x).
  • Uso de números octales para representar permisos.
  • Ejemplo: 755 (rwxr-xr-x)

Permisos en Windows

  • Leer, Escribir, Ejecutar.
  • Uso de cuadros de diálogo para configurar permisos.
  • Ejemplo: Propiedades del archivo -> Seguridad.

Buenas prácticas en Linux

  • Evitar el uso de superusuario (root) para tareas normales.
  • Usar permisos adecuados para archivos y directorios.
  • Mantener la estructura del sistema ordenada.

Buenas prácticas en Windows

  • No ejecutar programas desconocidos desde Internet.
  • Usar permisos adecuados para archivos y carpetas.
  • Mantener el sistema de archivos limpio y organizado.

Errores comunes en Linux

  • No tener permisos suficientes para acceder a un archivo/directorio.
  • Escribir en archivos de solo lectura.
  • Mover archivos importantes al directorio raíz por error.

Errores comunes en Windows

  • No tener permisos suficientes para acceder a un archivo/directorio.
  • Escribir en archivos de solo lectura.
  • Mover archivos importantes al directorio raíz por error.
