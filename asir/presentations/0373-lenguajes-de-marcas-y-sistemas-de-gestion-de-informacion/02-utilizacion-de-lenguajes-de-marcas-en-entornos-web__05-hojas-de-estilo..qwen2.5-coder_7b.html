Utilización de lenguajes de marcas en entornos Web: Hojas de estilo

Objetivo: Comprender los conceptos básicos y aplicaciones de las hojas de estilo en el desarrollo web.

¿Qué son las Hojas de Estilo?

Las hojas de estilo (CSS) son archivos que contienen reglas para definir la apariencia visual de un documento.

Estructura Básica de una Hoja de Estilo

<style>
        selector {
            propiedad: valor;
        }
    </style>

Tipos de Selectores

  • Selectores por etiqueta
  • Selectores por clase
  • Selectores por ID
  • Selectores atributo
  • Selectores descendientes

Propiedades Comunes en CSS

  • color: Define el color del texto.
  • font-size: Define el tamaño de la fuente.
  • background-color: Define el color de fondo.
  • margin: Define los márgenes.
  • padding: Define el relleno interno.

Cascada y Herencia en CSS

Cascada: Las reglas se aplican en orden de especificidad, siendo las más específicas las que tienen mayor prioridad.

Herencia: Algunas propiedades se heredan automáticamente a los elementos secundarios.

Cascada y Herencia Ejemplo

<style>
        body {
            font-size: 16px;
        }
        .container {
            color: blue;
        }
        p {
            margin: 0;
        }
    </style>

Importancia de la Separación de Contenido y Presentación

La separación de contenido () y presentación (CSS) facilita el mantenimiento y la actualización del diseño.

Optimización para SEO con CSS

  • Uso de selectores específicos.
  • Minificación del código CSS.
  • Uso de media queries para diseño responsivo.

Checklist de Buenas Prácticas en CSS

  • Separar contenido y presentación.
  • Usar selectores específicos.
  • Minificar el código CSS.
  • Utilizar media queries para diseño responsivo.

Ejemplo de Código CSS Completo

<style>
        body {
            font-family: Arial, sans-serif;
            margin: 0;
            padding: 0;
        }
        .header {
            background-color: #333;
            color: white;
            padding: 10px 20px;
        }
        .content {
            margin: 20px;
        }
    </style>

Errores Comunes en CSS

  • No separar contenido y presentación.
  • Usar selectores genéricos.
  • No minificar el código CSS.
  • No utilizar media queries para diseño responsivo.

Comparativa: CSS vs. Inline Styles

CSS Inline Styles
Mantiene la separación de contenido y presentación. No mantiene la separación de contenido y presentación.
Puede ser reutilizado en múltiples elementos. Solo se aplica al elemento donde se define.
