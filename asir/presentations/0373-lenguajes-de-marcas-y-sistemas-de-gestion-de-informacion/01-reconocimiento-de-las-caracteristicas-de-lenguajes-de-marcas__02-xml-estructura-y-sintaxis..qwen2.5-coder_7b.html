: Estructura y Sintaxis

Objetivo: Comprender las características básicas de la estructura y sintaxis del lenguaje.

¿Qué es?

(eXtensible Markup Language) es un lenguaje de marcado que permite definir su propio conjunto de etiquetas para representar datos estructurados.

Estructura Básica de

  • Declaración del Prologo:
  • Etiqueta Raíz: Define el punto de entrada del documento.
  • Etiquetas Anidadas: Permiten estructurar los datos en forma jerárquica.
  • Atributos: Proporcionan información adicional sobre las etiquetas.

Ejemplo de

<libro>
    <título>El Quijote</título>
    <autor>Miguel deervantes</autor>
    <año>1605</año>
</libro>

Sintaxis

  • Etiquetas deben estar bien formadas (cerrarlas correctamente).
  • No se permiten etiquetas mal cerradas.
  • No se pueden usar caracteres especiales sin codificación.
  • Los nombres de las etiquetas son sensibles a mayúsculas y minúsculas.

Ejemplo de Errores

<libro>
    <título>El Quijote</título>
    <autor>Miguel deervantes</autor>
    <año>1605</año>
</libro>

Características Especiales en

  • &: Ampersand.
  • <: Menor que.
  • >: Mayor que.
  • ": Comillas dobles.
  • ': Comillas simples.

Comentarios en

<!-- Este es un comentario -->

Ejemplo de Comentario

<libro>
    <título>El Quijote</título>
    <autor>Miguel deervantes</autor>
    <año>1605</año>
    <!-- Este es un comentario -->
</libro>

Namespaces en

Los namespaces permiten evitar conflictos de nombres al definir etiquetas similares en diferentes partes del documento.

Ejemplo de Namespace

<libro xmlns:dc="http://purl.org/dc/elements/1.1/">
    <título>El Quijote</título>
    <autor>Miguel deervantes</autor>
    <año>1605</año>
    <dc:fecha>1605</dc:fecha>
</libro>

Validación

La validación permite asegurar que un documento cumpla con un esquema predefinido.

Ejemplo de Esquema

<? version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
    <xs:element name="libro">
        <xs:complexType>
            <xs:sequence>
                <xs:element name="título" type="xs:string"/>
                <xs:element name="autor" type="xs:string"/>
                <xs:element name="año" type="xs:gYear"/>
            </xs:sequence>
        </xs:complexType>
    </xs:element>
</xs:schema>

Checklist

  • Declaración del Prologo.
  • Estructura jerárquica con etiquetas anidadas.
  • Uso de atributos para información adicional.
  • Codificación de caracteres especiales.
  • Comentarios en el documento.
