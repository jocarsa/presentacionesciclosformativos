XSLT (Extensible Stylesheet Language Transformations)

Descripción de XSLT y sus principales características.

Ejemplo básico de transformación con XSLT

<? version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method=""/>
  <xsl:template match="/libros">
    <ul>
      <xsl:for-each select="libro">
        <li><xsl:value-of select="titulo"/> - <xsl:value-of select="autor"/></li>
      </xsl:for-each>
    </ul>
  </xsl:template>
</xsl:stylesheet>

XPath ( Path Language)

Descripción de XPath y sus aplicaciones en la transformación.

Ejemplo de consulta XPath

/libros/libro[titulo='El Quijote']/autor

XQuery (XQuery for)

Descripción de XQuery y sus ventajas sobre XPath.

Ejemplo básico de consulta XQuery

xquery version "1.0";
for $libro in doc('biblioteca.xml')/libros/libro
where $libro/titulo = 'El Quijote'
return $libro/autor

Uso de XSLT para adaptar documentos

Cómo transformar un documento en otro con diferentes estructuras.

Ejemplo práctico: Adaptación de un documento

<? version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method=""/>
  <xsl:template match="/libros">
    <table>
      <tr><th>Título</th><th>Autor</th></tr>
      <xsl:for-each select="libro">
        <tr>
          <td><xsl:value-of select="titulo"/></td>
          <td><xsl:value-of select="autor"/></td>
        </tr>
      </xsl:for-each>
    </table>
  </xsl:template>
</xsl:stylesheet>

Buenas prácticas en la transformación

  • Validar los documentos antes de la transformación
  • Usar namespaces adecuadamente
  • Mantener el código XSLT legible y mantenible

Errores comunes en la transformación

  • No validar los documentos
  • Olvidar namespaces
  • Código XSLT mal estructurado y difícil de mantener
