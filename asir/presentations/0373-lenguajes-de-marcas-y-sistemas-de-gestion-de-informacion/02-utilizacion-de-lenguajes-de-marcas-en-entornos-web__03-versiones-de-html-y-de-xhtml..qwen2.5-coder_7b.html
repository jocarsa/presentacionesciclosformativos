Versiones de y XHTML en entornos Web

Objetivo: Comprender las principales versiones de y XHTML, sus características y diferencias.

4.01

  • La versión más ampliamente utilizada hasta principios del siglo XXI.
  • Incluye 3.2 y 4.0.
  • Soporta tablas, formularios y elementos multimedia básicos.
  • No es estricto (permite etiquetas mal cerradas).

X 1.0

  • Versionado de que se ajusta a las reglas estrictas del lenguaje.
  • Todas las etiquetas deben estar correctamente cerradas y formateadas.
  • Mejora la estructura y semántica del código.
  • No es tan ampliamente utilizado como 4.01.

5

  • Lanzado en 2012, reemplazando a 4.01 y XHTML 1.0.
  • Incluye nuevas etiquetas semánticas (header, footer, article, section).
  • Soporta multimedia nativo (video, audio), formularios avanzados y geolocalización.
  • Más estricto que 4.01 pero más flexible que XHTML 1.0.

Más semántica con nuevas etiquetas. Menos semántica, basado en 4.01.

Principales diferencias entre y XHTML

  • Sintaxis: es más flexible, XHTML estricto.
  • Cierre de etiquetas: permite etiquetas mal cerradas, XHTML requiereierre correcto.
  • Caracteres especiales: interpreta caracteres especiales como <, XHTML los codifica (&lt;).
  • Validación: XHTML se puede validar con, no.

Ejemplo de código XHTML 1.0

<!DOCTYPE PUBLIC "-//W3//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
< xmlns="http://www.w3.org/1999/xhtml" lang="es">
<head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    <title>Ejemplo XHTML 1.0</title>
</head>
<body>
    <header><h1>Encabezado</h1></header>
    <nav><a href="#">Enlace 1</a> | <a href="#">Enlace 2</a></nav>
    <section><p>Contenido principal.</p></section>
    <footer><p>Pie de página.</p></footer>
</body>
</>

Errores comunes en5

  • No cerrar etiquetas correctamente.
  • Uso de caracteres especiales sin codificación.
  • Mixto uso de mayúsculas y minúsculas en etiquetas.
  • Falta de metaetiqueta charset.

Errores comunes en XHTML 1.0

  • No cerrar etiquetas correctamente.
  • Uso de caracteres especiales sin codificación (&lt;, &gt;).
  • Mixto uso de mayúsculas y minúsculas en etiquetas.
  • Falta de metaetiqueta charset.

Buenas prácticas para5

  • Uso de etiquetas semánticas.
  • Cierre correcto de todas las etiquetas.
  • Validación del código con herramientas online.
  • Compatibilidad con navegadores modernos.

Buenas prácticas para XHTML 1.0

  • Cierre correcto de todas las etiquetas.
  • Validación del código con herramientas online.
  • Compatibilidad con navegadores modernos.
  • Uso de caracteres especiales codificados.
