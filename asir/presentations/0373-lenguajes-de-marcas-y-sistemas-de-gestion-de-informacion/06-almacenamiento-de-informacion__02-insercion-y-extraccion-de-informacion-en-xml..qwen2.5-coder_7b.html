Inserción y Extracción de Información en

Objetivo: Comprender los métodos para insertar y extraer información en formatos.

Introducción a

  • (eXtensible Markup Language) es un formato de texto que se utiliza para almacenar e interambiar datos.
  • Características: extensibilidad, independencia de software y plataforma, legibilidad humana.

Estructura Básica de

<? version="1.0" encoding="UTF-8"?>
<libro>
  <título>El Quijote</título>
  <autor>Miguel deervantes</autor>
  <año>1605</año>
</libro>

Inserción de Información en

  • Se puede insertar información utilizando lenguajes de programación como, o#.
  • Ejemplo con:
import.etree.ElementTree as ET

root = ET.Element("libro")
title = ET.SubElement(root, "título")
title.text = "El Quijote"
author = ET.SubElement(root, "autor")
author.text = "Miguel deervantes"
year = ET.SubElement(root, "año")
year.text = "1605"

tree = ET.ElementTree(root)
tree.write("libro.xml")

Extracción de Información en

  • Se puede extraer información utilizando lenguajes como XPath o XSLT.
  • Ejemplo con XPath:
import.etree.ElementTree as ET

tree = ET.parse("libro.xml")
root = tree.getroot()

title = root.find("título").text
author = root.find("autor").text
year = root.find("año").text

print(f"Título: {title}, Autor: {author}, Año: {year}")

Tipos de Datos en

  • String (texto)
  • Integer (número entero)
  • Float (número decimal)
  • Date (fecha)

Validación de

  • Se puede validar el contenido de un archivo utilizando DTD (Document Type Definition) o XSD ( Schema Definition).
  • Ejemplo con DTD:
<!ELEMENT libro (título, autor, año)>
<!ELEMENT título (#PCDATA)>
<!ELEMENT autor (#PCDATA)>
<!ELEMENT año (#PCDATA)>

Checklist de Buenas Prácticas

  • Usar etiquetas descriptivas y significativas.
  • Mantener una estructura jerárquica clara.
  • Validar el contenido del archivo.
  • Comentar el código cuando sea necesario.

Ejemplos de Errores Comunes

  • No cerrar correctamente las etiquetas.
  • Uso incorrecto de caracteres especiales.
  • Falta de validación del contenido.
