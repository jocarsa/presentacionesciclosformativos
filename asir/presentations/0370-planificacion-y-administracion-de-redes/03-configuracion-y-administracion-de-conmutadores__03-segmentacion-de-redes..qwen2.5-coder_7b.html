Segmentación de redes

La segmentación de redes es el proceso de dividir una red en subredes más pequeñas y manejables.

Beneficios de la segmentación

  • Mejora la seguridad al aislar áreas vulnerables
  • Aumenta el rendimiento al reducir el tráfico en una sola red
  • Facilita la gestión y escalabilidad de la infraestructura
  • Permite aplicar políticas de acceso más específicas

Técnicas de segmentación

  • Segmentación basada en VLANs (VLAN Trunking)
  • Segmentación basada en puertos
  • Segmentación basada en direcciones IP
  • Segmentación basada en aplicaciones

VLANs (VLAN Trunking)

Las VLANs permiten crear segmentos lógicos dentro de una red física.

Ejemplo de configuración de VLAN


        interface GigabitEthernet0/1
            switchport mode access
            switchport access vlan 10

Segmentación basada en puertos

Asigna puertos específicos a una VLAN.

Ejemplo de segmentación por puerto


        interface GigabitEthernet0/1
            switchport mode access
            switchport port-security maximum 2

Segmentación basada en direcciones IP

Asigna segmentos de red específicos a una VLAN.

Ejemplo de segmentación por dirección IP


        ip access-list extended VLAN_10
            permit ip 192.168.10.0 0.0.0.255 any
        interface GigabitEthernet0/1
            switchport mode access
            switchport access vlan 10

Segmentación basada en aplicaciones

Asigna segmentos de red a VLANs según el tipo de tráfico.

Ejemplo de segmentación por aplicación


        policy-map type inspect http HTTP_POLICY
            parameters
                class class-default
                    inspect http
        interface GigabitEthernet0/1
            service-policy input HTTP_POLICY

Consideraciones de seguridad

  • Implementa autenticación y autorización adecuada
  • Configura firewalls internos para controlar el tráfico entre VLANs
  • Monitorea y rastrea el tráfico en cada segmento

Ejemplo de firewall interno


        access-list 101 permit ip 192.168.10.0 0.0.0.255 192.168.20.0 0.0.0.255
        interface GigabitEthernet0/1
            ip access-group 101 in

Prácticas recomendadas

  • Documenta todas las configuraciones y segmentaciones realizadas
  • Realiza pruebas de conectividad y rendimiento en cada segmento
  • Mantén actualizadas las políticas de acceso y seguridad

Errores comunes

  • No asignar suficientes VLANs para todas las necesidades
  • No configurar correctamente los firewalls internos
  • No documentar adecuadamente las configuraciones realizadas

Mini-quiz

¿Cuál es el propósito de la segmentación basada en VLANs?

  1. Aumentar la velocidad de transmisión de datos
  2. Mejorar la seguridad al aislar áreas vulnerables
  3. Reducir el costo de la infraestructura física
  4. Facilitar la gestión y escalabilidad de la red

Checklist

  • Configurar VLANs según las necesidades del negocio
  • Implementar firewalls internos para controlar el tráfico entre VLANs
  • Mantener documentación actualizada de todas las configuraciones
  • Realizar pruebas de conectividad y rendimiento en cada segmento

Actividad

Configura una VLAN en un conmutador y configura un firewall interno para controlar el tráfico entre la VLAN y otra.

Resumen

  • La segmentación de redes es crucial para mejorar la seguridad, rendimiento y gestión de las infraestructuras de red.
  • Existen varias técnicas de segmentación como VLANs, puertos, direcciones IP y aplicaciones.
  • Es importante implementar prácticas recomendadas y prevenir errores comunes para un despliegue exitoso.

Normativa general

Seguir las normativas locales y nacionales de seguridad en redes es fundamental para cumplir con los requisitos legales.
