Dirección Dinámica (DHCP)

DHCP es un protocolo que asigna automáticamente direcciones IP a los dispositivos en una red.

Funcionamiento básico de DHCP

  • Solicita IP al servidor DHCP
  • El servidor DHCP responde con una oferta
  • El cliente acepta la oferta
  • El servidor confirma la asignación

Ventajas de DHCP

  • Reducción manual de configuraciones
  • Economía en tiempo y recursos
  • Facilita la gestión de cambios en la red

Desventajas de DHCP

  • Puede causar problemas si el servidor está sobrecargado
  • No es seguro para redes públicas
  • Puede asignar direcciones IP incorrectas

Configuración básica de DHCP en un router

1. Ingresar a la configuración del router
2. Buscar opciones de "DHCP Server"
3. Habilitar el servicio DHCP
4. Configurar el rango IP asignable

Ejemplo de configuración DHCP

IP Range: 192.168.1.10 - 192.168.1.100
Subnet Mask: 255.255.255.0
Default Gateway: 192.168.1.1

Caso de estudio: Implementación de DHCP en una pequeña empresa

Una empresa con 20 computadoras implementó DHCP para asignar automáticamente direcciones IP, reduciendo el tiempo de configuración y facilitando la gestión del inventario.

Proceso de asignación de IP por DHCP

  1. DHCPDISCOVER: El cliente envía una solicitud a todos los servidores DHCP disponibles
  2. DHCPOFFER: Un servidor DHCP responde con una oferta de dirección IP
  3. DHCPREQUEST: El cliente acepta la oferta y solicita la confirmación
  4. DHCPACK: El servidor confirma la asignación de la dirección IP

Buenas prácticas en DHCP

  • Configurar un rango IP adecuado para el tamaño de la red
  • Habilitar la asignación automática de puertas de enlace y DNS
  • Configurar tiempos de renovación y reasignación

Errores comunes en DHCP

  • Servidor DHCP sobrecargado
  • Direcciones IP asignadas incorrectamente
  • Tiempo de renovación y reasignación configurados incorrectamente
