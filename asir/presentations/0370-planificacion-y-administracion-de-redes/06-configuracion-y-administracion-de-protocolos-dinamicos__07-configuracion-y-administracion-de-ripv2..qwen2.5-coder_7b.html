Configuración y administración de RIPv2

Objetivo: Aprender cómo configurar y administrar el protocolo de enrutamiento RIPv2 en redes IP.

Introducción a RIPv2

  • RIP (Routing Information Protocol) es un protocolo de enrutamiento dinámico simple y eficiente.
  • RIPv2 es la versión 2 del protocolo RIP, que mejora la seguridad y la escalabilidad.
  • Características principales: autenticación5, límites de distancia (15 saltos), y soporte para subredes.

Configuración básica de RIPv2

  1. Habilitar el protocolo RIP en la interfaz deseada:
    2. router rip
 network 192.168.1.0
  2. Verificar la configuración y estado del proceso RIPv2:
    3. show ip protocols
 show ip route rip

Ejemplo de configuración en Cisco IOS

router rip
 version 2
 network 192.168.1.0
 network 10.0.0.0
 authentication5 key-chain RIPv2-KEY

Caso práctico: Implementación de RIPv2 en una red pequeña

  • Red A: 192.168.1.0/24
  • Red B: 192.168.2.0/24
  • Red: 192.168.3.0/24

Configurar RIPv2 en todas las redes para que se comuniquen entre sí.

Proceso de configuración paso a paso

  1. Habilitar RIP versión 2 en cada router.
  2. Configurar la autenticación5 para seguridad.
  3. Verificar el estado del proceso RIPv2 y las rutas aprendidas.

Buenas prácticas en la configuración de RIPv2

  • Usar autenticación5 para proteger contra falsos anuncios de enrutamiento.
  • Limitar el alcance de las rutas con límites de distancia (15 saltos).
  • Monitorear regularmente el estado del proceso RIPv2 y solucionar problemas.

Errores comunes en la configuración de RIPv2

  • No habilitar autenticación5, lo que expone a falsos anuncios.
  • No limitar el alcance de las rutas, lo que puede causar problemas de congestión.
  • Olvidar verificar el estado del proceso RIPv2 y solucionar problemas.
