Diagnóstico de incidencias de NAT

Objetivo: Aprender a identificar y solucionar problemas comunes en el funcionamiento del NAT.

¿Qué es NAT?

  • NAT (Network Address Translation)
  • Traduce direcciones IP internas a direcciones IP públicas
  • Permite que múltiples dispositivos compartan una única dirección IP

Tipos de NAT

  • NAT estándar (PAT)
  • NAT puerta única
  • NAT por puerto múltiple
  • NAT tradicional

Problemas comunes en NAT

  • No se puede acceder a los dispositivos internos desde Internet
  • Puertos bloqueados o no respondiendo
  • Direcciones IP incorrectas asignadas

Diagnóstico de problemas

  1. Verificar configuración del router
  2. Comprobar puertos abiertos con herramientas como nmap
  3. Revisar logs del router para errores
  4. Probar conexión desde una máquina externa

Ejemplo de diagnóstico

Supongamos que no se puede acceder a un servidor web en el interior de la LAN.

  • Verificar que el puerto 80 está abierto en el router
  • Probar conexión desde una máquina externa usando telnet [dirección IP] 80
  • Revisar logs del servidor web para errores de configuración

Caso práctico: NAT puerta única

Un usuario no puede acceder a su servidor FTP desde Internet.

  1. Configurar el router para abrir el puerto 21 (FTP)
  2. Verificar que el firewall del servidor permite conexiones entrantes en el puerto 21
  3. Probar conexión desde una máquina externa usando FTP client

Buenas prácticas en NAT

  • Usar NAT estándar (PAT) para mayor eficiencia
  • Mantener actualizado el firmware del router
  • Configurar correctamente los puertos y direcciones IP

Errores comunes en NAT

  • Configuración incorrecta de puertos
  • No asignar una dirección IP interna correcta
  • No habilitar el puerto forwarding en el router

  • Verificar configuración del router

  • Probar conexión desde una máquina externa

Actividad: Diagnóstico de NAT

Realizar el diagnóstico de un problema de acceso a Internet desde la LAN.

  1. Identificar el problema (no se puede acceder a un servidor web)
  2. Diagnóstico paso a paso
  3. Corregir el problema y verificar solución

Resumen

  • NAT es una técnica de traducción de direcciones IP
  • Problemas comunes incluyen no acceso a dispositivos internos y puertos bloqueados
  • Diagnóstico implica verificar configuración, puertos abiertos y logs del router

Normativa

No hay normativas específicas para NAT en español.

Se recomienda seguir las mejores prácticas recomendadas por fabricantes de dispositivos y organizaciones de seguridad informática.
