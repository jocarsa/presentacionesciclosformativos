Enrutamiento sin clase (CIDR)

  • Clase A, B y no utilizadas
  • Uso de prefijos IP para definir redes
  • Más eficiente que el enrutamiento por clases
  • Permite una mayor flexibilidad en la asignación de direcciones IP

Ejemplo de CIDR

Red: 192.168.0.0/24

  • 254 direcciones IP disponibles para hosts
  • Subredes más pequeñas pueden ser creadas fácilmente
  • Ejemplo de subred: 192.168.0.0/27 (30 hosts)

Caso práctico

Una empresa con 5 oficinas necesita conectarse a Internet.

  • CIDR: 192.168.0.0/24
  • Oficina principal: 192.168.0.0/27
  • Oficina secundaria: 192.168.0.32/27

Proceso de configuración CIDR

  1. Determinar la cantidad total de direcciones IP necesarias
  2. Elegir el prefijo adecuado para la red
  3. Asignar subredes según las necesidades de los usuarios
  4. Configurar los dispositivos de red con las nuevas direcciones CIDR

Buenas prácticas en CIDR

  • No asignar subredes muy pequeñas o grandes
  • Evitar la fragmentación de direcciones IP
  • Mantener un registro detallado de las direcciones asignadas

Errores comunes en CIDR

  • Asignar subredes que superan la capacidad disponible
  • No considerar las necesidades futuras de crecimiento
  • No documentar adecuadamente el plan de asignación de direcciones

Comparativa con enrutamiento por clases

Checklist de configuración CIDR

  1. Determinar el número total de direcciones IP necesarias
  2. Elegir el prefijo CIDR adecuado
  3. Asignar subredes según las necesidades
  4. Configurar los dispositivos de red
  5. Documentar la configuración

Actividad práctica

Asigne direcciones CIDR a las siguientes redes:

  • 10 usuarios en una oficina
  • 50 usuarios en otra oficina
  • 100 usuarios en la sede principal

Resumen

  • El enrutamiento sin clase (CIDR) es una técnica eficiente y flexible para asignar direcciones IP.
  • Permite una mayor eficiencia en la administración de redes.
  • Es recomendable utilizar CIDR en lugar del enrutamiento por clases para nuevas implementaciones.

Normativa general

La normativa ISO/IE 30109 establece las mejores prácticas para la administración de redes IP, incluyendo el uso de CIDR.
