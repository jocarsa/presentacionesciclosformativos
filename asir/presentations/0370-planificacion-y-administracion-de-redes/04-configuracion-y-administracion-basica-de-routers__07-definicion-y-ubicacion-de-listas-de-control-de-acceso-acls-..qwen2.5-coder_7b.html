Definición y ubicación de listas de control de acceso (ACLs)

Objetivo: Comprender qué son las ACLs, cómo se definen y dónde se aplican en los routers.

¿Qué son las Listas de Control de Acceso?

  • Conjunto de reglas que controlan el tráfico de red.
  • Pueden ser implícitas (por defecto) o explícitas (configuradas por el usuario).
  • Sirven para filtrar paquetes de red según criterios específicos.

Tipos de ACLs

  • ACL Extendida: Permite especificar criterios más detallados (por ejemplo, protocolo, puerto).
  • ACL Estándar: Filtra tráfico basado en el número IP del origen o destino.

Ejemplo de ACL Extendida

access-list 101 permit tcp any host 192.168.1.1 eq 80
access-list 101 deny ip any any

Permite tráfico TCP al puerto 80 desde cualquier origen, pero deniega todo el resto.

Ejemplo de ACL Estándar

access-list 10 permit ip 192.168.1.0 0.0.0.255 any
access-list 10 deny ip any any

Permite tráfico IP desde la red 192.168.1.0/24, pero deniega todo el resto.

Ubicación de las ACLs en los Routers

  • ACL Extendida: Pueden aplicarse a interfaces o a subinterfaces.
  • ACL Estándar: Generalmente se aplican a interfaces.

Ejemplo de Aplicación de ACL en una Interfaz

interface GigabitEthernet0/1
ip access-group 101 in

Aplica la ACL 101 a la interfaz GigabitEthernet0/1 para filtrar el tráfico entrante.

Ejemplo de Aplicación de ACL en una Subinterfaz

interface Vlan1
ip address 192.168.1.1 255.255.255.0
ip access-group 10 in subinterface GigabitEthernet0/1

Aplica la ACL 10 a la subinterfaz Vlan1 para filtrar el tráfico entrante.

Buenas Prácticas al Usar ACLs

  • Minimizar el número de ACLs para evitar problemas de rendimiento.
  • Ordenar las reglas de la ACL de mayor a menor especificidad.
  • Usar nombres descriptivos para las ACLs y sus reglas.

Errores Comunes al Usar ACLs

  • No ordenar las reglas de la ACL correctamente, lo que puede causar el descarte prematuro de paquetes.
  • No aplicar las ACLs a las interfaces correctas, lo que puede resultar en filtrado incorrecto del tráfico.

Mini-Quiz

¿Cuál es la diferencia entre una ACL Extendida y una ACL Estándar?

  1. ACL Extendida filtra por número IP, mientras que ACL Estándar filtra por protocolo.
  2. ACL Extendida filtra por protocolo, mientras que ACL Estándar filtra por número IP.
  3. No hay diferencia entre ambos tipos de ACLs.

Checklist para Configurar ACLs

  • Identificar el tipo de ACL necesario (Extendida o Estándar).
  • Especificar los criterios de filtrado en las reglas.
  • Aplicar la ACL a la interfaz o subinterfaz correcta.

Actividad: Configurar una ACL Extendida

Cree una ACL Extendida que permita tráfico ICMP solo desde la red 10.0.0.0/8 y deniega todo el resto.

Actividad: Aplicar una ACL Estándar

Configure una ACL Estándar que permita tráfico IP desde la red 172.16.0.0/16 a cualquier destino.

Resumen

  • ACLs son herramientas esenciales para filtrar el tráfico de red en los routers.
  • Existen dos tipos principales: ACL Extendida y ACL Estándar.
  • Las ACLs se aplican a interfaces o subinterfaces según sea necesario.
