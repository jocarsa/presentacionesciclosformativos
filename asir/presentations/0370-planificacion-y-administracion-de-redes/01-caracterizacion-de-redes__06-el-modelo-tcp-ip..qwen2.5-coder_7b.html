El modelo TCP/IP

Objetivo: Comprender la estructura y funcionamiento del modelo TCP/IP.

Introducción al Modelo TCP/IP

  • Modelo de protocolos de red utilizado por Internet y muchas redes privadas.
  • Divide la comunicación en capas para facilitar su diseño, implementación y mantenimiento.
  • Cinco capas principales: Aplicación, Transporte, Internet, Enlace de datos y Física.

Capa 1: Capa de Aplicación

  • Se encarga de las aplicaciones que utilizan la red (ej. navegador web, correo electrónico).
  • No es un protocolo específico, sino una interfaz entre el software de aplicación y los protocolos de red.

Capa 2: Capa de Transporte

  • Gestiona la transmisión confiable de datos entre aplicaciones en diferentes dispositivos.
  • Protocolos comunes: TCP (orientado a conexión) y UDP (no orientado a conexión).

Capa 3: Capa de Internet

  • Se encarga del enrutamiento de paquetes de datos entre diferentes redes.
  • Protocolo principal: IP (Internet Protocol).

Capa 4: Capa de Enlace de Datos

  • Gestiona la transmisión de datos entre dispositivos conectados a la misma red.
  • Protocolos comunes: Ethernet, Wi-Fi.

Capa 5: Capa Física

  • Se encarga de la transmisión física de los datos a través del medio de comunicación.
  • Incluye el hardware que permite la conexión entre dispositivos (ables, adaptadores).

Ejemplo Práctico: Transferencia de un Archivo

1. Aplicación envía datos al usuario.
2. Transporte confirma la recepción y solicita retransmisión si es necesario.
3. Internet enruta el paquete a través de diferentes enrutadores.
4. Enlace de datos transmite los datos a través del cable Ethernet.
5. Física envía los bits físicamente.

Caso Real: Implementación de una Red Local

  • Configuración de la capa de aplicación (servicios web).
  • Configuración de la capa de transporte (protocolo TCP).
  • Configuración de la capa de internet (asignación de IP).
  • Configuración de la capa de enlace de datos (configuración de red local).
  • Configuración de la capa física (conexión de dispositivos).

Proceso de Enrutamiento

1. Paquete enviado desde el origen.
2. Capa Internet examina la dirección IP del paquete.
3. Selecciona el siguiente enrutador según las tablas de enrutamiento.
4. El paquete se envía al siguiente enrutador.
5. Este proceso se repite hasta que el paquete llega a su destino.

Buenas Prácticas: Seguridad en TCP/IP

  • Uso de protocolos seguros (HTTPS, SSH).
  • Cifrado de datos en tránsito.
  • Autenticación y autorización de usuarios.
  • Monitoreo y detección de anomalías.

Errores Comunes en TCP/IP

  • Puerto no disponible (Port Unreachable).
  • Dirección IP incorrecta (Destination Host Unreachable).
  • Tiempo de espera excedido (Time Exceeded).
  • Paquete corrupto (Bad Packet).

Comparativa: TCP vs UDP

TCP UDP
Orientado a conexión. No orientado a conexión.
Fiable (confiabilidad). No confiable (no confiabilidad).
Reconoce errores y solicita retransmisión. No reconoce errores, no solicita retransmisión.
Manejo de flujo de datos. No maneja flujo de datos.

Checklist: Configuración Básica de TCP/IP

  1. Configurar la dirección IP.
  2. Configurar la máscara de subred.
  3. Configurar la puerta de enlace predeterminada.
  4. Configurar el servidor DNS.
  5. Verificar la conectividad con ping.

Actividad: Crear una Red Virtual

Crea una red virtual utilizando software como VirtualBox o VMware, y configura las capas TCP/IP correspondientes.

Resumen del Modelo TCP/IP

  • Modelo de cinco capas que divide la comunicación en partes para facilitar su diseño y implementación.
  • Cada capa tiene un protocolo específico encargado de una función particular.
  • La comunicación fluye desde la capa de aplicación hasta la capa física.
