Objetivo: Comprender los conceptos básicos de dominios de colisión y «broadcast» en conmutadores.

Definición de Dominio de Colisión

Un dominio de colisión es un área de red donde dos o más dispositivos pueden intentar enviar datos al mismo tiempo, causando una colisión.

Ejemplo de Dominio de Colisión

  • Un cable Ethernet simple entre dos computadoras.
  • Una red con varios dispositivos conectados a un solo conmutador.

Definición de «Broadcast»

El término «broadcast» se refiere al envío de datos a todos los dispositivos en una red, sin necesidad de especificar un destino individual.

Ejemplo de «Broadcast»

  • Un paquete IP con dirección de destino como "255.255.255.255".
  • La transmisión de un mensaje a todos los usuarios en una red local.

Impacto de la Colisión y el «Broadcast»

Colisión: Reduce la velocidad de transmisión y puede causar errores.

Broadcast: Puede sobrecargar la red, especialmente en redes con muchos dispositivos.

Prevenir la Colisión

  • Usar segmentación de red (subredes).
  • Implementar protocolos de control de colisiones como CSMA/CD.

Prevenir el «Broadcast»

  • Usar segmentación de red (subredes).
  • Configurar filtros de paquetes para limitar el «broadcast».

Buenas Prácticas

  • Evitar la configuración de redes en modo promiscuo.
  • Limitar el uso de «broadcast» en aplicaciones no necesarias.

Errores Comunes

  • No segmentar la red adecuadamente, causando colisiones y sobrecarga.
  • No configurar correctamente los filtros de paquetes para el «broadcast».

Comparativa

Colisión Broadcast

Resumen

  • Dominio de colisión: Área de red donde se producen colisiones.
  • Broadcast: Envío de datos a todos los dispositivos en la red.
  • Prevenirlos: Segmentación, CSMA/CD y filtros de paquetes.
