El modelo OSI: Caracterización de redes

Objetivo: Comprender la estructura y funcionalidades del modelo OSI en el diseño de redes.

Introducción al Modelo OSI

  • Modelo OSI (Open Systems Interconnection) es un estándar para la comunicación entre sistemas informáticos.
  • Fue propuesto por la Agencia Internacional de Normalización (ISO) en 1984.
  • Define una arquitectura de red que divide las funciones del sistema en siete capas interconectadas.

Estructura del Modelo OSI

Las capas son: Aplicación, Presentación, Sesión, Transporte, Red, Enlace de datos y Física.

Capa 1: Aplicación

  • Interfaz directa con los usuarios finales.
  • Ejemplos: Protocolos de correo electrónico, FTP, HTTP.
  • Proporciona servicios como la autenticación y el control de acceso.

Capa 2: Presentación

  • Formato, encriptación y compresión de datos.
  • Ejemplos: SSL/TLS para la seguridad web, JPEG para imágenes.
  • Se encarga de convertir los datos entre el formato interno del sistema y el formato que se enviará a través de la red.

Capa 3: Sesión

  • Establece, mantiene y termina las sesiones entre los sistemas.
  • Ejemplos: SSH para la conexión segura, FTP para el interambio de archivos.
  • Gestiona la autenticación, autorización y sincronización de datos.

Capa 4: Transporte

  • Especifica cómo se envían los datos de una capa a otra.
  • Ejemplos: TCP para transferencias confiables, UDP para transmisiones rápidas.
  • Gestiona la segmentación y reconstrucción de paquetes, así como el control de flujo y errores.

Capa 5: Red

  • Encapsula los datos en paquetes y los envía a través de la red.
  • Ejemplos: IP para ruteo, ICMP para errores.
  • Gestiona el enrutamiento de paquetes entre diferentes redes.

Capa 6: Enlace de datos

  • Especifica cómo se envían los bits a través del medio físico.
  • Ejemplos: Ethernet, Wi-Fi.
  • Gestiona la transmisión de bits y la detección de errores.

Capa 7: Física

  • Especifica los medios físicos y las señales que se utilizan para la transmisión de datos.
  • Ejemplos:ables Ethernet, señales radiofrecuencia.
  • Gestiona el hardware y las interfaces de red.

Comparación de Protocolos

  • b) Formatear, encriptar y compresionar los datos.
  • c) Establecer, mantener y terminar las sesiones entre sistemas.
  • d) Segmentar y reconstruir paquetes de datos.

Checklist

  • Comprender la estructura del Modelo OSI.
  • Distinguir las funciones principales de cada capa.
  • Comparar el Modelo OSI con otros modelos como TCP/IP.

Actividad Práctica

Resumen

  • El Modelo OSI es una estructura estándar para la comunicación entre sistemas informáticos.
  • Define siete capas interconectadas que realizan funciones específicas en el proceso de transmisión de datos.
  • Cada capa tiene un rol crucial en el funcionamiento de una red.

Normativa y Reglamentos

ISO/IE 7498-1:2015 es la norma internacional que define el Modelo OSI.
